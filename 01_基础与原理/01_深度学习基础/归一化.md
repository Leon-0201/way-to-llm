# 归一化（Normalization）

本章是深度学习基础中的核心内容，系统梳理两类完全不同的归一化技术：数据预处理层面的数据归一化与模型结构层面的网络层归一化，是面试高频考点与工程实践必备知识。
**内容脉络**：先从总览层面区分两类归一化的定位与作用差异，再分别讲解数据归一化的常见方法、数据泄漏问题、LLM场景下的文本规范化特殊要求，随后深入讲解网络层归一化的通用框架、BN/LN/RMSNorm等主流方法的原理与差异、Pre-Norm/Post-Norm等工程实践要点，最后整理面试高频考点。
**核心收益**：通过本章学习，你将能够清晰区分两类归一化的本质差异，掌握各类归一化方法的选型与调参技巧，解决训练中的数值稳定问题，从容应对相关面试问题。

> 本章核心定位：把“数据归一化（Data Normalization）”与“网络层归一化（Layer/Activation Normalization）”分开讲清：各自解决什么问题、怎么做、常见坑点与面试问法。

## 1 总览：两类“归一化”分别在解决什么
### 1.1 数据归一化（输入/特征预处理）
- 目标：让**输入特征的尺度与分布更适合优化**，减少某些维度主导梯度、提升训练稳定性与收敛速度。
- 发生位置：模型外（DataLoader / preprocessing pipeline），通常是 **offline/online 预处理**。

### 1.2 网络层归一化（层内/激活/残差路径上的归一化）
- 目标：让**中间激活/残差流**在训练过程中保持稳定尺度，改善深层网络的可训练性与数值稳定。
- 发生位置：模型内（如 BN/LN/RMSNorm 模块），属于网络结构的一部分。

## 2 数据归一化（Data Normalization）需要覆盖的知识点
### 2.1 为什么做数据归一化
- **优化稳定**：不同特征量纲差异大时，梯度下降会“走之字形”，收敛慢；缩放后更像各向同性。
- **学习率与初始化更好调**：输入尺度受控，网络前几层的激活与梯度更可控。
- **数值稳定**：避免极端取值导致溢出/下溢，尤其在混合精度或对数/指数变换附近。

### 2.2 常见方法（要会说清楚公式/适用场景）
- **标准化（Z-score / Standardization）**：
	- $x' = \frac{x-\mu}{\sigma}$
	- 适合“近似高斯”或你希望特征零均值单位方差的场景。
- **Min-Max 缩放**：
	- $x' = \frac{x-\min}{\max-\min}$（常映射到 $[0,1]$ 或 $[-1,1]$）
	- 对离群点敏感；适合已知上下界或分布稳定的数据。
- **Robust scaling**：
	- 用 median / IQR（四分位距）做缩放，抗离群点。
- **对数/幂变换**：
	- 如 $x' = \log(1+x)$ 处理长尾分布；需注意 0/负值。
- **向量归一化（L2 norm）**：
	- $x' = \frac{x}{\lVert x \rVert_2}$；常见于检索/度量学习或把输入当“方向”而非“幅值”的任务。

### 2.3 统计量如何估计（面试很爱问“数据泄漏”）
- **只用训练集估计** $\mu/\sigma$（或 min/max、median/IQR），验证/测试/线上用同一套统计量。
- **数据泄漏（Data Leakage）**：用全量数据统计会把测试信息泄漏到训练，评估偏乐观。
- **分布漂移**：线上分布变了，固定统计量可能失效；可讨论“周期性重估”或“滑动窗口”。

### 2.4 训练与推理的一致性
- 数据归一化必须保证 **训练/验证/推理使用同样的变换与参数**。
- 线上处理要可复现：保存 scaler 参数（如 `mean/std`、`min/max`）。

### 2.5 不同数据形态的常见做法（点到为止即可）
- **图像（少量了解即可）**：像素缩放到 $[0,1]$，再按通道减均值除方差。
- **数值特征表格（少量了解即可）**：连续特征 standardize/robust；类别特征 one-hot/embedding（严格不算归一化，但常被一起问）。
- **文本（LLM 面试重点）**：通常不做“按维度标准化”的数值归一化，而是做**文本清洗/规范化（text normalization）**，以提升数据一致性与可学习性。

### 2.6 文本清洗/规范化（Text Normalization，LLM 高频）
目标：减少“同义但不同写法”的表面差异、剔除噪声/不可见字符、让训练与推理输入在分布上尽量一致。

- **Unicode 规范化**：
	- 统一到 NFC/NFKC（常见做法是 NFKC 以合并兼容字符；是否使用取决于对语义/排版敏感度）。
	- 处理全角/半角、兼容等价字符（如某些数学字母、特殊空格）。
- **空白与换行**：
	- 合并连续空白、统一 `\r\n`/`\r`/`\n`、去除行尾空格；保留还是折叠换行要看任务（代码/诗歌等对换行敏感）。
- **大小写与标点**：
	- 是否 lowercasing（英文任务常见；多语种/实体名敏感场景要谨慎）。
	- 统一引号/破折号等标点的多种形态（“ ” vs " "，— vs -）。
- **控制字符与不可见字符**：
	- 移除或替换 `\u0000` 等控制字符、零宽字符（ZWJ/ZWNJ/ZWSP），避免训练噪声与 tokenization 异常。
- **HTML/富文本**：
	- 是否 strip HTML、转义/反转义、保留正文（新闻/网页语料常见）。
- **数字/日期/单位**（可选，按任务）：
	- 将不同格式统一（如 `2026-03-05` vs `03/05/2026`），或用占位符（`<NUM>`）降低稀疏性。
	- 注意：过度规范化可能损失关键信息（金额、版本号、代码等）。
- **重复与近重复**（更像“规范化/清洗”而非归一化，但面试常连问）：
	- 文档级去重、段落级去重、模板化内容清理（降低过拟合与记忆）。

### 2.7 与分词器（Tokenizer）的关系（LLM 高频）
- 文本规范化会改变 token 分布：
	- 例如统一空白、全角半角会显著影响 BPE/Unigram 的切分与词表覆盖。
- 一般原则：**先做确定性的 text normalization，再分词**；并确保训练与线上推理走同一条规范化链路。

### 2.8 “训练集统计的一致性”在 LLM 里的典型落点
虽然 LLM 很少做传统意义上的 `mean/std` 特征缩放，但“统计一致性”的思想仍然重要，体现在：
- **仅用训练语料拟合/确定规则**：例如去重阈值、语言识别规则、质量过滤阈值、长度截断策略、特殊符号映射表。
- **验证/测试/线上复用同一套规则与参数**：避免数据泄漏与评估偏乐观。
- **记录版本与可复现性**：规范化规则、正则、黑白名单、映射表都需要版本化；否则线上输入分布会悄悄漂移。

### 2.9 数据归一化 vs BN/LN 的关系（要能一句话分清）
- 数据归一化是**输入预处理**，固定变换；网络层归一化是**模型内部**动态/可学习的一部分。
- 做了数据归一化并不意味着不需要 BN/LN；两者作用位置与对象不同。
- **能不能互相替代**：通常**不能**。
	- 数据归一化无法替代 BN/LN：它只对输入做一次性变换，无法随训练过程动态适配层内激活的尺度漂移。
	- BN/LN 也无法替代数据清洗/规范化：尤其在 LLM 场景，text normalization 影响分词与 token 分布，这是模型内归一化解决不了的。

## 3 网络层归一化（Layer/Activation Normalization）需要覆盖的知识点
### 3.1 为什么需要网络层归一化
- **优化视角**：缓解层间激活尺度漂移，提升梯度下降的稳定性与收敛速度。
- **梯度/数值稳定**：避免激活值或梯度在层间逐步放大/缩小，降低溢出/下溢风险（尤其混合精度）。
- **尺度不变性（invariance）**：某些归一化带来对输入缩放的鲁棒性，使训练对学习率、初始化更不敏感。
- **正则化效应**：以 Batch 为单位的统计量引入噪声，可能带来类似正则化的效果（典型是 BatchNorm）。

### 3.2 归一化的通用模板（建议先统一语言）
多数归一化都可以写成：

$$\hat{x} = \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}}, \quad y = \gamma \hat{x} + \beta$$

#### 3.2.1 统计量沿哪些维度计算：一张表背下来就够
先约定两类最常见的激活形状：

- CNN 常见：$x\in\mathbb{R}^{N\times C\times H\times W}$（或 1D/3D 卷积把 $H,W$ 换成 $L$ / $D,H,W$）
- Transformer 常见：$x\in\mathbb{R}^{B\times T\times D}$（batch、token、hidden）

核心：**归一化就是对某些维度求均值/方差（或均方根），然后对剩下的维度做逐元素缩放**。

| 方法 | 典型输入 | 统计量怎么求（reduce 轴） | 是否依赖 batch | 训练/推理是否一致 |
| --- | --- | --- | --- | --- |
| BatchNorm（BN） | $N\times C\times H\times W$ | 对每个通道 $c$，沿 $N,H,W$ 求 $\mu_c,\sigma_c^2$ | 是 | 否（推理用 running stats） |
| LayerNorm（LN） | $B\times T\times D$ | 对每个 token 向量，沿 $D$ 求 $\mu,\sigma^2$（不跨 token、不跨 batch） | 否 | 是 |
| RMSNorm | $B\times T\times D$ | 对每个 token 向量，沿 $D$ 求 RMS（通常不减均值） | 否 | 是 |
| GroupNorm（GN） | $N\times C\times H\times W$ | 对每个样本，把通道分成 $G$ 组；每组沿 $C/G,H,W$ 求统计量 | 否 | 是 |
| InstanceNorm（IN） | $N\times C\times H\times W$ | 对每个样本每个通道，沿 $H,W$ 求统计量 | 否 | 是（常见实现也可带 running stats，但默认不依赖 batch） |

记忆口诀：

- BN：**跨 batch**（+ 空间），按**通道**归一化
- LN/RMSNorm：**不跨 batch、不跨 token**，按**hidden**归一化
- GN：不跨 batch，按“**通道分组** + 空间”归一化
- IN：不跨 batch，按“**每通道每样本** + 空间”归一化

##### 3.2.1.1 用 NLP/LLM 的张量形状理解 BN / GN / IN（面试常问“沿哪个维度”）
在语言模型里，中间激活最常见形状是 $x\in\mathbb{R}^{B\times T\times D}$（batch、token、hidden）。

为了把 BN/GN/IN 的“通道维”概念搬过来，通常把 **hidden 维 $D$ 当作 channel**，把 **token 维 $T$ 当作 1D 的 length/space**。

> PyTorch 里 `BatchNorm1d/InstanceNorm1d/GroupNorm` 往往按 `N×C×L`（或 `N×C`）来理解，因此工程上经常需要把 $B\times T\times D$ 转成 $B\times D\times T$ 再做归一化。

- **如果把 BN（BatchNorm1d）硬套到 $B\times T\times D$**：
	- 以每个 hidden 维 $d$ 为一组统计量，沿 **batch + token** 聚合：对每个 $d$，reduce 轴是 $(B, T)$。
	- 等价写法（省略 affine 与 $\epsilon$）：

	$$\mu_d = \frac{1}{BT}\sum_{b=1}^{B}\sum_{t=1}^{T} x_{b,t,d},\quad \sigma_d^2 = \frac{1}{BT}\sum_{b,t}(x_{b,t,d}-\mu_d)^2$$

	- 直觉：把“同一个特征维度”在不同样本、不同 token 位置上的值当作一批来标准化。

- **如果把 IN（InstanceNorm1d）套到 $B\times T\times D$**：
	- 对每个样本 $b$、每个通道/维度 $d$，只沿 **token 维** 聚合：reduce 轴是 $(T)$。
	- 也就是对每个 $(b,d)$ 计算：

	$$\mu_{b,d} = \frac{1}{T}\sum_{t=1}^{T} x_{b,t,d},\quad \sigma_{b,d}^2 = \frac{1}{T}\sum_{t}(x_{b,t,d}-\mu_{b,d})^2$$

	- 直觉：把一个样本内部“整段序列”的每个通道沿时间轴标准化。

- **如果把 GN（GroupNorm）套到 $B\times T\times D$**：
	- 把 $D$ 个通道分成 $G$ 组，每组大小约为 $D/G$。
	- 对每个样本 $b$、每个组 $g$，沿 **(组内通道维 + token 维)** 聚合：reduce 轴是 $(D/G, T)$（不跨 batch）。
	- 直觉：每个样本内部，对“通道分组后的子空间 + 序列长度”做标准化。

**为什么它们在 LLM 里不常用（你可以在面试里补一句）**
- **变长与 padding**：BN/GN/IN 在 token 维 $T$ 上做统计时，padding token 会污染均值/方差；做 mask-aware 的归一化会显著复杂化实现。
- **训练/推理不一致**：BN 需要 running stats，而自回归推理常是 $B=1$ 且 token 是逐步生成的，统计量与训练分布很难对齐。
- **并行与切分维度**：当 $D$ 做张量并行切分时，BN/GN 这类以“通道”为核心的归一化更容易引入跨卡聚合需求；而 LN/RMSNorm 的生态（fused kernel + 工程实践）更成熟。

#### 3.2.2 可学习参数 $\gamma$（scale）与 $\beta$（shift）到底解决什么
如果只做标准化 $\hat{x}$，会强行把表示限制在固定尺度/均值附近。$\gamma,\beta$ 让网络把“该归一化带来的好处”与“该有的表达自由度”解耦：

- **$\gamma$（缩放）**：让模型在归一化后仍能学回合适的幅值（否则所有层输出都被钉在 unit variance 量级，表达与优化灵活性都会受限）。
- **$\beta$（平移）**：让模型能学回需要的偏置/均值（否则很多“有用的偏置模式”会被强行去掉）。

实现层面常考点：

- BN：$\gamma,\beta$ 通常是**按通道**一维参数（$C$ 维）。
- LN：$\gamma,\beta$ 通常是**按 hidden 维**一维参数（$D$ 维）。
- RMSNorm：不少实现**只保留 $\gamma$**，不设 $\beta$（因为它不做去均值，均值项本身仍可通过残差/线性层表示；是否需要 $\beta$ 依架构与实现而定）。

#### 3.2.3 为什么必须有 $\epsilon$（尤其混合精度）
$\epsilon$ 的唯一目的就是：**防止分母过小导致除零/数值爆炸**。

- 当某些维度方差非常小（例如激活几乎常数、或被 mask/裁剪后大量相同值），$\sigma^2\approx 0$ 会让 $\frac{1}{\sqrt{\sigma^2}}$ 爆掉。
- FP16/BF16 下，方差/均值的累加精度更敏感，$\epsilon$ 过小更容易出现 NaN/Inf。

工程常见取值（记“数量级”，别死背某个唯一值）：

- BN：常见 $\epsilon\sim 10^{-5}$
- LN/RMSNorm：常见 $\epsilon\sim 10^{-5}$ 或 $10^{-6}$；混合精度不稳时通常**先尝试略增大 $\epsilon$**，或保证统计量在 FP32 中计算（fused kernel 往往会这么做）。

#### 3.2.4 训练 vs 推理差异：谁会缓存统计量
- **BatchNorm**：训练用当前 batch 统计量；推理用 **running mean/var**（训练期间维护的滑动平均）。
- **LayerNorm / RMSNorm / GroupNorm / InstanceNorm（常见默认）**：不依赖 batch 统计，训练与推理**形式一致**（没有 running stats 这条分支）。

#### 3.2.5 成本与并行：最大的坑在“跨设备同步”和“并行切分维度”
- **BN 的主要成本**：需要对 $N,H,W$ 做 reduction；分布式小 batch 时如果要更稳定统计量，会用 **SyncBN** 把统计量跨设备 all-reduce（带来额外通信与延迟）。
- **LN/RMSNorm 的主要成本**：对每个 token 的 $D$ 维做 reduction（均值/方差或 RMS）。
	- 在单卡上通常可用 fused kernel（如 fused layernorm/rmsnorm）把 reduction + 仿射变换融合，降低访存。
		- fused kernel：把原本分成多个算子/多个 kernel 的步骤合并成一次 kernel 执行，减少中间张量反复读写显存（访存）与 kernel launch 开销。
	- 在张量并行（tensor parallel）把 hidden 维 $D$ 切到多卡时，统计量需要跨卡聚合（通常是 all-reduce 求 sum / sumsq），否则每卡只看到局部 $D$ 会算错归一化。

### 3.3 Batch Normalization（BatchNorm, BN）
#### 3.3.1 定义与统计维度
- 常见于 CNN：对每个通道（channel）在 **batch + 空间维** 上计算均值与方差。

$$\mu_c = \frac{1}{m}\sum_{i=1}^{m} x_{i,c}, \quad \sigma_c^2 = \frac{1}{m}\sum_{i=1}^{m} (x_{i,c}-\mu_c)^2$$

#### 3.3.2 训练/推理行为（面试高频）
- **训练**：用当前 mini-batch 的 $\mu,\sigma^2$ 做归一化。
- **推理**：用训练过程中维护的 **running mean/var（滑动平均）**。
- running 统计量怎么更新（记住这是个 **EMA** 就行）：
	- 设 batch 统计量为 $\mu_{batch},\sigma^2_{batch}$，running 为 $\mu_{run},\sigma^2_{run}$，则常见形式：

	$$\mu_{run}\leftarrow (1-m)\mu_{run} + m\mu_{batch},\quad \sigma^2_{run}\leftarrow (1-m)\sigma^2_{run} + m\sigma^2_{batch}$$

	- 其中 $m$ 就是很多框架里叫 `momentum` 的那个系数（注意：不同框架对 “momentum” 的命名/含义可能有差异，但**本质都是滑动平均**）。
	- PyTorch 里：`momentum=None` 时会变成累积平均（cumulative moving average）；`track_running_stats=False` 时不会维护 running stats，推理阶段也会继续用 batch 统计量（这在部署时通常不想要）。

#### 3.3.3 BN 的优势与局限
- 优势：训练更稳、可用更大学习率、对初始化更不敏感，CNN 上经验效果好。
- 局限：
	- **小 batch 不稳定**：统计噪声大，性能下降。
	- **分布式**：每卡 batch 太小会需要 **SyncBN**（有通信开销）。
	- **序列建模不友好**：batch/序列长度变化、微批训练、推理单样本等场景统计量不匹配。

#### 3.3.4 工程点（建议要会说）
- **SyncBN**：把各卡的 batch 统计量做同步（all-reduce）得到“全局 batch”的均值/方差；解决每卡 batch 太小导致统计噪声大的问题，但会引入通信开销。
- **GhostBN**：把一个大 batch 在统计上切成多个“虚拟小 batch”分别算统计量（增加噪声、带来类似正则化效果）；常用于 batch 很大时的泛化/稳定折中。
- **梯度累积 ≠ BN 统计量变大**：BN 的统计量来自每次 forward 的 micro-batch；梯度累积只是把多个 step 的梯度累加，并不会让 BN 的 $\mu/\sigma^2$ 自动变成“更大的有效 batch”。
- **`track_running_stats` / `momentum`（PyTorch）**：
	- `track_running_stats=True` 才会在推理用 running stats。
	- `momentum` 控制 running stats 更新速度：太小适应慢、太大噪声大；分布漂移或强数据增强时更要关注它。
- **与数据增强/Dropout 的交互**：强增强会改变激活分布，导致 running stats 偏移；常见做法是更谨慎的增强策略、足够长的 warmup、或在特定阶段冻结 BN 统计量。

### 3.4 Layer Normalization（LayerNorm, LN）
#### 3.4.1 定义与统计维度
- 对 **单个样本** 的某个“层/特征维（通常 hidden）”做归一化。
- 对于输入 $x \in \mathbb{R}^{d}$（每个 token/样本一条向量）：

$$\mu = \frac{1}{d}\sum_{j=1}^{d} x_j, \quad \sigma^2 = \frac{1}{d}\sum_{j=1}^{d} (x_j-\mu)^2$$

#### 3.4.2 训练/推理一致性
- LN **不依赖 batch 统计**，因此训练与推理形式一致；特别适合 Transformer 这类经常使用小 batch、变长序列、以及单样本推理的场景。

面试一句话：Transformer 常用 LN/RMSNorm 的核心原因是 **不依赖 batch 统计、训练/推理一致、分布式与小 batch 更友好**；而 CNN 上 BN 的经验优势更多来自其对卷积特征的稳定化与一定正则化效应。

#### 3.4.3 LN 的特点与常见问题
- LN 会做 **去均值 + 方差归一**；它改变了向量的分布形态。
- 常见问题：
	- 数值稳定：$\epsilon$ 过小可能导致混合精度下 NaN。
	- 归一化位置影响训练稳定（Pre-Norm vs Post-Norm 的现象与直觉）。

#### 3.4.4 Pre-Norm vs Post-Norm：是什么、为什么影响稳定性
这里的 “Pre / Post” 指的是：**LayerNorm（或 RMSNorm）相对残差（residual）与子层（attention/FFN）的位置**。

为方便表述，把一个子层写成 $F(\cdot)$（可能是 Attention 或 FFN），输入为 $h$。

- **Post-Norm（原始 Transformer 常见写法）**：
	- 先做残差相加，再做归一化：

	$$h' = \mathrm{LN}(h + F(h))$$

- **Pre-Norm（大模型/深层 Transformer 更常见）**：
	- 先归一化，再进子层，然后做残差相加：

	$$h' = h + F(\mathrm{LN}(h))$$

	- 通常还会在整个网络末尾再接一个最终的 LN/RMSNorm（FinalNorm）。

**核心现象（经验规律）**
- 模型越深（层数越多）、训练越“硬”（更大学习率、混合精度、更小 batch、强并行切分），**Post-Norm 越容易不稳定/发散**；Pre-Norm 往往更稳。
- 反过来，在一些相对浅的网络或特定任务上，Post-Norm 也可能有不错效果，但通常更依赖 **学习率 warmup、梯度裁剪、初始化与残差缩放** 等技巧。

**取舍（面试可用的“优缺点对比”）**
- Pre-Norm：
	- 优点：深层更稳、对学习率/初始化更鲁棒，尤其适合大规模 Transformer 训练。
	- 代价/注意点：因为每个子层的输出是“残差相加”直接回到主干，表示的均值项不会被每层强制拉回到 0，激活尺度可能随深度漂移；通常通过 FinalNorm、合适的初始化/残差缩放来控制。
- Post-Norm：
	- 优点：每个 block 输出都被归一化到相对稳定尺度，有时在较浅网络上好调或效果不错。
	- 缺点：深层更容易不稳定，往往必须依赖 warmup、较小初始学习率、梯度裁剪、以及更谨慎的残差/初始化设计。

### 3.5 RMSNorm（Root Mean Square Norm）
#### 3.5.1 定义（与 LN 的关系）
- RMSNorm 通常 **不做去均值**，只按均方根缩放：

$$\mathrm{RMS}(x)=\sqrt{\frac{1}{d}\sum_{j=1}^{d} x_j^2 + \epsilon}, \quad y = \gamma \cdot \frac{x}{\mathrm{RMS}(x)}$$

- 与 LN 相比：少一步减均值，计算更省，且在一些大模型实践中表现良好。

#### 3.5.2 为什么大模型常用 RMSNorm（要能讲清“直觉”）
- 训练稳定性与吞吐的折中：减少计算与访存。
- 与残差连接配合时：RMSNorm **不做去均值**，因此 token 向量的均值/offset 不会被每层强制拉回 0；一些“偏置/平移相关”的模式更容易沿主干与残差路径自然传递（具体体验依赖架构）。
- 许多实现只保留缩放参数 $\gamma$、不显式使用平移参数 $\beta$：
	- 一方面减少一次逐元素加法与参数量，fused kernel 更容易做快；
	- 另一方面偏置通常可以由线性层的 bias、残差分支或后续层来表达，因此不一定必须在 RMSNorm 里单独配一个 $\beta$。

### 3.6 相关但可选：GroupNorm / InstanceNorm / WeightNorm（点到为止）
- **GroupNorm（GN）**：按组对通道归一化，不依赖 batch；小 batch 的 CNN 常见替代。
- **InstanceNorm（IN）**：按样本/通道归一化，风格迁移等任务常见。
- **WeightNorm**：对权重做重参数化，属于另一条思路（可简单对比）。

### 3.7 训练与工程实践要点（面试“落地”部分）
- **小 batch 怎么办**：SyncBN、GN/LN 替代、梯度累积带来的“统计量并不会变大”的事实。
- **混合精度稳定性**：
	- LN/RMSNorm 通常需要合适的 $\epsilon$；必要时用 fused kernel/高精度累加。
	- NaN 排查：先看归一化层输出与方差是否为 0、是否有 Inf。
- **位置与结构**：归一化层放在残差之前/之后对训练稳定影响很大。
- **推理部署**：BN 推理依赖 running stats；跨域/分布漂移时可能出问题。

## 面试高频考点
### 算法岗通用题
#### Q1：**标准化 vs Min-Max：什么时候选哪个？离群点多怎么办？**
- 标准化（Z-score）：将数据变换为零均值、单位方差，适合近似高斯分布或希望特征分布更符合统计假设的场景
- Min-Max 缩放：将数据线性映射到指定区间（通常[0,1]或[-1,1]），适合已知上下界、分布稳定且对输出范围有明确要求的场景，对离群点敏感
- 离群点多的场景：优先选择 Robust scaling，使用中位数和四分位距进行缩放，对离群点鲁棒性更强

#### Q2：**统计量用训练集还是全数据？为什么？**
- 必须只用训练集估计归一化所需的统计量（均值/方差、min/max、中位数/IQR等），验证/测试/线上推理时使用同一套统计量
- 原因：如果使用全量数据（包含测试集）计算统计量，会发生数据泄漏（Data Leakage），将测试集的信息引入训练过程，导致评估结果过于乐观，无法反映模型真实泛化能力

#### Q3：**线上分布漂移怎么处理？**
- 可采用"周期性重估"或"滑动窗口"的方式更新统计量，适应线上数据分布的变化
- 需要注意：更新统计量时必须严格基于新的训练数据，不能使用线上未标注数据直接更新，避免引入分布偏差
- 规范化规则、正则表达式、黑白名单、映射表等都需要版本化管理，保证可复现性

#### Q4：**数据归一化和 BN 有什么区别？能互相替代吗？**
- 本质区别：数据归一化是模型外的输入预处理操作，是固定变换；BN（批量归一化）是模型内部的网络层组件，属于可学习的动态变换
- 作用位置不同：数据归一化只对输入特征做一次性处理，BN在模型每层前向传播时动态计算当前batch的统计量进行归一化
- 不能互相替代：
  - 数据归一化无法解决训练过程中深层网络激活尺度漂移的问题
  - BN无法替代数据清洗/规范化操作，尤其在LLM场景中，文本规范化影响分词与token分布，是模型内归一化无法解决的

#### Q5：**BN / LN / RMSNorm 的归一化维度、是否依赖 batch、训练/推理是否一致。**
- BN（BatchNorm）：
  - 归一化维度：对每个通道沿batch+空间维度计算统计量（CNN场景：N,H,W维度；LLM场景：B,T维度）
  - 依赖batch：是
  - 训练/推理一致性：否，训练用当前batch统计量，推理用训练期间维护的滑动平均统计量
- LN（LayerNorm）：
  - 归一化维度：对每个token向量沿hidden维度计算统计量（不跨batch、不跨token）
  - 依赖batch：否
  - 训练/推理一致性：是，训练与推理逻辑完全相同
- RMSNorm：
  - 归一化维度：与LN相同，对每个token向量沿hidden维度计算均方根
  - 依赖batch：否
  - 训练/推理一致性：是

#### Q6：**为什么 Transformer 更偏向 LN/RMSNorm，而 CNN 里 BN 很常见？**
- Transformer 场景特点：
  - 常用小batch、变长序列、单样本自回归推理，BN依赖batch统计量的特性会导致训练/推理分布不匹配
  - LN/RMSNorm不依赖batch统计，训练与推理一致，分布式与小batch场景更友好
  - LN/RMSNorm的生态更成熟，有高效的fused kernel实现，适配张量并行等分布式训练策略
- CNN 场景特点：
  - 通常使用较大batch，BN的batch统计量噪声较小
  - BN对卷积特征有良好的稳定化作用，同时引入的batch级噪声带来一定正则化效果，能提升泛化能力
  - CNN输入通常是固定尺寸，序列长度一致，padding问题不显著，BN统计量更准确

#### Q7：**小 batch 或分布式训练下 BN 的问题与解决思路是什么？**
- 问题：
  - 小batch下统计量噪声大，归一化效果下降，模型性能降低
  - 分布式训练时每卡batch更小，统计量偏差更严重
- 解决思路：
  - 使用 SyncBN：跨设备同步batch统计量，得到全局batch的均值和方差，解决小batch统计噪声问题，但会引入额外通信开销
  - 用 GN/LN/RMSNorm 替代：这些归一化方法不依赖batch统计，天然适合小batch和分布式场景
  - 注意：梯度累积只是累加多个step的梯度，并不会增大BN统计量的有效batch大小，无法解决小batch下BN的统计噪声问题

#### Q8：**RMSNorm 相比 LN 去掉了什么？可能带来哪些利弊？**
- 去掉的部分：RMSNorm去除了LN中的去均值操作，只按均方根对向量进行缩放
- 优势：
  - 计算量更小，减少了一次求均值和逐元素减法的操作，访存开销更低，训练吞吐更高
  - 不做去均值，token向量的均值/offset不会被每层强制拉回0，偏置/平移相关的模式更容易沿主干与残差路径传递
  - 许多实现可以只保留缩放参数γ，不需要平移参数β，减少参数量和计算量
- 潜在劣势：
  - 对于某些对均值偏移敏感的任务，可能需要其他组件来补偿去均值操作的缺失
  - 当激活分布存在显著偏移时，归一化效果可能弱于LN

#### Q9：**$\gamma/\beta$ 的作用是什么？如果去掉会怎样？**
- γ（缩放参数）：让模型在归一化后仍能学习回合适的输出幅值，避免所有层输出都被限制在单位方差量级，提升表达与优化的灵活性
- β（平移参数）：让模型能学习回需要的偏置/均值，避免归一化操作将有用的偏置模式强行移除
- 如果去掉：
  - 归一化会强行将输出限制在固定的尺度和均值附近，损失模型的表达能力
  - 网络无法自适应调整归一化后的分布，可能导致训练不稳定或性能下降
  - RMSNorm部分实现可以去掉β，因为偏置可以通过线性层的bias、残差分支或后续层来表达

#### Q10：**$\epsilon$ 太大/太小分别可能带来什么问题？**
- ε的作用：防止归一化时分母（方差/均方根）过小导致除零错误和数值爆炸
- ε太小：
  - 当某些维度方差非常小时，会导致分母接近0，归一化后数值爆炸，出现NaN/Inf
  - 混合精度训练（FP16/BF16）下精度不足，更容易出现数值不稳定问题
- ε太大：
  - 归一化效果被削弱，无法有效将分布缩放到期望的尺度
  - 模型训练稳定性可能下降，收敛速度变慢
- 常见取值范围：1e-5 ~ 1e-6，混合精度不稳定时可适当增大ε
