# 2.2 位置编码
---
## 一、主流位置编码详解
### 🔹 位置编码基础
Transformer完全基于注意力机制，没有RNN/CNN的时序/局部结构，天然无法感知token的顺序信息，必须显式加入位置编码把位置信息注入输入表示。
### 🔹 好的位置编码的核心特性
一个合格的位置编码需要满足以下特性：
1. **唯一性**：为每个位置输出唯一的编码，不同位置不能有相同的编码
2. **相对一致性**：不同长度的句子之间，任何两个位置之间的差值表示应该保持一致
3. **有界性**：编码值应该在固定范围内，避免数值过大影响训练稳定
4. **远程衰减性**：随着序列中元素之间相对位置的增加，位置编码对模型性能的影响逐渐减弱，符合人类处理长文本时注意力逐渐分散的规律
### 🔹 1. 绝对位置编码
- 核心原理：给每个位置分配一个固定的向量，直接加到token的embedding中，让模型知道每个token的具体绝对位置
- 两种主流实现：
  - Learned（可学习位置编码）：随机初始化位置向量，和模型一起训练，BERT早期用的是这种，缺点是无法泛化到训练时没见过的更长序列
  - Sinusoidal（正弦余弦位置编码）：用正弦余弦函数生成固定的位置向量，不需要训练，是原始Transformer采用的方案
    - 公式：
      $$PE(pos, 2i) = sin(pos / 10000^{(2i / d_{model})})$$
      $$PE(pos, 2i+1) = cos(pos / 10000^{(2i / d_{model})})$$
      其中pos是词在序列中的位置，i是维度索引，d_model是模型隐藏层维度
    - 设计逻辑：正弦余弦函数的周期性使得模型可以自然捕捉相对位置关系，不需要训练即可泛化到更长的序列
- 常见疑问（FAQ）：
  1. **Q：为什么选择三角函数生成位置编码？**
     答：正弦和余弦函数是周期性函数，其周期性特征可以让模型捕捉序列中元素之间的相对位置关系，同时三角函数是连续可导的，适合梯度下降训练。
  2. **Q：为什么要除以10000？**
     答：除以10000是为了缩放位置编码的值，控制不同频率的周期范围。较小的缩放因子会使得位置编码对短距离更敏感，较大的缩放因子则可以让模型捕捉更长距离的位置关系，10000是原始论文经过实验验证的最优值。
  3. **Q：为什么要正弦和余弦函数交替使用？**
     答：交替使用正弦和余弦函数可以让每个维度的编码都有不同的周期性特征，组合起来可以唯一表示任意位置，同时保证相对位置的一致性。
- 缺陷：都是基于绝对位置的表示，对于长序列来说，绝对位置的区分度会下降，长序列效果不如相对位置编码。
### 🔹 2. 相对位置编码
- 核心原理：不编码绝对位置，而是编码token之间的相对距离，让模型学习两个token之间的位置关系，更符合语言的规律，长序列效果更好
- 主流实现：
  - RoPE（旋转位置编码）：现在大模型主流使用的位置编码，LLaMA、Qwen、Mistral等都用RoPE，由苏剑林提出，巧妙借助了欧拉公式的数学特性
    - 设计理念：RoPE的核心目标是让query向量$q_m$和key向量$k_n$的内积可以自然包含两者的相对位置信息$m-n$，即存在函数$g$满足：$<q_m, k_n> = g(x_m, x_n, m-n)$
    - 数学原理：
      1. 借助欧拉公式$e^{ix} = cos(x) + i\sin(x)$，将二维词向量映射到复平面，实部对应第一维，虚部对应第二维
      2. 对于位置m的query向量，乘以旋转因子$e^{im\theta}$，相当于在复平面上将向量旋转m个单位角度，旋转角度$\theta_i = 10000^{-2i/d_{model}}$
      3. 两个旋转后的向量的内积的实数部分，只和它们的相对位置$m-n$有关，和绝对位置无关，自然包含相对位置信息
    - 多维推广：由于内积满足线性叠加性质，任意偶数维的RoPE都可以表示为二维情形拼接而成的形式，通过块对角旋转矩阵实现高维向量的位置编码
    - 优势：
      ① 可以捕捉相对位置关系，比绝对位置编码效果更好
      ② 可以泛化到更长的序列，支持上下文扩展技术
      ③ 没有额外计算开销，效率高，只需要在Q和K计算后做旋转变换即可
    - 实现：通过旋转矩阵对Q和K做变换，内积之后自然包含相对位置的信息，不需要修改其他模块
  - ALiBi（注意力线性偏置）：不需要给输入加位置编码，直接在注意力分数上加一个和相对距离成正比的偏置，距离越远偏置越小
    - 优势：实现非常简单，不需要修改模型输入，长序列泛化能力极强，不需要微调就可以支持比训练时长得多的序列
    - 适用场景：适合需要支持超长上下文的模型，比如Mistral、Qwen-7B等都用了ALiBi
### 🔹 3. 上下文扩展技术
- 核心概念：**外推性**是指模型在处理推理阶段比训练阶段更长的序列时，仍然能保持性能稳定的能力。由于显存限制，大模型通常预训练时的最大序列长度有限（如4k/8k），但推理时可能需要处理32k/128k甚至更长的文本，上下文扩展技术就是为了解决这个问题
- 核心目标：让训练时只有4k/8k上下文的模型，可以支持32k/128k甚至更长的上下文，不需要重新预训练
- 主流方案：
  - NTK-Aware：不需要微调，通过修改RoPE的缩放因子，让模型可以支持更长的上下文，精度损失很小，是最常用的零样本上下文扩展方案
  - YaRN：改进版的NTK-Aware，通过动态调整缩放因子和温度，精度损失更小，支持更长的上下文，现在长上下文模型主流使用
  - 线性插值：最简单的扩展方案，直接对位置编码做线性插值，但是精度损失较大，现在已经很少用
  - 微调扩展：用长序列数据对模型做少量微调，可以让模型完美支持更长的上下文，但是需要额外的训练数据和计算资源

## 二、面试高频考点
### ✅ 开发岗通用题
1. **Q：主流大模型都用什么位置编码？RoPE相比绝对位置编码的优势是什么？**
答：现在主流大模型（LLaMA、Qwen、Mistral等）都用RoPE旋转位置编码。优势：① 可以捕捉相对位置关系，比绝对位置编码效果更好；② 可以泛化到更长的序列，支持上下文扩展；③ 没有额外计算开销，效率高。
2. **Q：上下文扩展技术的作用是什么？常用的零成本扩展方案是什么？**
答：作用是让训练时短上下文的模型可以支持更长的上下文，不需要重新预训练。常用的零成本方案是NTK-Aware，不需要微调，只需要修改RoPE的缩放因子，就可以支持数倍长度的上下文，精度损失很小。
3. **Q：好的位置编码需要满足哪些核心特性？**
答：需要满足四个特性：① 唯一性：每个位置有唯一编码；② 相对一致性：不同长度句子中相同相对距离的编码表示一致；③ 有界性：编码值在固定范围内，不影响训练稳定；④ 远程衰减性：位置距离越远，编码的影响越小，符合人类注意力规律。
4. **Q：原始Transformer使用的正弦余弦位置编码有什么优势？**
答：优势：① 不需要训练，不占用模型参数；② 可以泛化到训练时未见过的更长序列，公式是固定的；③ 可以自然捕捉相对位置关系，通过三角函数的周期性实现。
### 🔬 算法岗专项题
1. **Q：RoPE的核心原理是什么？为什么可以捕捉相对位置关系？**
答：RoPE的核心是通过旋转矩阵把位置信息编码到Q和K的内积中，对于位置m和位置n的Q和K，经过旋转之后的内积只和m-n的相对距离有关，和绝对位置无关，所以可以自然捕捉相对位置关系，不需要额外的位置编码输入。
2. **Q：RoPE是如何借助欧拉公式实现位置编码的？**
答：RoPE借助欧拉公式$e^{ix} = cos(x) + i\sin(x)$，将二维词向量映射到复平面，给位置m的向量乘以旋转因子$e^{im\theta}$，相当于在复平面旋转m个角度。两个旋转后的向量内积的实数部分只和相对位置$m-n$有关，从而实现相对位置编码。高维向量可以拆分为多个二维向量分别旋转，实现多维RoPE。
3. **Q：ALiBi相比RoPE的优势是什么？适合什么场景？**
答：ALiBi的优势：① 实现极其简单，不需要修改模型输入，只需要在注意力分数上加一个线性偏置；② 长序列泛化能力极强，不需要微调就可以支持比训练时长得多的上下文；③ 计算开销极小，几乎没有额外成本。适合需要支持超长上下文的模型，不需要专门的长序列预训练就可以实现长上下文支持。
4. **Q：正弦余弦位置编码为什么要除以10000？这个参数的作用是什么？**
答：除以10000是为了控制不同维度的位置编码的周期范围，低频维度（i小）的周期长，可以捕捉长距离位置关系，高频维度（i大）的周期短，可以捕捉短距离位置关系。10000是原始论文经过实验验证的最优值，可以平衡长短距离的位置捕捉能力。
