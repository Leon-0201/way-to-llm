# 2.5 大模型架构演进
> 核心考点：主流大模型架构特点、差异对比、适用场景，所有岗位高频考点
---
## 一、主流架构详解
### 🔹 1. Decoder-only 架构（GPT系列）
- 核心原理：纯解码器架构，只保留Transformer解码器栈，去除编码器，现在所有主流大模型（GPT、LLaMA、Qwen、Mistral等）都是Decoder-only架构，已经成为工业界标准
- 架构特点：
  - 没有编码器，也没有交叉注意力模块，所有注意力都是自注意力
  - 生成流程和Transformer解码器完全一致，基于输入上下文自回归生成下一个词
- 优势：
  - 预训练和微调对齐，预训练就是做下一词预测，微调也是生成任务，对齐效果好
  - 生成能力强，适合所有生成类任务，通用能力强
  - 架构简单，容易扩展，支持超大规模模型
- 缺陷：理解类任务相比编码器-解码器架构略差一点，但是差距很小，现在大模型的通用能力已经可以覆盖
- 为什么现在大模型都用Decoder-only：
  1. 预训练和微调对齐：预训练目标是下一词预测，和下游生成任务完全对齐，微调后效果更好
  2. 统一架构：同一个架构可以同时支持理解和生成任务，不需要单独设计编码器，泛化能力更强
  3. 实验证明：相同参数量和训练数据下，纯Decoder的效果优于编解码器架构，模型容量利用率更高
  4. 训练和推理优化成熟，生态完善
### 🔹 2. LLaMA 架构详解
- 核心特点：现在开源大模型的事实标准架构，所有开源模型（Qwen、Mistral、Llama3、DeepSeek等）都基于LLaMA架构改进，代表了当前大模型的主流设计方向
- 核心改进点相比原始Transformer：
  1. **Pre-Norm**：用Pre-Norm代替原始Transformer的Post-Norm，归一化放在模块输入前，公式：`Output = x + Module(RMSNorm(x))`，梯度流直接通过短路连接回传，训练非常稳定，不需要太长的warmup，现在所有大模型都采用。原始的Post-Norm是归一化放在残差连接之后，梯度需要经过归一化层才能回传，深层模型容易出现梯度消失，训练不稳定，需要精细的warmup和学习率调整，现在只有小模型可能用。
  2. **RMSNorm**：用RMSNorm代替原始Transformer的LayerNorm，是大模型标配归一化方案：
     - LayerNorm公式：`LayerNorm(x) = γ * (x - mean(x)) / sqrt(var(x) + ε) + β`，需要计算均值和方差，减均值去中心
     - RMSNorm公式：`RMSNorm(x) = γ * x / sqrt(mean(x²) + ε)`，只计算均方根，不需要减均值，计算速度快15~20%，训练更稳定，大模型下效果和LayerNorm相当甚至更好
     - 大模型实践：取消归一化的偏置β，只保留缩放因子γ，减少参数量，提升训练稳定性，LLaMA、Qwen等都采用这种设计
  3. **SwiGLU**：用SwiGLU作为FFN的激活函数，代替原始Transformer的普通ReLU，是现在大模型标配的FFN变体：
     - **技术演进背景**：激活函数从Sigmoid→ReLU→Swish→GLU逐步演化，SwiGLU是Swish和GLU的结合体，充分吸收了两者的优势：
       - Swish是Google提出的平滑激活函数，公式：`Swish(x) = x * σ(x)`，相比ReLU更平滑，梯度更稳定，效果更好
       - GLU（门控线性单元）通过Sigmoid门控控制信息流，提升模型表达能力，公式：`GLU(x) = (XW1 + b1) ⊗ σ(XW2 + c)`
     - **核心原理**：SwiGLU将门控的激活函数从Sigmoid换成Swish，公式：`SwiGLU(x) = (xW1 + b1) ⊗ Swish(xW2 + b2) W3 + b3`，其中⊗表示逐元素乘法
     - **核心优势**：
       1. **非线性能力强**：通过Swish激活引入更强的非线性，能够学习更复杂的数据模式
       2. **门控特性**：动态调整信息流，更好地捕捉长距离依赖关系，尤其适合长序列任务
       3. **梯度稳定**：负输入区域仍有非零梯度，有效缓解梯度消失问题，训练更稳定
       4. **计算效率高**：虽然参数量比普通ReLU FFN多50%，但矩阵乘法可以充分利用GPU Tensor Core，实际计算开销增长很小
     - **参数量与收益**：普通FFN参数量是8d²（d×4d +4d×d），SwiGLU是12d²（d×4d +d×4d +4d×d），参数量多50%，但效果提升明显（困惑度降低2~3%），收益远大于成本，所以现在所有主流大模型都采用
     - **维度设计原则**：LLaMA系列把中间维度特殊设计为11008，是为了适配GPU的内存对齐规则，提升计算效率，实际效果和4倍维度（4×4096=16384）差异很小
     - **其他变体**：GeGLU（用GELU激活）、ReGLU（用ReLU激活），效果略差于SwiGLU，现在很少用
  4. **RoPE**：用RoPE旋转位置编码，代替原始Transformer的正弦余弦绝对位置编码，长序列效果更好
  5. **取消偏置**：所有线性层都取消偏置，减少参数量，提升训练稳定性
- 大模型设计原则：头维度通常固定为128（适配GPU硬件对齐），比如7B模型h=32，总维度d=32×128=4096；70B模型h=128，总维度d=128×128=8192
- 优势：架构成熟，训练稳定，效果好，生态完善，所有开源框架都支持，是现在大模型的标准架构
### 🔹 3. MoE 架构（混合专家模型）
- 核心原理：把传统Transformer的FFN层替换为稀疏MoE层，通过条件计算的思路，在不增加推理计算量的前提下大幅提升模型总参数量，是现在大模型扩容的主流方向，GPT-4、Claude 3、Qwen 2.5 MoE、DeepSeek V2等最新大模型都采用MoE架构
- **核心组件**：
  1. **稀疏MoE层**：包含N个独立的专家（通常是FFN网络），代替原来的单个FFN层，专家之间参数不共享
  2. **门控网络/路由（Router）**：可学习的网络，用于决定每个token被发送到哪些专家处理，通常采用带噪声的TopK门控机制：
     - 给每个专家的打分添加少量噪声，增加路由的多样性，避免专家固化
     - 选择得分最高的K个专家（通常K=2）处理当前token，其他专家不激活
     - 经过Softmax归一化后得到每个激活专家的权重，最终输出是多个专家输出的加权和
- **核心特点**：
  - **稀疏性计算**：每个token只激活少数专家（通常2个），总参数量可以做得很大，但推理时实际激活的参数量很小。比如Mixtral 8x7B总参数量47B，但每个token只激活2个专家，实际计算量和12B稠密模型相当，推理速度和7B模型差不多，效果接近34B模型，性价比极高
  - **预训练更快**：相同计算预算下，MoE模型可以比稠密模型更快达到相同的效果，或者在相同训练时间下达到更好的效果，非常适合超大规模预训练
  - **显存需求高**：推理时需要加载所有专家的参数到显存，比如Mixtral 8x7B需要的显存量接近47B稠密模型，对显存容量要求高
  - **训练难度大**：需要设计负载均衡损失，避免出现"专家偏置"（部分专家被频繁激活，部分专家几乎不被使用），训练调参比稠密模型复杂
- **优劣势总结**：
  | 优势 | 劣势 |
  |------|------|
  | 预训练速度快，相同计算成本下效果更好 | 训练难度大，需要解决负载均衡问题 |
  | 推理性价比高，激活参数量小，速度快 | 显存占用高，需要加载所有专家参数 |
  | 模型扩容方便，只需要增加专家数量 | 微调容易过拟合，泛化难度比稠密模型大 |
  | 高吞吐场景下成本优势明显 | 推理延迟有抖动，不同token激活专家不同 |
- **专家数量的权衡**：
  - 增加专家数量可以提升模型效果和训练速度，但收益会递减，通常专家数量超过256/512之后提升不明显
  - 专家数量和显存占用线性正相关，需要根据显存容量和业务需求权衡，常用配置是8、16、32个专家
- **适用场景选择**：
  - ✅ 适合云侧高吞吐、大算力场景：比如在线大模型服务、大规模预训练，MoE的性价比优势非常明显
  - ❌ 不适合端侧小显存场景：显存占用高，端侧难以承载，端侧更适合稠密小模型
- **最新研究方向**：
  - 蒸馏：把MoE模型蒸馏回稠密模型，兼顾大模型效果和小模型部署成本
  - 量化：MoE的极端量化研究，比如QMoE可以把1.6万亿参数模型压缩到160GB
  - 专家合并：通过合并相似专家减少推理时需要加载的专家数量，降低显存需求

### 🔹 4. DeepSeek MLA 架构（多头潜在注意力）
- **技术背景**：KV Cache是大模型推理的核心瓶颈，MQA/GQA通过减少KV头数降低KV Cache大小的优化已经遇到瓶颈：
  - MQA所有Q头共享一组KV，虽然KV Cache最小，但精度损失明显
  - GQA分组共享KV，是精度和性能的折中，但进一步减少KV头数会导致精度快速下降
  - MLA从降低KV本身的维度入手，突破了MQA/GQA的优化极限，是目前最先进的KV优化方案
- **核心思路**：通过低秩分解技术压缩KV向量的维度，而不是减少KV头的数量，在保持精度的前提下进一步降低KV Cache大小40%以上
- **核心原理**：
  1. **K/V低秩分解**：
     - 传统K向量计算：`K = X Wk`，需要缓存完整的K向量，维度是d
     - MLA把Wk拆成两个矩阵的乘积：`Wk = Wk_up × Wk_down`，其中Wk_down是降维矩阵，把X映射到低维空间
     - 缓存时不需要缓存完整的K向量，只需要缓存低维的潜在向量`C_k = X Wk_down`，维度可以降低到原来的1/2~1/3
     - K和V共享同一个降维矩阵Wk_down，进一步减少计算量和缓存开销
  2. **矩阵吸收（Absorb）避免额外计算量**：
     - 低秩分解后如果直接计算，推理时需要多做一次`K = C_k × Wk_up`的矩阵乘法，会增加计算量
     - 利用矩阵乘法结合律，把Wk_up吸收到Q的投影矩阵中：`Q^T K = (X Wq)^T (C_k Wk_up) = X^T (Wq^T Wk_up) C_k`
     - 提前合并Wq和Wk_up得到新的Q投影矩阵，推理时不需要额外计算，完全不增加计算开销
     - 同理，V的升维矩阵可以吸收到输出投影矩阵中，也没有额外计算量
- **RoPE兼容性解决方案**：
  - RoPE位置编码需要对K向量的每个维度进行旋转，低秩分解会破坏位置编码的结构
  - DeepSeek的解决方案是在K向量中新增专门的位置编码维度，这部分维度不参与低秩分解，独立计算和存储
  - 缓存时同时存储降维后的语义向量和位置编码专用向量，总大小仍然比传统KV小40%以上
- **实际收益**：
  - KV Cache大小降低40%以上，长序列场景下显存占用减少明显
  - 推理速度比GQA提升15%以上，长序列提升更明显
  - 精度和GQA持平甚至略优，没有明显精度损失
- **应用案例**：DeepSeek V2/V3/R1系列模型均采用MLA架构，在长序列推理场景下表现优异，支持128K甚至更长的上下文窗口，推理成本显著降低

## 二、面试高频考点
### ✅ 开发岗通用题
1. **Q：为什么现在所有主流大模型都用Decoder-only架构？**
答：原因：① 相同参数量下，Decoder-only架构的效果比Encoder-only和编码器-解码器架构更好，通用能力更强；② 预训练和微调对齐，预训练的下一词预测目标和微调的生成任务完全一致，微调后效果更好；③ 架构简单，容易扩展，支持超大规模模型，训练和推理优化都很成熟。
2. **Q：MoE架构相比普通稠密模型的优势是什么？适合什么场景？**
答：优势：性价比极高，总参数量大，效果好，但是实际激活的参数量小，推理速度快，比如Mixtral 8x7B效果接近34B模型，推理速度和7B差不多。适合云侧高吞吐、低成本的部署场景，适合需要大模型效果但是又要控制推理成本的业务。
3. **Q：大模型的FFN为什么用SwiGLU代替普通ReLU？**
答：SwiGLU是门控线性单元，用Swish激活函数，有两个线性层做门控，可以更好地捕捉非线性关系，表达能力更强，比普通ReLU的FFN效果提升2~3%（困惑度降低），同时计算量只增加50%，收益远大于成本，所以现在所有主流大模型都使用SwiGLU作为FFN的激活函数。
4. **Q：MLA架构的核心作用是什么？相比GQA有什么优势？**
答：MLA是多头潜在注意力，核心作用是进一步降低KV Cache的大小，突破MQA/GQA的优化瓶颈。优势：在精度持平的前提下，KV Cache大小比GQA降低40%以上，推理速度提升15%以上，尤其适合长序列推理场景，是目前最先进的KV优化方案。
5. **Q：MoE模型为什么显存占用高？相比同效果稠密模型有什么优势？**
答：MoE需要加载所有专家的参数到显存，而推理时只激活少数专家，所以显存占用接近总参数量对应的稠密模型，但实际推理计算量只和激活参数量相关。优势是效果接近大模型，但推理速度和小模型差不多，性价比极高，适合云侧高吞吐场景。
### 🔬 算法岗专项题
1. **Q：LLaMA架构相比原始Transformer做了哪些改进？为什么这些改进有效？**
答：改进点：① RMSNorm代替LayerNorm，不需要减均值，计算更快，训练更稳定；② SwiGLU代替ReLU FFN，门控机制提升模型表达能力；③ RoPE旋转位置编码，捕捉相对位置关系，长序列效果更好；④ Pre-Norm代替Post-Norm，训练更稳定，适合大模型；⑤ 取消线性层偏置，减少参数量，提升训练稳定性。这些改进都是针对大模型训练的痛点，让模型可以做得更大，训练更稳定，效果更好。
2. **Q：SwiGLU相比ReLU、GELU等激活函数有什么优势？为什么现在大模型都用SwiGLU？**
答：优势：① 门控机制：通过Swish门控动态调整信息流，表达能力更强，能更好捕捉长距离依赖；② 梯度稳定：负输入区域有非零梯度，缓解梯度消失问题，训练更稳定；③ 非线性强：Swish的平滑非线性比ReLU更适合大模型的复杂模式学习；④ 计算高效：虽然参数量多50%，但矩阵乘法可以充分利用Tensor Core，实际开销增长很小。综合收益远高于成本，所以成为大模型标配。
3. **Q：MoE架构的核心难点是什么？怎么解决？**
答：核心难点：① 路由策略，需要设计合理的路由算法，让专家负载均衡，避免部分专家被频繁激活，部分专家很少被用，通常用负载均衡损失优化门控网络；② 训练不稳定，MoE模型训练难度比稠密模型大，需要更精细的调参和带噪声的路由机制；③ 推理延迟抖动，不同token激活的专家不同，延迟不稳定，通过专家并行调度和批量路由优化解决。现在成熟的框架（vLLM、TensorRT-LLM）都已经支持MoE的优化。
4. **Q：MLA的核心创新点是什么？怎么解决低秩分解和RoPE的兼容性问题？**
答：核心创新点：突破了MQA/GQA减少KV头数的优化瓶颈，从降低KV本身维度入手，通过低秩分解压缩KV向量，再通过矩阵吸收技术完全消除额外推理计算量，在不损失精度的前提下降低KV Cache 40%以上。RoPE兼容性问题通过新增独立的位置编码维度解决，这部分维度不参与低秩分解，专门处理位置信息，既保证RoPE效果，又降低了整体KV大小。
5. **Q：MoE的门控网络为什么要加入噪声？TopK路由的K值怎么选择？**
答：加入噪声是为了增加路由的多样性，避免门控网络固化，让所有专家都能得到充分训练，缓解负载不均衡问题。K值通常选择2，是效果和计算量的权衡：K=1时计算量最小，但效果较差，容易出现路由错误；K≥3时计算量增加明显，但效果提升很小，K=2是工业界的标准选择，兼顾效果和效率。
6. **Q：Pre-Norm和Post-Norm的核心区别是什么？为什么现在大模型都用Pre-Norm？**
答：核心区别是归一化的位置，Pre-Norm是在模块输入前做归一化，Post-Norm是在残差连接后做归一化。大模型用Pre-Norm的原因：Pre-Norm的梯度流直接通过残差连接回传，不需要经过归一化层，梯度消失问题更轻，训练更稳定，不需要太长的warmup，更容易训练超大规模模型；Post-Norm训练不稳定，需要精细的warmup和学习率调整，不适合10B以上大模型。
7. **Q：RMSNorm相比LayerNorm的优势是什么？为什么大模型都用RMSNorm？**
答：RMSNorm的优势：① 计算更快，只需要计算均方根，不需要计算均值和方差，速度提升15~20%；② 训练更稳定，不需要减均值去中心，避免了均值偏移带来的训练不稳定问题；③ 效果相当，大模型下效果和LayerNorm相当甚至更好。同时大模型实践中会取消归一化的偏置β，只保留缩放因子γ，进一步减少参数量，提升训练稳定性，所以现在所有主流大模型都用RMSNorm。
8. **Q：为什么纯Decoder架构现在成为大模型的标配，而不是编解码器架构？**
答：原因：① 预训练和微调对齐：预训练的下一词预测目标和下游生成任务完全对齐，微调后效果更好；② 模型利用率高：所有参数都用于生成和理解任务，编解码器的编码器参数在生成任务中利用率低；③ 实验验证：相同参数量和训练数据下，纯Decoder的通用能力优于编解码器架构，泛化性更强。
