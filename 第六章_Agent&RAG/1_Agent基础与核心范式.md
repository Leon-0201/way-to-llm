# 1. Agent基础与核心范式
## 一、Agent核心定义
### 1. 什么是Agent
Agent是能够自主理解用户意图、制定执行计划、调用外部工具、反思调整结果的智能体，核心是实现从"被动响应"到"主动执行"的跨越。
### 2. 核心三要素
✅ 感知：理解用户输入、环境状态、工具返回结果
✅ 决策：规划任务、选择工具、判断是否需要调整
✅ 行动：调用工具、输出结果、和用户/其他Agent交互

---
## 二、核心范式演进（2026主流：Plan-Execute-Reflect闭环）
### 🔹 过时范式对比（了解即可）
| 范式 | 核心逻辑 | 淘汰原因 |
|------|----------|----------|
| ReAct（2022） | Thought→Action→Observation 单步交替 | 效率低，token消耗大，缺乏全局规划 |
| Plan-and-Solve（2023） | 先全局规划→再分步执行 | 灵活性差，无法动态调整计划 |

### 🔹 2026主流范式：Plan-Execute-Reflect 闭环
#### 1. 核心流程
```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  任务规划    │ →   │  分步执行    │ →   │  反思调整    │
└──────▲──────┘     └──────┬──────┘     └──────┬──────┘
       │                   │                   │
       └───────────────────┴───────────────────┘
                动态调整，迭代优化
```
#### 2. 三个阶段详解
| 阶段 | 核心能力 | 技术实现 |
|------|----------|----------|
| 任务规划 | 理解任务目标，分解为可执行的子任务，生成执行计划 | 大模型原生规划能力 + 依赖图自动生成 |
| 分步执行 | 按计划执行子任务，调用对应工具，处理执行结果 | 原生Function Call + MCP工具协议 |
| 反思调整 | 判断执行结果是否符合预期，失败则调整方案重试，完成则汇总结果 | 自我校验Prompt + 错误回滚机制 |

#### 3. 面试重点
- **Q：Plan-Execute-Reflect和ReAct的区别是什么？**
  答：① 效率更高：先做全局规划，避免边想边做的冗余推理；② 灵活性更强：执行中发现问题可以动态调整计划，不需要重新走完整流程；③ 能力边界更广：支持更复杂的多步骤任务，任务完成率提升40%以上。

## 三、Agent核心能力
### 🔹 1. 任务规划能力
#### 主流方案（2026）
- 自动子问题分解：大模型根据任务复杂度自动拆分为3~5个可执行子任务
- 依赖图生成：自动识别子任务之间的依赖关系，并行执行无依赖的任务，提升效率
- 动态调整：执行中遇到失败自动调整计划，更换子任务执行顺序或替换工具
#### 面试重点：任务规划的常见失败场景及解决方案
- 任务分解过细：增加执行轮次，提升成本 → 控制子任务数量，合并相似子任务
- 依赖关系识别错误：导致执行顺序错误 → 增加依赖校验环节，执行前检查前置条件
- 规划脱离实际：子任务无法完成 → 增加规划验证步骤，提前判断子任务可行性

### 🔹 2. 工具调用能力
#### 2026成熟方案
- 原生Function Call：GPT-5、Claude 4、豆包4等主流模型工具调用准确率达99%+，无需额外Prompt工程
- 工具自动选择：根据任务目标自动匹配最适合的工具，支持同时调用多个并行工具
- 参数自动补全：根据上下文自动填充工具参数，减少用户输入
#### 面试重点：工具调用错误处理方案
- 参数错误：返回结构化错误信息，引导模型重新生成正确参数
- 工具执行失败：区分可重试错误（如网络超时）和不可重试错误（如权限不足），可重试的自动重试3次，不可重试的反馈给用户
- 工具返回结果过长：自动摘要，提取关键信息返回给模型，避免超出上下文窗口

### 🔹 3. 记忆管理能力
✅ **开发岗要求**：
#### 2026标准三层记忆架构
| 记忆层级 | 用途 | 实现方案 | 保留时长 |
|----------|------|----------|----------|
| 短期记忆 | 存储当前对话的上下文 | 滑动窗口 + 对话摘要 | 当前会话 |
| 长期记忆 | 存储历史对话、用户偏好、历史执行结果 | 向量数据库 + 知识图谱混合存储 | 永久 |
| Profile记忆 | 存储用户固定属性、权限、使用习惯 | 结构化数据库存储 | 永久 |
#### 面试重点：长期记忆的检索优化
- 混合检索：语义匹配 + 时间过滤 + 标签过滤，提升记忆召回准确率
- 记忆衰减：自动降低久远历史记忆的权重，优先召回近期相关记忆
- 记忆更新：自动更新用户偏好，覆盖过时的记忆内容

🔬 **算法岗要求·记忆算法原理**：
#### 核心记忆算法
1. **短期记忆算法**：
   - 滑动窗口记忆：保留最近K轮对话，简单高效，是基础方案
   - 滚动摘要记忆：定期对历史对话生成摘要，用摘要替换原始对话，节省上下文空间，保留关键信息
   - 动态窗口记忆：根据对话复杂度动态调整窗口大小，复杂对话保留更多上下文
2. **长期记忆检索算法**：
   - 语义检索：记忆向量化存储，用余弦相似度匹配相关记忆
   - 时序加权检索：给记忆添加时间衰减权重，`score = similarity * e^(-λ*t)`，λ是衰减系数，t是时间间隔
   - 混合检索：语义匹配 + 关键词匹配 + 元数据过滤，提升召回准确率
3. **记忆更新算法**：
   - 插入策略：新记忆自动关联相关的旧记忆，构建记忆关联网络
   - 遗忘策略：基于使用频率和时间衰减，自动遗忘低价值记忆，节省存储空间
   - 冲突解决：同一主题的新旧记忆冲突时，保留最新记忆，标记旧记忆为过时
#### 前沿优化：
- 记忆 Bank 机制：按主题聚类记忆，检索时先匹配主题Bank，再在Bank内检索，速度提升5倍
- 记忆蒸馏：定期对大量历史记忆蒸馏，提取核心知识，减少冗余

### 🔹 4. 多轮交互与自我反思能力
#### 核心能力
- 歧义主动澄清：任务不明确时主动向用户询问关键信息，避免盲目执行
- 结果自我校验：执行完成后自动校验结果是否符合任务要求，不符合则重新执行
- 错误归因：执行失败后自动分析失败原因，调整方案重试
#### 面试重点：反思的粒度控制
- 避免过度反思：简单任务不需要反思，直接返回结果，降低延迟
- 关键节点反思：子任务完成、工具调用返回结果后再反思，提升效率
- 反思次数限制：最多重试3次，失败则转人工介入，避免死循环

## 四、面试高频考点（分岗位）
### ✅ 开发岗通用题
1. **Q：Agent和普通大模型对话的核心区别是什么？**
   答：核心区别是Agent具备自主决策和行动能力：① 普通对话是被动响应，Agent是主动执行任务；② 普通对话只能输出文本，Agent可以调用工具执行外部操作；③ 普通对话没有记忆和规划能力，Agent可以长期记忆、制定计划、调整方案。

2. **Q：Agent的核心能力有哪些？**
   答：四大核心能力：任务规划能力、工具调用能力、记忆管理能力、自我反思能力，四个能力缺一不可，共同构成完整的Agent闭环。

3. **Q：如何设计Agent的记忆系统？**
   答：采用三层记忆架构：① 短期记忆用滑动窗口存储当前会话上下文；② 长期记忆用向量数据库存储历史信息，支持语义检索；③ Profile记忆用结构化数据库存储用户固定属性，同时需要设计记忆检索、更新、衰减机制，保证记忆的准确性和相关性。

---
### 🔬 算法岗专项题
1. **Q：Tree-of-Thought规划比传统Chain-of-Thought规划有什么优势？**
   答：Chain-of-Thought是单路径推理，容易走偏；Tree-of-Thought是生成多个候选推理路径，每个路径评估打分，选择最优路径，或者融合多个路径的结果，复杂任务的成功率比CoT提升30%以上。核心优势是考虑了多种可能的规划方案，避免单路径的局部最优问题，鲁棒性更强，适合复杂任务规划场景。

2. **Q：Agent的长期记忆检索如何平衡召回率和上下文长度的矛盾？**
   答：常用优化手段：① 记忆分层：短期记忆用滑动窗口保留最近对话，长期记忆用向量检索召回相关历史；② 记忆蒸馏：对历史记忆定期生成摘要，用摘要代替原始对话，减少长度；③ 动态召回：只返回Top-K最相关的记忆片段，而不是全部历史；④ 记忆衰减：给记忆添加时间权重，优先召回近期相关记忆，过滤过时内容。通过这些手段，可以在不超出上下文窗口限制的前提下，保证相关记忆的召回率>90%。

3. **Q：HTN规划和大模型驱动规划的区别是什么？适用什么场景？**
   答：HTN是传统符号规划方法，需要预定义任务分解规则和动作空间，优势是可控性强，可解释性好，适合结构化、规则明确的封闭场景，比如工业控制、工作流执行；大模型驱动规划不需要预定义规则，基于大模型的常识推理能力自动规划，灵活性强，适合开放场景、复杂任务，比如通用助手、多轮对话任务。现在主流是两者结合，核心流程用HTN保证可控性，开放部分用大模型提升灵活性。

4. **Q：Agent的反思机制有哪些常用实现方式？效果如何评估？**
   答：常用实现方式：① 自我校验：任务执行完成后让模型自己判断结果是否符合要求；② 反馈驱动反思：根据工具返回的错误信息调整方案；③ 多智能体反思：多个Agent互相评审方案，提出改进意见。评估指标：任务完成率提升幅度、重试次数减少比例、用户满意度提升。优秀的反思机制可以让任务完成率提升20%以上，重试次数减少50%。
