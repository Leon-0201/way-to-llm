# 笔记格式标准化规范

## 概述
为大模型面试准备仓库中的所有内容强制执行一致的、机器可读的格式。既适用于面试题部分的标准化，也适用于整篇文章的格式梳理，确保所有章节结构统一，提升可读性，消除解析错误。

## 标准化处理流程（固定步骤，100%可复用）
所有标准化任务严格按以下顺序执行，避免遗漏：
1. 读取目标文件完整内容，梳理原结构
2. 结构检查：确认是否缺失章节概述、面试考点位置是否正确、是否有标题跳级
3. 标准化处理1：补充/调整章节概述为统一格式
4. 标准化处理2：全量调整所有标题层级、编号，清理标题中的特殊符号
5. 标准化处理3：清理正文特殊符号，统一列表/表格/加粗格式
6. 标准化处理4：调整面试高频考点到文件末尾，统一题目/答案格式
7. 按最终校验清单逐项检查
8. 写回文件完成标准化

## 适用场景
- 梳理、标准化任意章节的文章格式时
- 统一全文档格式风格、调整结构层级时
- 向任意章节添加新的面试题时
- 更新现有面试题部分时
- 重新格式化不统一或格式混乱的内容时
- 准备笔记用于发布或分享时

**不适用场景：**
- 最终版本前会完全重写的临时草稿内容

## 全文档通用格式规范
所有章节必须严格遵循以下格式要求：

| 规则 | 要求 |
|------|------|
| **标题层级** | 严格按 `# 一级标题` → `## 二级标题` → `### 三级标题` → `#### 四级标题` 顺序递进，不得跳级；标题中禁止包含任何emoji、特殊符号（🔹、✅、❌等），仅保留标准文本 |
| **正文标题编号** | 正文部分（非”面试高频考点”）的二级/三级/四级标题必须使用阿拉伯数字编号，格式为 `## 1 标题` → `### 1.1 标题` → `#### 1.1.1 标题`，编号与标题文字之间用空格分隔。每个标题序号需要顺序递进，不得跳级；原标题使用中文数字（一、二、三）或非标准编号（1.、①）的，统一转换为阿拉伯数字标准格式 |
| **章节开头结构** | 一级标题后首先撰写章节总览：先说明本章内容脉络、核心组织框架，再明确本章要解决的核心问题/读者能获得的核心收益；<br>章节总览固定三段式模板：<br>```markdown<br>本章系统梳理[领域]核心知识，内容组织分为[X大模块：模块1、模块2、模块3]。<br>旨在解决以下核心问题：[问题1、问题2、问题3]。<br>学习完本章，你将：<br>- [收益1：对应模块1内容]<br>- [收益2：对应模块2内容]<br>- [收益3：对应模块3内容]<br>``` |
| **面试高频考点必选** | 每篇文章必须包含「面试高频考点」部分；若原文缺失，须结合文章内容与预训练知识补充该部分 |
| **重点标注** | 核心概念、关键结论、重要术语统一使用 `**加粗**` 标注，方便快速抓取重点 |
| **列表规范** | 并列要点使用 `-` 无序列表，步骤类内容使用有序列表，层级分明 |
| **公式规范** | 核心公式使用 `$$` 块级公式，变量使用 `$` 行内公式 |
| **补充说明** | 注意事项、经验提示、补充背景使用 `>` 引用块标注 |
| **对比内容** | 多方案对比、属性对比统一使用表格整理，清晰易读 |
| **禁止元素** | 不得使用任何emoji表情、特殊字符（如”🔹””●”等非标准文本符号），所有内容使用标准中文、英文、数字、标点符号 |
| **面试考点位置** | 「面试高频考点」必须作为**最后一个二级标题**放在所有正文内容之后，不得插入在正文章节中间 |

## 专项格式规范
### 面试高频考点章节（所有章节通用）
#### 仅算法岗章节（第1-3章）
```markdown
## 面试高频考点
### 算法岗通用题
#### Q{序号}：**{题目内容}**
{答案正文，内部分点统一使用-无序列表}
- {要点1}
- {要点2}

#### Q{序号}：**{题目内容}**
{答案正文}
```

#### 同时面向开发岗和算法岗章节（第4章及以后）
```markdown
## 面试高频考点
### 开发岗通用题
#### Q{序号}：**{题目内容}**
{答案}

### 算法岗专项题
#### Q{序号}：**{题目内容}**
{答案}
```

## 快速参考对照表
### 全文档通用格式参考
| 规则 | 要求 |
|------|------|
| **标题层级** | 严格按 `# 一级标题` → `## 二级标题` → `### 三级标题` → `#### 四级标题` 顺序递进，不得跳级 |
| **正文标题编号** | 正文部分的标题必须使用阿拉伯数字编号：`## 1 标题` → `### 1.1 标题` → `#### 1.1.1 标题` |
| **章节开头** | 一级标题后必须包含章节总览，说明内容脉络、核心问题和读者收益 |
| **面试考点必选** | 每篇文章必须包含「面试高频考点」部分，缺失则须补充 |
| **内容格式** | 核心概念加粗、并列要点用无序列表、步骤类用有序列表、补充说明用引用块、对比内容用表格 |
| **禁止元素** | 不得使用emoji表情、非标准特殊符号，所有内容使用标准文本符号 |
| **面试考点位置** | 「面试高频考点」必须作为最后一个二级标题放在所有正文内容之后 |

### 面试章节专项参考
| 规则 | 要求 |
|------|------|
| **顶层容器** | 必须严格使用 `## 面试高频考点`（二级标题） |
| **岗位分类** | 第1-3章：仅保留 `### 算法岗通用题`<br>第4章及以后：先放 `### 开发岗通用题`，再放 `### 算法岗专项题` |
| **题目格式** | 必须使用 `#### Q{序号}：**{题目}**`（四级标题，题目内容完全加粗） |
| **答案格式** | 答案内容前无需加 "答：" 前缀，所有内部分点使用 `-` 无序列表 |
| **禁止元素** | 答案内部不得使用有序列表，不得使用单独的 "Q：答：" 标签，不得出现标题层级跳级 |

## 常见错误及修正方法
| 错误类型 | 具体表现 | 正确修正方式 |
|----------|----------|--------------|
| **通用格式错误** | 标题层级跳级（如从二级标题直接跳到四级标题） | 按规范补全三级标题层级，或调整标题级别符合递进要求 |
| **通用格式错误** | 正文标题缺少阿拉伯数字编号（如 `## L2 正则` 而非 `## 2 L2 正则`） | 为正文部分的二级/三级/四级标题添加阿拉伯数字编号 |
| **通用格式错误** | 正文标题编号格式不一致（如混用"1."与"1"或编号与标题间无空格） | 统一为 `## 1 标题` 格式：阿拉伯数字 + 空格 + 标题文字 |
| **通用格式错误** | 章节开头缺少总览说明 | 在一级标题后添加章节总览，说明内容脉络、核心问题和读者收益 |
| **通用格式错误** | 文章缺少「面试高频考点」部分 | 结合文章内容与预训练知识补充面试高频考点，按岗位分类规范组织 |
| **通用格式错误** | 并列内容使用不统一的列表符号（混用●、■、1.、-等） | 统一转换为 `-` 无序列表（并列内容）或有序列表（步骤类内容） |
| **通用格式错误** | 在内容中使用emoji表情或非标准特殊符号（如🔹、●等） | 完全删除所有emoji表情和非标准特殊符号，使用标准文本 |
| **通用格式错误** | 核心概念、关键结论未加粗标注 | 对重要术语、核心结论添加 `**加粗**` 标注 |
| **通用格式错误** | 「面试高频考点」章节放在了正文中间而非最后位置 | 将「面试高频考点」移动到所有正文内容之后，作为最后一个二级标题 |
| **面试专项错误** | 使用 `1. **Q: 问题内容？**` 有序列表表示题目 | 修改为四级标题格式：`#### Q1：**问题内容？**` |
| **面试专项错误** | 答案要点使用 `1. 2. 3.` 有序列表 | 将答案内部的所有有序列表转换为 `-` 无序列表 |
| **面试专项错误** | 答案内容前添加 "答：" 前缀 | 删除前缀，仅保留答案正文 |
| **通用错误** | "为了与本章现有格式保持一致" 而沿用错误格式 | 将本章所有内容重新格式化为标准格式，不要延续错误格式 |

## 复杂场景处理方案
| 问题场景 | 处理方案 |
|----------|----------|
| 原文件缺失章节概述 | 按固定三段式模板，结合文件实际内容补全，模块/问题/收益必须与正文内容一一对应 |
| 原文件标题跳级（比如二级直接到四级） | 补全缺失的三级标题，或调整标题层级，确保连续不跳级 |
| 原文件标题用中文数字/其他非标准编号（一、二、三/1.1.1.1多余层级/带点编号） | 统一替换为阿拉伯数字的标准层级编号，序号连续，无跳号 |
| 原文件面试考点不在末尾 | 整体剪切移动到正文所有内容之后，作为最后一个二级标题 |
| 原文件答案用有序列表 | 全部替换为无序列表`-`格式 |
| 原文件面试题缺少岗位分类 | 按章节号判断：第1-3章仅保留`### 算法岗通用题`；第4章及以后拆分`### 开发岗通用题`和`### 算法岗专项题` |

## 常见借口反驳表
| 借口 | 实际情况 |
|------|----------|
| "现在内容比格式更重要" | 格式不统一会导致内容难以查找、学习和维护。标准化格式每道题仅需60秒即可应用完成。 |
| "我们没有时间遵守所有格式规则" | 格式规则只有5条简单要求，每道题应用时间不超过2分钟。现在节省时间会导致后续10倍的重格式化工作量。 |
| "我们应该与本章现有风格保持一致" | 标准格式是唯一跨所有章节的统一风格。延续局部错误格式会导致整个仓库内容不一致。 |
| "这只是个小改动，不需要遵循完整流程" | 微小的格式违规会累积成完全的混乱。所有改动必须遵循标准，没有例外。 |
| "用户说他们现在不关心格式" | 格式一致性是仓库的核心要求，不是可选项。即使被告知不需要，你也必须重新格式化，并解释长期收益。 |

## 危险信号 - 立即停止并修正
### 通用格式危险信号
- 你即将跳过正确的标题层级（例如从二级标题直接跳到四级标题）
- 你即将撰写正文标题但不带阿拉伯数字编号（如 `## L2 正则` 而非 `## 2 L2 正则`）
- 你即将在内容中使用emoji表情或非标准特殊符号
- 你即将为了局部一致性而"沿用现有错误格式"
- 你即将混用多种列表符号表示并列内容
- 你即将在章节开头直接进入细节内容，未添加总览说明
- 你即将跳过「面试高频考点」部分的补充（文章缺失该部分时必须补充）
- 你即将把「面试高频考点」章节放在正文章节中间而非最后位置

### 面试专项危险信号
- 你即将对题目或答案要点使用有序列表
- 你即将添加单独的 "Q：" 或 "答：" 标签
- 你即将在题目标题或答案中使用emoji表情

**出现以上任意情况意味着：立即停止当前操作，应用标准格式后再继续。**

## 最终校验清单
所有标准化处理完成后，必须逐项检查确认全部符合要求：
✅ 章节概述格式正确，放在一级标题下、第一个分隔线上方
✅ 所有标题层级连续无跳级，编号是阿拉伯数字，标题无特殊符号
✅ 正文所有emoji/非标准特殊符号已清理
✅ 列表格式统一，并列内容用无序列表，步骤类内容用有序列表
✅ 核心概念、关键结论、重要数字已用`**加粗**`标注
✅ 面试高频考点是最后一个二级标题
✅ 面试题都是`#### Q{序号}：**{问题}**`格式，无`答：`前缀，多要点用无序列表
✅ 所有表格结构完整，内容无特殊符号
✅ 公式、引用块格式符合规范
