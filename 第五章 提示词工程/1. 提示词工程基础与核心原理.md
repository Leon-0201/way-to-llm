# 1. 提示词工程基础与核心原理
## 一、提示词工程核心定义
### 🔹 什么是提示词工程
✅ **开发岗要求**：
提示词工程是通过设计、优化输入给大模型的文本（Prompt），引导大模型输出符合预期结果的技术，是大模型应用开发的基础能力，直接决定了大模型的输出质量、稳定性和效率。
#### 核心价值
- 提升输出准确率：合适的Prompt可以让任务准确率从60%提升到95%以上
- 降低输出随机性：结构化Prompt可以大幅减少大模型的幻觉和偏离需求的情况
- 降低成本：优化后的Prompt可以减少推理轮次，缩短输出长度，降低token消耗
- 提升安全性：通过Prompt约束可以避免大模型输出违规、敏感内容

🔬 **算法岗要求·底层理论基础**：
提示词工程的本质是**大模型的上下文学习（In-Context Learning, ICL）能力**：
1. **核心原理**：大模型在预训练阶段学习了大量知识和模式，推理时通过输入Prompt中的上下文信息（角色、指令、示例），激活对应的知识和模式，生成符合预期的输出
2. **理论基础**：
   - 少样本学习（Few-shot Learning）：通过输入少量示例，模型可以快速学习任务模式
   - 零样本学习（Zero-shot Learning）：不需要示例，仅通过指令即可完成任务
   - 指令对齐（Instruction Tuning）：大模型经过指令微调后，能够更好地理解和遵循人类指令
3. **关键影响因素**：
   - 上下文窗口限制：Prompt长度受模型上下文窗口限制，超出部分会被截断
   - 注意力机制：模型会重点关注Prompt中的关键信息，合理的结构设计可以引导注意力到重要位置
   - 近因效应：Prompt末尾的信息对输出的影响通常比开头更大

## 二、核心基础原则（2026通用准则）
### 🔹 CLEAR原则（开发岗必备）
| 原则 | 说明 | 具体要求 |
|------|------|----------|
| C（Clear）清晰明确 | 指令清晰无歧义，避免模糊表述 | 不用"写一篇文章"，要用"写一篇500字左右的人工智能应用科普文章，面向中学生，语言通俗易懂" |
| L（Logical）逻辑结构化 | 内容分层次，结构清晰 | 按角色→任务→要求→输出格式的顺序组织，用分隔符区分不同模块 |
| E（Explicit）要求明确 | 明确约束条件和输出要求 | 明确要求输出格式、字数限制、专业领域、风格要求等 |
| A（Accurate）信息准确 | 提供的输入信息准确完整 | 需要处理的信息要完整给出，不要让模型猜测缺失信息 |
| R（Relevant）内容相关 | 只包含和任务相关的信息 | 避免无关信息干扰模型注意力，无关内容放在最后或者删除 |

### 🔹 经典三段式结构（通用模板）
```
# 角色设定
你是一名资深的Python开发工程师，有10年以上的后端开发经验，擅长编写高质量、易维护的代码。

# 任务要求
请实现一个快速排序算法，要求：
1. 支持整数和浮点数排序
2. 包含详细的中文注释
3. 时间复杂度为O(nlogn)，空间复杂度为O(logn)
4. 处理边界情况：空数组、单元素数组、重复元素

# 输出格式
- 先写Python代码
- 再写算法说明和复杂度分析
- 最后写测试用例
```
✅ **开发岗面试重点**：
- 清晰的结构可以让大模型的输出符合率提升50%以上
- 输出格式明确约束可以大幅减少后续的代码/文本解析成本

## 三、效果评估体系
### 🔹 评估维度
✅ **开发岗要求**：
| 维度 | 说明 | 评估方法 |
|------|------|----------|
| 准确率 | 输出内容是否符合任务要求，事实是否正确 | 人工抽检+自动化规则校验 |
| 相关性 | 输出是否和任务相关，没有无关内容 | 语义相似度匹配+人工检查 |
| 稳定性 | 多次调用的输出是否一致，波动小 | 多次重复调用，统计结果差异率 |
| 效率 | 输出长度是否合适，token消耗是否合理 | 统计平均token消耗、推理时间 |
| 安全性 | 输出是否包含违规、敏感、幻觉内容 | 内容安全检测+人工审核 |

🔬 **算法岗要求·自动评估方法**：
#### 1. 量化指标
- **BLEU/ROUGE**：适用于文本生成、翻译等场景，衡量输出和参考文本的重合度
- **BERTScore**：基于预训练模型的语义相似度指标，比BLEU更符合人类语义判断
- **LLM-as-Judge**：用大模型作为裁判，自动评估输出质量，和人工标注一致性达85%以上，是当前主流评估方案
  - 评估Prompt：`"请评估以下回答是否符合任务要求：任务：{task}，回答：{output}，请从准确率、相关性、完整性三个维度打分，1-10分，并给出理由"`
#### 2. 评估数据集构建
- 测试集需要覆盖常见场景、边界场景、异常场景
- 标注标准统一，避免人工标注偏差
- 测试集大小根据任务复杂度确定，通常100~1000个样本即可获得统计显著性

## 四、面试高频考点（分岗位）
### ✅ 开发岗通用题
1. **Q：什么是提示词工程？它的核心价值是什么？**
   答：提示词工程是通过优化输入给大模型的文本，引导大模型输出符合预期结果的技术。核心价值是：① 提升输出准确率，让任务效果达到可用水平；② 降低输出随机性，减少幻觉和偏离需求的情况；③ 降低token消耗和推理成本；④ 提升输出安全性，避免违规内容。

2. **Q：写好提示词的核心原则是什么？有什么通用结构？**
   答：核心原则是CLEAR原则：清晰明确、逻辑结构化、要求明确、信息准确、内容相关。通用结构是三段式：① 角色设定，定义模型的身份和专业领域；② 任务要求，明确具体要做什么，有哪些约束条件；③ 输出格式，规定输出的结构、格式、长度要求。

3. **Q：如何评估提示词的效果？有哪些常用维度？**
   答：从五个维度评估：① 准确率：输出内容是否正确符合要求；② 相关性：输出是否和任务相关，没有无关内容；③ 稳定性：多次调用的输出是否一致；④ 效率：token消耗是否合理，推理速度是否够快；⑤ 安全性：是否有违规或幻觉内容。评估方法可以用人工抽检+自动化校验+A/B测试结合。

4. **Q：什么是上下文窗口？它对提示词有什么影响？**
   答：上下文窗口是大模型能够处理的最大输入+输出长度，不同模型的窗口大小不同，比如GPT-4是128k，豆包4是256k。影响是：① Prompt长度不能超过窗口限制，否则会被截断；② 合理分配窗口空间，优先放关键信息，避免无关内容占用窗口；③ 输出长度也会占用窗口，需要预留足够的输出空间。

### 🔬 算法岗专项题
1. **Q：提示词工程的底层原理是什么？什么是上下文学习？**
   答：提示词工程的底层是大模型的上下文学习（ICL）能力：大模型在预训练阶段学习了大量知识和模式，推理时通过Prompt中的上下文信息激活对应的知识和模式，不需要微调模型就能完成特定任务。上下文学习包括零样本学习（只有指令没有示例）和少样本学习（带有少量示例），核心是大模型通过少量示例就能快速学习任务分布，这是大模型的涌现能力之一。

2. **Q：为什么思维链（CoT）能提升大模型的推理能力？背后的原理是什么？**
   答：思维链提升推理能力的核心原理有三个：① 中间监督信号：让模型输出中间步骤相当于提供了更细粒度的监督信号，引导模型按正确的推理路径解码；② 计算资源分配：多步推理将复杂问题分解为多个简单步骤，每个步骤需要的计算资源更少，更容易得到正确结果；③ 错误定位：中间步骤可以帮助定位推理错误的位置，也方便人类校验。从概率角度看，分步推理将一个低概率的复杂任务分解为多个高概率的简单任务，整体成功率大幅提升。

3. **Q：LLM-as-Judge评估方法有什么优缺点？如何提升评估的准确性？**
   答：优点：① 不需要大量人工标注，成本低效率高；② 可以评估语义相关性、逻辑合理性等传统指标难以衡量的维度；③ 和人类判断的一致性可达85%以上。缺点：① 有评估偏见，不同大模型的评估结果可能不一致；② 对复杂任务的评估准确性下降；③ 成本比传统指标高。提升准确性的方法：① 用能力更强的大模型作为裁判；② 提供明确的评估标准和评分规则；③ 用Few-shot示例给裁判参考；④ 多个裁判投票，取平均结果。

4. **Q：Few-shot学习的效果受哪些因素影响？如何优化样本选择？**
   答：影响因素：① 样本数量：通常2~8个样本效果最好，过多会占用窗口空间且边际收益递减；② 样本质量：样本的代表性、标注正确性、多样性；③ 样本排序：一般来说，相似的样本放在一起，难度由易到难排序效果更好；④ 样本和测试样例的相似度越高，效果越好。优化样本选择的方法：① 基于语义相似度选择和任务最相关的样本；② 基于难度选择覆盖不同边界情况的样本；③ 多样性采样，避免样本重复；④ 动态选择，根据输入查询动态匹配最相关的样本。
