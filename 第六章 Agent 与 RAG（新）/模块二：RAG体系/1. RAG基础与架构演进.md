## 一、RAG核心基础
### 1. 什么是RAG
RAG（Retrieval-Augmented Generation，检索增强生成）是将外部知识库检索与大模型生成相结合的技术，核心解决大模型的三大痛点：
✅ 幻觉问题：基于检索到的事实生成，减少编造内容
✅ 知识过时：知识库可以实时更新，不需要重新训练模型
✅ 领域知识不足：可以接入专业领域知识库，不需要微调大模型
### 2. 核心价值
- 事实准确率提升60%以上，专业领域场景可达95%
- 知识更新成本低，只需要更新知识库，不需要微调模型
- 可解释性强，生成结果可以溯源到对应的知识库片段

## 二、RAG架构演进（2026主流：Agentic RAG）「面试重点」
### 🔹 RAG 1.0：Naive RAG（2023年初）
#### 核心流程
```
用户查询 → 向量检索 → 拼接上下文 → LLM生成回答
```
#### 优缺点
✅ 实现简单，容易落地
❌ 检索质量差，召回率低，效果上限低
❌ 不支持复杂查询和多跳推理
#### 淘汰原因：只能处理简单的单片段查询，无法满足复杂场景需求

### 🔹 RAG 2.0：Advanced RAG（2023年底~2024年）
#### 核心优化点
- 数据处理：语义切片、元数据管理、文档清洗
- 查询优化：Query改写、HyDE、子问题分解
- 检索优化：多路检索、混合检索、重排序
- 后处理：结果校验、引用溯源、去重过滤
#### 核心流程
```
用户查询 → 查询优化 → 多路检索 → 结果融合 → 重排序 → 拼接上下文 → LLM生成 → 结果校验
```
#### 优缺点
✅ 检索准确率大幅提升，召回率提升40%以上
✅ 支持中等复杂度的查询
❌ 检索逻辑固定，无法自适应不同查询的需求
❌ 检索和生成割裂，无法根据生成结果动态调整检索策略
#### 淘汰原因：对复杂查询和多跳推理场景的支持依然不足，检索策略不灵活

### 🔹 RAG 3.0：Agentic RAG（2025年至今，当前主流）「面试重点」
#### 核心思想
将检索作为Agent的一个工具，由大模型自主决策：是否需要检索、什么时候检索、检索什么内容、如何利用检索结果，实现检索和生成的深度融合。
#### 核心流程
```
用户查询 → LLM判断是否需要检索 → 需要则调用检索工具 → 拿到结果后判断是否需要二次检索 → 综合所有信息生成回答
```
#### 核心优势
✅ 自适应检索：简单问题直接回答，复杂问题才检索，节省30%以上成本
✅ 支持多跳推理：可以多次检索，关联多段信息回答复杂问题
✅ 动态调整：可以根据检索结果调整查询策略，重新检索
✅ 准确率高：复杂任务准确率比RAG 2.0提升50%以上
#### 面试重点：三代RAG的对比
| 维度 | RAG 1.0 | RAG 2.0 | RAG 3.0 |
|------|---------|---------|---------|
| 核心逻辑 | 固定检索流程 | 优化检索环节 | 检索作为工具，自适应决策 |
| 准确率 | 60%~70% | 70%~85% | 85%~95% |
| 多跳支持 | 不支持 | 有限支持 | 原生支持 |
| 成本 | 低 | 中 | 中（自适应检索降低无效检索成本） |
| 适用场景 | 简单FAQ | 通用场景 | 复杂场景、专业领域 |

## 三、RAG效果评估体系（2026标准方案）
### 🔹 核心评估指标
| 指标类型 | 指标 | 说明 | 达标值 |
|----------|------|------|--------|
| 检索指标 | 召回率 | 相关文档被检索到的比例 | >90% |
|  | 精确率 | 检索到的文档中相关的比例 | >80% |
|  | NDCG@K | 排序质量指标 | >0.85 |
| 生成指标 | 忠实度 | 生成结果和检索内容的一致性 | >90% |
|  | 相关性 | 生成结果和用户查询的相关性 | >85% |
|  | 事实准确率 | 生成内容的事实正确性 | >95% |
| 业务指标 | 满意度 | 用户点赞率 | >85% |
|  | 解决率 | 问题被直接解决的比例 | >80% |

### 🔹 主流评估工具
- **RAGAS 3.0**：开源自动评估工具，支持自动计算忠实度、相关性、准确率等指标，无需人工标注
- **LLM-as-Judge**：用大模型作为裁判，自动评估回答质量，和人工标注的一致性达85%以上
- **人工评估**：核心场景和bad case人工审核，保证评估准确性

## 四、面试高频考点
1. **Q：RAG解决了大模型的什么核心问题？**
   答：三大核心问题：① 幻觉问题，基于检索到的事实生成，减少编造；② 知识过时问题，知识库可以实时更新，不需要重新训练模型；③ 领域知识不足问题，可以接入专业领域知识库，不需要微调模型。

2. **Q：RAG 3.0（Agentic RAG）和传统RAG的核心区别是什么？**
   答：核心区别是检索的控制权从固定流程交给了大模型自主决策：① 传统RAG是固定流程，不管什么查询都走完整检索链路；② Agentic RAG把检索作为工具，大模型自主判断是否需要检索、检索什么内容，支持多次检索和动态调整；③ 支持多跳推理，复杂任务准确率提升50%以上。

3. **Q：三代RAG各有什么优缺点？分别适用什么场景？**
   答：① RAG 1.0：实现简单，效果差，适合简单FAQ场景和快速原型验证；② RAG 2.0：检索准确率高，流程固定，适合通用场景和中等复杂度查询；③ RAG 3.0：自适应检索，支持复杂查询和多跳推理，适合专业领域和复杂场景，是2026年的主流方案。

4. **Q：如何评估RAG系统的效果？**
   答：从三个维度评估：① 检索指标：召回率、精确率、NDCG@K，衡量检索质量；② 生成指标：忠实度、相关性、事实准确率，衡量生成质量；③ 业务指标：用户满意度、问题解决率，衡量实际业务效果。现在主流用RAGAS 3.0自动评估，配合LLM-as-Judge和人工抽检，效率高且准确。
