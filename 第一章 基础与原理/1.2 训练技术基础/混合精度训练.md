## 1. 一句话定义：混合精度训练是什么

**混合精度训练**：同时使用 16 位浮点数（FP16）和 32 位浮点数（FP32）进行训练，在保持模型精度的前提下，显著降低显存占用并提升计算速度。

直觉理解：

- 前向/反向传播：使用 FP16（更快、更省显存）
- 参数更新：使用 FP32（避免精度损失）
- 关键：通过 Loss Scaling 解决 FP16 下的梯度下溢问题

---

## 2. 为什么需要混合精度训练：三个核心动机

### 2.1 显存节省：FP16 仅需 FP32 的一半显存

**显存对比**：

| 数据类型 | 位宽 | 每个参数大小 | 7B 模型占用 | 13B 模型占用 | 70B 模型占用 |
| --- | --- | --- | --- | --- | --- |
| FP32 | 32 位 | 4 字节 | ~28GB | ~52GB | ~280GB |
| FP16 | 16 位 | 2 字节 | ~14GB | ~26GB | ~140GB |
| BF16 | 16 位 | 2 字节 | ~14GB | ~26GB | ~140GB |

**LLM 训练场景**：

- 7B 模型 + FP16：单卡 24GB 即可训练（如 RTX 3090/4090）
- 13B 模型 + FP16：单卡 40GB 即可训练（如 A100 40GB）
- 70B 模型 + FP16：8 卡 A100 40GB 即可训练

### 2.2 计算加速：FP16 计算更快

**硬件支持**：

- NVIDIA GPU：Tensor Cores 对 FP16 计算有**2-8倍**加速
- 现代 GPU：FP16 吞吐量远高于 FP32

**训练速度提升**：

- 前向/反向传播：~2× 速度提升
- 激活值存储：~50% 显存节省
- 总体训练速度：1.5-2× 提升（取决于模型和硬件）

### 2.3 数值精度：如何在节省显存的同时保持精度

**挑战**：

- FP16 动态范围小（-65504 到 65504）
- 梯度值可能很小，容易下溢为 0
- 损失函数值也可能很小，导致精度损失

**解决方案**：

- **Loss Scaling**：训练时缩放损失值，避免梯度下溢
- **Master Parameters**：使用 FP32 存储主参数，FP16 用于计算
- **自动混合精度**（AMP）：PyTorch 自动处理类型转换

## 3. 核心原理：混合精度的数学基础

### 3.1 FP16 vs FP32 的数值特性

**FP32（单精度）**：
- 符号位：1 位
- 指数位：8 位
- 尾数位：23 位
- 动态范围：约 ±1.18×10^-38 到 ±3.4×10^38
- 精度：约 7 位小数

**FP16（半精度）**：
- 符号位：1 位
- 指数位：5 位
- 尾数位：10 位
- 动态范围：约 ±6.1×10^-5 到 ±65504
- 精度：约 3-4 位小数

**BF16（Brain Float 16）**：
- 符号位：1 位
- 指数位：8 位
- 尾数位：7 位
- 动态范围：与 FP32 相同
- 精度：约 2-3 位小数
- 优点：适合深度学习，抗溢出能力强

### 3.2 Loss Scaling 原理

**问题**：梯度值可能远小于 FP16 的最小可表示值（约 6.1×10^-5），导致下溢为 0。

**解决思路**：
1. 训练前将损失值放大 $s$ 倍
2. 反向传播得到放大的梯度
3. 更新参数前将梯度缩小 $s$ 倍

**数学推导**：

设原始损失为 $L$，缩放因子为 $s$，参数为 $	heta$。

**缩放后的损失**：
$$
L_{scaled} = s dot L
$$

**缩放后的梯度**：
$$
\nabla L_{scaled} = s dot \nabla L
$$

**参数更新**：
$$
\theta \leftarrow \theta - \eta \cdot \frac{\nabla L_{scaled}}{s} = \theta - \eta \cdot \nabla L
$$

**结论**：缩放不影响最终的参数更新，只是避免了梯度下溢。

### 3.3 混合精度的计算流程

```
1. 模型参数：FP32（主参数）
2. 前向传播：
   - 参数转换为 FP16
   - 计算得到 FP16 激活值
   - 计算损失（FP16）
3. Loss Scaling：
   - 损失放大 s 倍
4. 反向传播：
   - 计算 FP16 梯度
   - 梯度放大 s 倍
5. 参数更新：
   - 梯度转换为 FP32
   - 梯度缩小 s 倍
   - 使用 FP32 梯度更新 FP32 主参数
```

## 4. 工程实现：PyTorch AMP 代码模板

### 4.1 基础模板（单卡）

```python
from torch.cuda.amp import autocast, GradScaler

# 1. 初始化 GradScaler
scaler = GradScaler()

# 2. 训练循环
for epoch in range(num_epochs):
    for step, (inputs, labels) in enumerate(dataloader):
        inputs, labels = inputs.to('cuda'), labels.to('cuda')
        
        # 3. 前向传播（自动转换为 FP16）
        with autocast():
            outputs = model(inputs)
            loss = criterion(outputs, labels)
        
        # 4. 反向传播（带缩放）
        scaler.scale(loss).backward()
        
        # 5. 更新参数
        scaler.step(optimizer)
        scaler.update()
        optimizer.zero_grad()
```

**关键点**：
- `autocast()` 上下文：自动处理类型转换
- `scaler.scale(loss)`：放大损失
- `scaler.step(optimizer)`：更新参数（内部自动 unscale 梯度）
- `scaler.update()`：动态调整缩放因子

### 4.2 与梯度累积结合

```python
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()
accumulation_steps = 8

for epoch in range(num_epochs):
    for step, (inputs, labels) in enumerate(dataloader):
        inputs, labels = inputs.to('cuda'), labels.to('cuda')
        
        with autocast():
            outputs = model(inputs)
            loss = criterion(outputs, labels) / accumulation_steps  # 缩放 loss
        
        # 缩放 loss 后反向传播
        scaler.scale(loss).backward()
        
        if (step + 1) % accumulation_steps == 0:
            # 可选：梯度裁剪（在 unscale 之后）
            scaler.unscale_(optimizer)
            torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
            
            scaler.step(optimizer)
            scaler.update()
            optimizer.zero_grad()
```

**注意**：
- `loss / accumulation_steps` 必须在 `autocast()` 内部
- 梯度裁剪必须在 `scaler.unscale_()` 之后

### 4.3 与分布式训练（DDP）结合

```python
from torch.cuda.amp import autocast, GradScaler
from torch.nn.parallel import DistributedDataParallel as DDP

# 初始化 DDP
model = DDP(model, device_ids=[local_rank])
scaler = GradScaler()

for epoch in range(num_epochs):
    sampler.set_epoch(epoch)  # 关键：确保每个 epoch 不同的 shuffle
    
    for step, (inputs, labels) in enumerate(dataloader):
        inputs, labels = inputs.to('cuda'), labels.to('cuda')
        
        with autocast():
            outputs = model(inputs)
            loss = criterion(outputs, labels)
        
        scaler.scale(loss).backward()
        scaler.step(optimizer)
        scaler.update()
        optimizer.zero_grad()
```

**注意**：
- DDP 的 `all_reduce` 会自动处理 FP16 梯度
- 不需要额外修改代码

## 5. GradScaler 的工作原理

### 5.1 动态缩放因子调整

GradScaler 会**自动调整**缩放因子 $s$：

1. **初始值**：默认 2^16（65536）
2. **每次迭代**：
   - 如果出现溢出（NaN/Inf）：
     - 缩小缩放因子（乘以 0.5）
     - 跳过本次更新
   - 如果连续多次无溢出：
     - 增大缩放因子（乘以 2）
3. **目标**：找到最大的 $s$，既不溢出又能避免下溢

### 5.2 处理溢出

```python
# 溢出检测与处理（GradScaler 内部逻辑）
try:
    # 尝试更新参数
    scaler.step(optimizer)
except RuntimeError as e:
    if "float16" in str(e) or "nan" in str(e):
        # 发生溢出，缩小缩放因子
        scaler.update(new_scale=scaler.get_scale() * 0.5)
        # 跳过本次更新，继续下一次迭代
    else:
        raise e
```

### 5.3 性能优化

- **内存复用**：GradScaler 会复用中间缓冲区，减少内存分配
- **异步操作**：与 CUDA 流配合，重叠计算与内存操作
- **自适应缩放**：根据实际情况调整缩放因子，平衡速度与稳定性

## 6. 常见问题与解决方案

### 6.1 梯度溢出（NaN/Inf）

**症状**：
- 训练中出现 `NaN` 或 `Inf`
- 损失值突然变得很大

**原因**：
- 缩放因子过大
- 学习率过高
- 模型结构不稳定（如激活函数选择）

**解决方案**：
- 降低初始学习率（如 1e-5 开始）
- 使用 warmup 策略
- 检查模型是否有数值不稳定的操作
- 考虑使用 BF16（抗溢出能力更强）

### 6.2 梯度下溢（Underflow）

**症状**：
- 梯度变为 0
- 模型停止学习
- 验证精度不提升

**原因**：
- 缩放因子过小
- 损失值过小
- 学习率过小

**解决方案**：
- 检查损失函数计算是否正确
- 确保使用了 `scaler.scale(loss).backward()`
- 增加学习率（如果合适）

### 6.3 精度损失

**症状**：
- 模型精度低于 FP32 训练
- 收敛速度变慢

**原因**：
- FP16 精度不足
- 缩放因子选择不当
- 某些操作对精度敏感

**解决方案**：
- 关键层使用 FP32（如 embedding 层）
- 启用 `torch.backends.cudnn.benchmark = True`
- 考虑使用 BF16（如果硬件支持）
- 适当增加训练 epochs

### 6.4 硬件兼容性

**症状**：
- 某些 GPU 不支持 FP16
- 训练速度没有提升

**原因**：
- 旧 GPU 缺少 Tensor Cores
- 驱动版本过低

**解决方案**：
- 检查 GPU 是否支持 FP16（如 Pascal 及以上架构）
- 更新 CUDA 驱动
- 对于不支持的 GPU，考虑使用 BF16 或保持 FP32

## 8. 面试高频问题清单

### 8.1 基础概念

**Q1: 混合精度训练的核心思想是什么？**

A: 同时使用 FP16 和 FP32 进行训练：前向/反向传播使用 FP16（节省显存、加速计算），参数更新使用 FP32（保持精度），通过 Loss Scaling 解决 FP16 下的梯度下溢问题。

**Q2: FP16 相比 FP32 有什么优缺点？**

A: 
- **优点**：显存减半，计算速度提升 2-8 倍（使用 Tensor Cores）
- **缺点**：动态范围小，容易发生梯度下溢或溢出

**Q3: Loss Scaling 的原理是什么？**

A: 训练时放大损失值，使梯度值落在 FP16 的有效范围内，更新参数前再缩小梯度，不影响最终更新结果。

### 8.2 工程实现

**Q4: PyTorch 中如何实现混合精度训练？**

A: 使用 `torch.cuda.amp` 模块：
1. `autocast()` 上下文管理器：自动处理类型转换
2. `GradScaler`：处理 Loss Scaling 和溢出检测
3. 关键步骤：`scaler.scale(loss).backward()`、`scaler.step(optimizer)`、`scaler.update()`

**Q5: 如何处理混合精度训练中的溢出问题？**

A: GradScaler 会自动处理：
- 检测到溢出时，缩小缩放因子并跳过本次更新
- 连续无溢出时，增大缩放因子
- 目标是找到最大的安全缩放因子

**Q6: 混合精度训练与梯度累积如何配合？**

A: 
- `loss / accumulation_steps` 必须在 `autocast()` 内部
- 每 `accumulation_steps` 步调用一次 `scaler.step()` 和 `scaler.update()`
- 梯度裁剪在 `scaler.unscale_()` 之后进行

### 8.3 进阶问题

**Q7: BF16 和 FP16 有什么区别？**

A: 
- **BF16**：8 位指数 + 7 位尾数，动态范围与 FP32 相同，抗溢出能力强
- **FP16**：5 位指数 + 10 位尾数，动态范围小，精度稍高
- **选择**：现代 GPU 推荐 BF16（如 A100、H100），旧 GPU 用 FP16

**Q8: 混合精度训练对模型精度有影响吗？**

A: 通常影响很小，甚至没有影响。通过以下措施可以保证精度：
- 使用 FP32 存储主参数
- 合理的 Loss Scaling
- 关键层保持 FP32（如 embedding 层）

**Q9: 哪些操作不适合使用 FP16？**

A: 
- 数值范围大的操作（如 softmax 归一化）
- 精度要求高的操作（如 batch normalization）
- 梯度裁剪前的操作（需要 unscale 后进行）

## 9. 实战建议与最佳实践

### 9.1 调参建议

**1. 硬件选择**：
- 优先选择支持 Tensor Cores 的 GPU（Pascal 及以上架构）
- A100/H100 推荐使用 BF16
- 旧 GPU 推荐使用 FP16

**2. 初始设置**：
- 学习率：从 FP32 的 1-2 倍开始（因为批量更大）
- 缩放因子：使用默认值（65536）
- `torch.backends.cudnn.benchmark = True`

**3. 监控指标**：
- 显存占用（应比 FP32 减少约 50%）
- 训练速度（应比 FP32 快 1.5-2×）
- 损失曲线（应与 FP32 类似）
- 缩放因子变化（稳定在某个值附近）

### 9.2 常见坑点

- **忘记使用 `autocast()`**：没有启用 FP16 计算
- **错误的缩放位置**：`loss / accumulation_steps` 不在 `autocast()` 内部
- **梯度裁剪时机错误**：在 `scaler.unscale_()` 之前进行
- **学习率未调整**：仍使用 FP32 的学习率，导致收敛慢
- **忽略溢出警告**：训练中出现 `NaN` 但继续训练

### 9.3 性能优化技巧

**1. 内存优化**：
- 使用 `pin_memory=True` 加速数据传输
- 启用 `torch.cuda.empty_cache()` 定期清理缓存
- 考虑使用梯度检查点进一步节省显存

**2. 计算优化**：
- 批处理大小：在显存允许范围内尽量大
- 多卡并行：结合 DDP/FSDP 扩展训练规模
- 混合精度 + 梯度累积：显存与速度的最佳组合

**3. 稳定性优化**：
- 使用 warmup 策略（前 1000-5000 step 线性增长学习率）
- 监控缩放因子变化，及时调整
- 对于不稳定的模型，考虑使用 BF16

## 10. 总结

混合精度训练是 LLM 训练中的**必备技术**，核心价值：

1. **显存节省**：FP16 使模型规模翻倍（如 24GB 显存可训练 7B 模型）
2. **计算加速**：Tensor Cores 带来 1.5-2× 训练速度提升
3. **精度保持**：通过 Loss Scaling 和 FP32 主参数，不损失模型精度

**关键公式**：

- Loss Scaling：$L_{scaled} = s \cdot L$
- 梯度缩放：$\nabla L_{scaled} = s \cdot \nabla L$
- 参数更新：$\theta \leftarrow \theta - \eta \cdot \frac{\nabla L_{scaled}}{s}$

**工程要点**：

- 使用 `torch.cuda.amp` 模块（`autocast` + `GradScaler`）
- 与梯度累积、分布式训练、梯度检查点兼容
- 注意监控缩放因子和溢出情况
- 合理调整学习率和 batch size

掌握混合精度训练后，可以进一步学习：

- 梯度检查点（用计算换显存）
- 分布式训练（多卡/多机扩展）
- 模型并行（超大模型训练）
