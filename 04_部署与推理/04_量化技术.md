## 1 量化核心基础
量化是将模型权重和激活值从高精度（FP16/BF16/FP32）映射到低精度（FP8/INT8/INT4）表示的技术，核心目标是降低显存占用、提升推理速度，同时最小化精度损失。
### 1.1 核心原理
- 本质是用更低的比特数表示数值，通过线性映射公式实现高低精度的双向转换：
  ```
  量化（高精度FP→低精度INT）：INT_value = round(FP_value / scale + zero_point)
  反量化（低精度INT→高精度FP）：FP_value = (INT_value - zero_point) * scale
  ```
  两个核心参数解释：
  - **scale（缩放因子）**：控制范围压缩比例，等于「FP数值的动态范围」除以「INT整数的表示范围」，意思是每个INT整数步长对应的FP数值大小
  - **zero_point（零点偏移）**：解决正负值不对称问题，是FP数值0对应的INT整数值，保证0可以被精确表示，避免量化0的时候产生固定误差
  常见量化模式和粒度：
  - **对称/非对称模式**：对称量化zero_point=0，计算更快，适合对称分布的权重；非对称量化带偏移，精度更高，适合非对称分布的激活值
  - **量化粒度**：per-tensor（层级，整个层共享一组参数，开销小）、per-channel（通道级，每个输出通道一组参数，精度高性能开销低，是权重标配）、per-token（Token级，每个输入Token一组参数，精度最高开销大，适合长上下文激活）
- 显存/计算收益：
  - INT4比FP16降低75%显存占用，推理速度提升2-4倍
  - INT8比FP16降低50%显存占用，推理速度提升1-2倍
  - FP8比FP16降低50%显存占用，精度损失极小（<1%）

### 1.2 常见量化位宽对比
| 位宽 | 显存占比（相对FP16） | 精度损失 | 适用场景 | 硬件支持 |
|------|----------------------|----------|----------|----------|
| FP32 | 200% | 0 | 训练阶段 | 全支持 |
| FP16/BF16 | 100% | 0 | 默认推理、精度要求极高场景 | 全支持 |
| FP8 | 50% | <1% | 生产部署首选，平衡精度和性能 | H100/RTX 40系列及以上 |
| INT8 | 50% | 1%~3% | 高吞吐场景、精度要求中等 | 全支持 |
| INT4 | 25% | 3%~5% | 端侧部署、显存受限场景、成本敏感场景 | 主流GPU均支持 |
| INT3/INT2 | 18%/12% | >10% | 极限成本场景、非关键业务 | 部分框架支持 |

### 1.3 量化的核心收益
- 显存降低：直接线性降低权重和KV Cache显存占用，INT8降低50%、INT4降低75%，相同硬件可以跑更大模型、更高并发
- 速度提升：主流GPU原生支持低精度Tensor Core加速，低精度算力是FP16的2-4倍，同时访存量降低缓解显存带宽瓶颈，实际推理速度提升1.5-4倍
- 成本降低：相同吞吐下硬件成本降低50%以上，是生产部署的标配优化手段
### 1.4 常见认知误区澄清
**误区**：量化推理需要先反量化回FP16，速度比原始模型慢
**真相**：主流GPU原生支持INT8/FP8低精度硬件计算，不需要反量化回FP16，直接用低精度Tensor Core做矩阵乘法，速度远快于FP16；只有非常老旧的硬件才会用反量化兼容模式，生产环境几乎不会遇到。量化的速度收益是确定性的，是现在所有生产服务的必开优化。

## 2 主流量化技术对比
所有量化技术可以分为两类：**独立端到端量化方案（可单独完成全流程量化）** 和 **通用量化增强插件（搭配其他方案使用，提升精度）**，核心目标都是在可接受的精度损失范围内，最大化降低显存占用、提升推理速度。

### 2.1 PTQ（Post-Training Quantization，训练后量化）
- 类型：独立端到端量化方案
- 原理：模型训练完成后直接做量化，不需要重新训练，仅通过少量校准数据统计数值分布计算量化参数，是成本最低的量化方案
- 流程：
  1. 加载训练好的FP16/BF16高精度预训练模型
  2. 准备100~1000条和业务场景分布匹配的无标注校准数据
  3. 用校准数据跑前向传播，统计每层权重、激活值的动态范围（最大值、最小值、分布）
  4. 根据统计结果计算每个层/每个通道的量化参数（scale缩放因子、zero_point零点）
  5. 对权重做离线量化，转成低精度存储；对激活做在线量化，推理时实时转换
- 优势：实现简单、速度快，整个过程仅需几分钟到十几分钟；不需要训练数据和训练资源；对推理框架无特殊要求，所有框架都支持
- 缺陷：INT8精度损失2%左右，INT4精度损失5%以上，对精度要求高的场景不够理想；无法处理激活存在大量异常值的情况，需要搭配SmoothQuant使用
- 适用场景：快速上线验证、数据不足、精度要求中等的场景；端侧轻量化部署的首选低成本方案

### 2.2 QAT（Quantization Aware Training，量化感知训练）
- 类型：独立端到端量化方案
- 原理：在训练/微调阶段就插入伪量化节点模拟真实量化的噪声，让模型提前适应低精度下的数值分布，从而最小化量化后的精度损失，是精度最高的量化方案
- 流程：
  1. 加载训练好的FP16/BF16高精度预训练模型
  2. 在模型的所有权重、激活输入输出位置插入伪量化节点，模拟真实量化的舍入、截断误差
  3. 用和业务场景匹配的标注数据做微调，冻结大部分模型参数，仅微调少量顶层参数或者全参数微调1~2个epoch，学习率用预训练的1/10左右
  4. 微调完成后，移除伪量化节点，导出带有训练好的量化参数的低精度模型
- 优势：精度损失极小，INT8损失<1%，INT4损失<3%，几乎和原模型精度一致；可以处理各种复杂场景的量化需求
- 缺陷：需要标注训练数据；需要GPU训练资源，耗时更长（几小时到几天不等，取决于模型大小）；部分特殊结构的模型适配难度高
- 适用场景：精度要求极高、可以接受微调成本的生产场景；核心业务模型的首选量化方案
- 增强搭配：可搭配SmoothQuant进一步提升INT8精度

### 2.3 GPTQ（Generative Pre-trained Transformer Quantization）
- 类型：独立端到端量化方案
- 原理：专门针对Transformer架构优化的后训练量化方案，核心思想是逐层优化量化后的权重，让该层的输出和原高精度输出的均方误差最小，不需要全局训练就能达到接近QAT的INT4精度
- 流程：
  1. 加载训练好的FP16/BF16高精度预训练模型
  2. 准备128~512条校准数据，从Transformer第一层开始逐层处理
  3. 对当前层的权重做初始INT4量化，然后用分块坐标下降法（类似梯度下降但效率高10倍以上）逐列优化量化后的INT权重，最小化该层输出和原高精度输出的误差
  4. 处理完当前层后，将校准数据输入量化后的层，得到下一层的输入，继续处理下一层，直到所有层都量化完成
  5. 保存量化后的INT4模型，不需要额外微调
- 优势：INT4精度损失<2%，远好于普通PTQ的5%损失，接近QAT精度；不需要训练，仅需要少量无标注校准数据；量化速度介于PTQ和QAT之间，7B模型仅需1~2小时
- 缺陷：需要推理框架专门支持（vLLM、TGI、TensorRT-LLM等主流框架都已支持）；仅优化权重量化，对激活量化没有优化；量化耗时比普通PTQ长
- 适用场景：INT4量化的经典主流方案，生产部署7B/13B/70B大模型的首选，在AWQ普及前是行业标准
- 增强搭配：可搭配AWQ前置优化进一步提升INT4精度

### 2.4 AWQ（Activation-aware Weight Quantization）
- 类型：通用量化增强插件
- 原理：感知激活值分布的权重量化前置优化，核心思想是**仅1%的权重对输出精度影响最大（称为 salient weight，显著权重）**，通过数学等价变换在FP16高精度阶段调整权重分布，放大这部分敏感权重的后续量化精度，不需要保留任何FP16权重，调整后的权重再交给GPTQ/PTQ量化到INT4，在几乎不损失精度的前提下实现高质量INT4量化
- 流程：
  1. 用少量和业务场景匹配的校准数据跑前向传播，统计每层权重和激活的分布，计算每个权重通道对输出结果的灵敏度
  2. 识别出对精度影响最大的 salient weight 通道，计算对应平滑因子
  3. 对敏感权重通道乘平滑因子，对应激活通道除同一个平滑因子（数学上矩阵乘法结果完全不变），放大敏感权重的后续量化精度，缩小激活动态范围，调整后的权重再交由GPTQ/PTQ统一做INT4量化，不需要保留混合精度
  4. 对模型做轻量微调（可选），补偿少量精度损失，得到最终量化模型
- 优势：INT4精度损失<1.5%，比GPTQ精度更高；推理速度比GPTQ快10%-20%；显存占用更低，不需要额外运行时开销
- 缺陷：量化速度较慢，比GPTQ耗时高30%左右；部分老版本推理框架支持不完善
- 适用场景：INT4量化的标配前置优化，是2026年生产部署INT4的首选方案，通常搭配GPTQ/PTQ使用

### 2.5 SmoothQuant
- 类型：通用量化增强插件
- 原理：专门解决INT8量化的核心痛点：**激活值存在大量异常值（outlier）导致动态范围过大，scale被异常值拉得很大，正常数值的量化精度极低，误差过大**。核心思想是把激活的方差平滑迁移到权重上，同时保证计算结果不变，降低激活的动态范围，大幅提升INT8量化精度
- 流程：
  1. 用少量校准数据跑前向传播，统计每层激活值每个通道的动态范围，识别出存在大量异常值的通道
  2. 计算每个通道的平滑因子s，对激活值除以s，对对应的权重乘以s，数学上保证矩阵乘法结果完全不变
  3. 平滑后的激活动态范围大幅缩小，异常值的影响被消除，然后做普通PTQ/QAT INT8量化
  4. 直接输出量化模型，不需要任何微调，精度损失极小
- 优势：不需要微调，INT8精度损失<1%，和QAT相当；量化速度和普通PTQ一样快，几乎没有额外开销；和所有主流推理框架兼容
- 缺陷：仅适用于INT8量化，对INT4/FP8量化没有优化效果；只能解决激活异常值导致的精度损失，无法解决其他量化误差
- 适用场景：INT8量化的标配前置优化方案，可搭配所有INT8量化方案使用，效果接近QAT但成本和PTQ一样低

### 2.9 常见认知误区澄清
**误区1**：QAT和PTQ的推理逻辑不一样，需要框架单独适配
**真相**：推理阶段两者的模型结构、计算逻辑、硬件兼容性完全一致，差异只在权重和scale/zp的生成阶段，推理框架不需要区分，导出的模型可以无缝替换。

**误区2**：AWQ是混合精度量化，需要保留部分权重的FP16精度
**真相**：AWQ通过数学等价变换优化权重分布，所有权重都统一量化到INT4，无混合精度额外开销，推理速度比GPTQ还快10%~20%。

**误区3**：GPTQ用了和QAT一样的反向传播梯度下降训练
**真相**：GPTQ用的是分块坐标下降的贪心优化算法，只在量化后的INT矩阵上做局部调整，不需要反向传播训练原模型，也不需要训练pipeline。

**误区4**：SmoothQuant需要改动模型结构或者微调
**真相**：SmoothQuant是纯离线数值变换，数学上完全等价，不需要微调，也不需要改动推理逻辑，零额外成本。

     - 直接采用FP8原生量化，精度损失<1%，几乎无额外成本

## 3 量化精度影响与优化
### 3.1 精度损失的核心原因
- 截断误差：数值范围超出量化表示范围，被截断导致误差
- 舍入误差：四舍五入导致的误差
- 异常值影响：少数极大/极小值导致scale过大，大部分数值量化精度不足
### 3.2 优化手段
- 校准数据选择：用和业务场景一致的校准数据，分布匹配度越高精度损失越小
- 量化粒度：按通道量化（per-channel）比按层量化（per-layer）精度更高，计算量稍大
- 混合精度量化：对精度敏感的层用高位宽，不敏感的层用低位宽，平衡精度和性能
- 量化感知微调：量化后用少量数据微调，恢复精度损失

## 4 面试高频考点
### 开发岗通用题
#### Q1：**量化的核心作用是什么？生产部署为什么必须做量化？**
量化核心作用是降低显存占用、提升推理速度、降低部署成本。生产部署必须做量化的原因：
- 显存占用降低50%~75%，相同GPU可以跑更大模型、更高并发
- 推理速度提升1~4倍，吞吐更高，成本更低
- 当前主流量化方案精度损失已经控制在5%以内，大部分业务场景可以接受，收益远大于损失

#### Q2：**PTQ和QAT有什么区别？分别适合什么场景？**
核心区别：
- 实现方式：PTQ是训练后直接量化，不需要重新训练；QAT是训练过程中模拟量化噪声，需要微调
- 精度：PTQ INT8精度损失2%左右，INT4损失5%以上；QAT INT8损失<1%，INT4损失<3%
- 成本：PTQ成本极低，几分钟完成，不需要数据；QAT成本高，需要训练数据和GPU资源，耗时几小时到几天
适用场景：PTQ适合快速上线、精度要求中等的场景；QAT适合精度要求高、可以接受微调成本的生产场景。

#### Q3：**GPTQ和AWQ都是INT4量化方案，有什么区别？怎么选？**
GPTQ是逐层最小化量化误差的经典INT4方案，支持度好，精度不错；AWQ是更新的方案，感知激活值分布，优化权重的量化精度，精度更高（损失<1.5% vs GPTQ的<2%），推理速度更快。选型建议：优先选AWQ，除非框架不支持，目前vLLM、TGI等主流框架都已经支持AWQ，是当前INT4量化的首选。

#### Q4：**FP8和INT8有什么区别？哪个更好？**
FP8是浮点量化，INT8是整数量化：
- 精度：FP8精度损失<1%，几乎和FP16相当，远好于INT8的1%~3%损失
- 显存占用：两者都是50% FP16占用，一样
- 硬件支持：FP8需要H100/RTX40系列及以上新硬件支持，INT8全硬件支持
- 速度：FP8与INT8推理速度相当，部分场景下略快，核心优势是精度损失更小
选型建议：硬件支持的情况下优先选FP8，精度更高速度更快；旧硬件用INT8+SmoothQuant也能达到不错的效果。

### 算法岗专项题
#### Q1：**量化的误差来源有哪些？怎么最小化INT4量化的精度损失？**
误差来源：
- 截断误差：数值超出量化范围被截断
- 舍入误差：四舍五入引入的误差
- 分布不匹配误差：校准数据和实际数据分布不一致导致的scale计算不准
最小化INT4损失的方法：
- 用AWQ/GPTQ等专门针对Transformer的量化方案，比普通PTQ精度高很多
- 用业务场景的真实数据做校准，分布匹配度越高越好
- 用per-channel量化粒度，比per-layer精度更高
- 量化后用少量数据做LoRA微调，可以恢复大部分精度损失

#### Q2：**为什么SmoothQuant可以大幅提升INT8量化的精度？原理是什么？**
INT8量化的主要误差来源是激活值的分布有很多异常值（outlier），导致scale计算过大，大部分正常数值的量化精度不足。SmoothQuant的原理是：将激活值的异常值平滑处理，把激活值的方差迁移到权重上，让激活值的分布更均匀，从而可以用更小的scale，提升量化精度。它不需要微调，只需要对权重和激活做简单的线性变换，就可以让INT8的精度接近FP16，损失<1%。

#### Q3：**量化后的模型推理速度提升的核心原因是什么？**
核心原因有两个：
- 计算量降低：低精度计算单元的吞吐量更高，比如A100的FP16算力是312TFLOPS，INT8算力是624TOPS，是两倍
- 访存量降低：模型权重和KV Cache的体积变小，显存带宽压力降低，而大模型推理通常是显存带宽 bound，访存量降低直接提升推理速度
两者共同作用，量化后速度通常可以提升1~4倍。

#### Q4：**什么是混合精度量化？怎么设计混合精度量化策略？**
混合精度量化是不同的层用不同的量化位宽，对精度敏感的层用高位宽（FP16/FP8），对精度不敏感的层用低位宽（INT8/INT4），平衡精度和性能。设计策略：
- 先做敏感性分析，测试各层量化对最终精度的影响，敏感度高的层用高位宽
- 通常注意力层对量化更敏感，FFN层敏感度低
- 第一层和最后一层对精度影响大，通常用高位宽
- 控制高位宽层的比例，在精度损失<1%的前提下尽量用低位宽，最大化性能收益
