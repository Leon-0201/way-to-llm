# 6.2 Agent 核心能力

> **核心理解**：Agent 的核心能力是支撑其自主执行任务的关键技能。包括任务规划与分解、工具使用与选择、自我反思与修正、多 Agent 协作四大能力。这些能力共同决定了 Agent 的智能水平和任务完成效果。

---

## 1. 任务规划与分解

### 1.1 什么是任务规划

**定义**：
任务规划是将复杂、模糊的用户需求转化为清晰、可执行的行动序列的过程。

**核心挑战**：
```
1. 目标模糊：用户需求不明确或不完整
2. 依赖复杂：子任务之间存在依赖关系
3. 资源约束：时间、成本、工具限制
4. 环境不确定：外部条件可能变化
5. 错误处理：执行失败需要调整计划
```

---

### 1.2 规划方法

#### 方法 1：层次化分解（Hierarchical Decomposition）

**核心思想**：将任务按层次逐级分解，形成树状结构。

**分解原则**：
```
1. MECE 原则：相互独立，完全穷尽
2. 可执行性：叶子节点必须是可执行的动作
3. 逻辑清晰：父子关系明确
4. 粒度适中：不过粗也不过细
```

**示例**：
```
任务：组织一场技术会议

Level 1:
├── 会前准备
├── 会议执行
└── 会后跟进

Level 2（会前准备）:
├── 确定会议主题和日期
├── 预订场地
├── 邀请嘉宾
├── 宣传推广
└── 参会者管理

Level 3（确定会议主题和日期）:
├── 调研热门话题
├── 征询潜在嘉宾意见
├── 考察场地可用性
└── 确定最终方案
```

---

#### 方法 2：依赖图规划（Dependency Graph Planning）

**核心思想**：使用有向图表示任务依赖关系，基于图算法确定执行顺序。

**使用示例**：
```python
# 创建规划器
planner = TaskPlanner()

# 添加任务（带依赖）
planner.add_task("收集数据", collect_data)
planner.add_task("清洗数据", clean_data, ["收集数据"])
planner.add_task("分析数据", analyze_data, ["清洗数据"])
planner.add_task("生成报告", generate_report, ["分析数据"])
planner.add_task("可视化", visualize, ["分析数据"])
planner.add_task("最终演示", final_presentation, ["生成报告", "可视化"])

# 执行
results = planner.execute()
```

---

#### 方法 3：动态规划（Dynamic Planning）

**核心思想**：根据执行情况和环境变化动态调整计划。

**实现流程**：
```
1. 制定初始计划
2. 执行当前步骤
3. 监控执行结果和环境变化
4. 评估是否需要调整计划
5. 如需调整，重新规划
6. 重复 2-5 直到完成
```

---

### 1.3 规划质量评估

**评估维度**：
```
1. 完整性
   - 是否覆盖所有必要步骤
   - 是否考虑边界情况
   - 是否有应急方案

2. 可行性
   - 资源是否充足
   - 时间是否合理
   - 技术是否可行

3. 效率
   - 步骤数量是否合理
   - 是否有冗余步骤
   - 并行化程度

4. 鲁棒性
   - 错误处理能力
   - 环境变化适应性
   - 备用方案
```

**评估提示词**：
```
请评估以下计划的质量：

计划：
{plan}

评估维度（1-5 分）：
1. 完整性：是否覆盖所有必要步骤？
2. 可行性：计划是否可执行？
3. 效率：步骤是否精简？
4. 鲁棒性：是否有错误处理？

请给出具体分数和改进建议。
```

## 2. 工具使用与选择

### 2.1 工具表示

**工具定义结构**：
```python
class Tool:
    def __init__(self, name, description, parameters, func):
        self.name = name  # 工具名称
        self.description = description  # 功能描述
        self.parameters = {  # 参数定义
            "param1": {
                "type": "string",
                "description": "参数说明",
                "required": True
            },
            # ...
        }
        self.func = func  # 执行函数
    
    def to_prompt(self):
        """转换为提示词格式"""
        return f"""
工具名称：{self.name}
功能描述：{self.description}
参数：
{json.dumps(self.parameters, indent=2)}
"""
```

**工具注册表**：
```python
class ToolRegistry:
    def __init__(self):
        self.tools = {}
    
    def register(self, tool):
        """注册工具"""
        self.tools[tool.name] = tool
    
    def get_tool(self, name):
        """获取工具"""
        return self.tools.get(name)
    
    def list_tools(self):
        """列出所有工具"""
        return list(self.tools.keys())
    
    def search_tools(self, query):
        """根据查询搜索相关工具"""
        # 使用语义相似度搜索
        relevant = []
        for name, tool in self.tools.items():
            similarity = calculate_similarity(query, tool.description)
            if similarity > THRESHOLD:
                relevant.append((name, similarity))
        
        return sorted(relevant, key=lambda x: x[1], reverse=True)
```

### 2.2 工具选择策略

**基于规则的选择**：使用预定义规则匹配工具和任务。
**基于语义的选择**：使用向量相似度匹配任务和工具。
**基于 LLM 的选择**：让 LLM 根据任务描述选择合适工具。

---

### 2.3 工具使用最佳实践

**设计原则**：
```
1. 单一职责：每个工具只做一件事
2. 明确接口：参数和返回值定义清晰
3. 错误处理：完善的异常处理
4. 幂等性：重复调用结果一致
5. 可观测性：详细的日志记录
```

**安全考虑**：
```
1. 权限控制
   - 工具调用权限验证
   - 敏感操作需要确认
   - 资源使用限制

2. 输入验证
   - 参数合法性检查
   - 注入攻击防护
   - 大小限制

3. 输出过滤
   - 敏感信息脱敏
   - 结果完整性验证
   - 错误信息处理
```

## 3. 自我反思与修正

### 3.1 反思的类型

#### 类型 1：输出反思（Output Reflection）

**目的**：评估生成内容的质量。

**反思维度**：
```
1. 准确性
   - 信息是否正确
   - 计算是否准确
   - 逻辑是否严密

2. 完整性
   - 是否覆盖所有要点
   - 是否遗漏重要信息
   - 是否回答用户问题

3. 一致性
   - 前后是否矛盾
   - 与已知事实是否一致
   - 格式是否统一

4. 相关性
   - 是否切题
   - 是否有冗余信息
   - 是否满足用户需求
```

---

#### 类型 2：过程反思（Process Reflection）

**目的**：评估执行过程的有效性。

**反思内容**：
```
1. 策略评估
   - 选择的策略是否合适
   - 是否有更好的方法
   - 工具选择是否最优

2. 效率评估
   - 步骤是否冗余
   - 是否有不必要的调用
   - 时间消耗是否合理

3. 资源评估
   - token 使用是否合理
   - 工具调用次数
   - 成本控制
```

---

#### 类型 3：错误反思（Error Reflection）

**目的**：分析错误原因，避免重复犯错。

**错误分类**：
```
1. 理解错误
   - 误解用户意图
   - 忽略关键信息
   - 错误假设

2. 规划错误
   - 计划不完整
   - 依赖关系错误
   - 资源估计不足

3. 执行错误
   - 工具调用失败
   - 参数错误
   - 超时或异常

4. 验证错误
   - 未检测到错误
   - 错误判断
```

---

### 3.2 修正策略

**直接修正**：小错误，局部修改即可。
**重新生成**：错误严重，需要重新思考。
**增量改进**：输出基本正确，需要补充或优化。

---

### 3.3 反思循环

```
1. 执行任务
2. 反思输出
3. 评估是否需要修正
4. 选择修正策略
```

## 4. 多 Agent 协作

### 4.1 协作架构

#### 架构 1：中心化协作（Centralized Collaboration）

**特点**：
```
- 一个主 Agent 负责协调
- 多个工作 Agent 执行具体任务
- 主 Agent 负责任务分配和结果整合
```

**架构图**：
```
        ┌─────────────┐
        │ 主 Agent    │
        │ (Coordinator)│
        └──────┬──────┘
               │
       ┌───────┼───────┐
       │       │       │
  ┌────▼───┐ ┌▼────┐ ┌─▼────┐
  │Agent A │ │Agent B│ │Agent C│
  └────────┘ └─────┘ └──────┘
```

---

#### 架构 2：对等协作（Peer-to-Peer Collaboration）

**特点**：
```
- Agent 之间地位平等
- 通过对话协商完成任务
- 没有固定的主从关系
```

**架构图**：
```
  ┌────────┐
  │Agent A │◄────┐
  └───┬────┘     │
      │          │
  ┌───▼────┐     │
  │Agent B │     │
  └───┬────┘     │
      │          │
  ┌───▼────┐     │
  │Agent C │─────┘
  └────────┘
（网状协作）
```

---

#### 架构 3：流水线协作（Pipeline Collaboration）

**特点**：
```
- 任务按顺序流经多个 Agent
- 每个 Agent 负责一个处理阶段
- 输出作为下一个 Agent 的输入
```

**架构图**：
```
任务 → [Agent A] → [Agent B] → [Agent C] → 结果
       (处理 1)    (处理 2)    (处理 3)
```

---

### 4.2 协作机制

#### 机制 1：任务分配（Task Assignment）

**分配策略**：
```
1. 分析任务需求
2. 评估 Agent 能力
3. 选择最优 Agent
```

---

#### 机制 2：冲突解决（Conflict Resolution）

**冲突类型**：
```
1. 资源冲突：多个 Agent 竞争同一资源
2. 决策冲突：对同一问题有不同方案
3. 优先级冲突：任务优先级不一致
```

**解决方法**：
```
1. 收集各方案
2. 评估各方案
3. 选择最优方案
```

---

#### 机制 3：结果整合（Result Integration）

**整合策略**：
```
请综合以下结果：
{format_results(results)}

要求：
1. 提取各结果的关键信息
2. 消除矛盾和重复
3. 形成连贯的整体
4. 保持逻辑清晰

综合结果：
```

---

### 4.3 协作最佳实践

**设计原则**：
```
1. 明确角色：每个 Agent 的职责清晰
2. 有效沟通：建立标准化的通信协议
3. 容错机制：单个 Agent 失败不影响整体
4. 可扩展性：易于添加新的 Agent
5. 可观测性：协作过程可追踪
```

**通信协议**：
```python
class AgentMessage:
    def __init__(self, sender, receiver, content, message_type):
        self.sender = sender
        self.receiver = receiver
        self.content = content
        self.type = message_type  # request, response, notify, etc.
        self.timestamp = time.time()
        self.thread_id = None  # 对话线程 ID
    
    def to_dict(self):
        return {
            "sender": self.sender,
            "receiver": self.receiver,
            "content": self.content,
            "type": self.type,
            "timestamp": self.timestamp,
            "thread_id": self.thread_id
        }
```

## 5. 面试高频问答

### 5.1 基础概念类

**Q1: Agent 的核心能力有哪些？**

**速答模板**：
```
四大核心能力：

1. 任务规划与分解
   - 将复杂任务分解为可执行步骤
   - 方法：层次化分解、依赖图规划、动态规划
   - 关键：MECE 原则、可执行性、依赖管理

2. 工具使用与选择
   - 根据任务选择合适的工具
   - 策略：基于规则、基于语义、基于 LLM
   - 关键：工具表示、选择算法、执行监控

3. 自我反思与修正
   - 评估输出质量，发现并修正错误
   - 类型：输出反思、过程反思、错误反思
   - 关键：评估标准、修正策略、反思循环

4. 多 Agent 协作
   - 多个 Agent 协同完成复杂任务
   - 架构：中心化、对等、流水线
   - 关键：任务分配、冲突解决、结果整合

这四大能力共同决定了 Agent 的智能水平。
```

---

**Q2: 任务规划有哪些方法？**

**速答模板**：
```
三种主要方法：

1. 层次化分解
   - 树状结构，逐级分解
   - 原则：MECE、可执行性
   - 适用：结构清晰的任务
   - 示例：会议组织 → 会前/会中/会后

2. 依赖图规划
   - 有向图表示依赖关系
   - 算法：拓扑排序确定执行顺序
   - 适用：有复杂依赖的任务
   - 示例：数据处理流水线

3. 动态规划
   - 根据执行情况动态调整
   - 流程：执行 → 监控 → 评估 → 调整
   - 适用：环境变化快的场景
   - 示例：实时任务调度

选择建议：
- 结构清晰：层次化分解
- 依赖复杂：依赖图规划
- 环境多变：动态规划
```

### 5.2 实践应用类

**Q3: 如何设计工具选择机制？**

**速答模板**：
```
我的设计框架：

1. 工具表示

   - 注册表管理

   - 执行前检查
   - 工具调用
   - 结果验证
   - 错误处理

4. 优化措施
   - 缓存常用工具选择
   - 批量调用减少延迟
   - 监控工具使用情况

实践建议：
- 小系统：基于规则即可
- 中等系统：规则 + 语义
- 复杂系统：三种策略结合
- 关键：完善的错误处理和监控
```

**速答模板**：
```
实现框架：

   - 直接修正：小错误局部修改

4. 反思循环
   - 执行 → 反思 → 评估 → 修正

实践建议：
- 设置合理的评估阈值
- 避免过度反思（成本高）
- 记录反思历史用于学习
- 关键场景使用，简单场景可省略

### 5.3 进阶理解类

**Q5: 多 Agent 协作有哪些挑战？如何解决？**

**速答模板**：
```
主要挑战：

1. 任务分配
   - 挑战：如何选择合适的 Agent

   - 挑战：如何整合多个 Agent 的输出
   - 解决：合并、投票、选择、综合

5. 错误传播
   - 挑战：单个 Agent 错误影响整体
   - 解决：容错机制、检查点、回滚

6. 可观测性
   - 挑战：协作过程难以追踪
   - 解决：详细日志、可视化、监控

架构选择：
- 简单任务：流水线协作
- 复杂任务：中心化协作
- 创新任务：对等协作
```

## 6. 总结

### 6.1 核心要点
```
1. 任务规划：层次化分解、依赖图、动态规划
2. 工具使用：基于规则/语义/LLM 的选择策略
3. 自我反思：输出/过程/错误反思 + 修正循环
4. 多 Agent 协作：中心化/对等/流水线架构
5. 四大能力相互支撑，共同决定 Agent 智能水平
```

### 6.2 实践建议
```
1. 从简单场景开始，逐步增加复杂度
2. 建立完善的评估和监控体系
3. 重视错误处理和容错机制
4. 根据场景选择合适的架构和策略
5. 持续收集数据，优化核心能力
6. 平衡能力和成本（token、延迟）
```

### 6.3 前沿方向
```
1. 自动化能力学习和优化
