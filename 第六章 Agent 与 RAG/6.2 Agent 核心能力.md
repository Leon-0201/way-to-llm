# 6.2 Agent 核心能力

> **核心理解**：Agent 的核心能力是支撑其自主执行任务的关键技能。包括任务规划与分解、工具使用与选择、自我反思与修正、多 Agent 协作四大能力。这些能力共同决定了 Agent 的智能水平和任务完成效果。

## 1. 任务规划与分解

### 1.1 什么是任务规划

**定义**：
任务规划是将复杂、模糊的用户需求转化为清晰、可执行的行动序列的过程。

**核心挑战**：
```
1. 目标模糊：用户需求不明确或不完整
2. 依赖复杂：子任务之间存在依赖关系
3. 资源约束：时间、成本、工具限制
4. 环境不确定：外部条件可能变化
5. 错误处理：执行失败需要调整计划
```

---

### 1.2 规划方法

#### 方法 1：层次化分解（Hierarchical Decomposition）

**核心思想**：将任务按层次逐级分解，形成树状结构。

**分解原则**：
```
1. MECE 原则：相互独立，完全穷尽
2. 可执行性：叶子节点必须是可执行的动作
3. 逻辑清晰：父子关系明确
4. 粒度适中：不过粗也不过细
```

**示例**：
```
任务：组织一场技术会议

Level 1:
├── 会前准备
├── 会议执行
└── 会后跟进

Level 2（会前准备）:
├── 确定会议主题和日期
├── 预订场地
├── 邀请嘉宾
├── 宣传推广
└── 参会者管理

Level 3（确定会议主题和日期）:
├── 调研热门话题
├── 征询潜在嘉宾意见
├── 考察场地可用性
└── 确定最终方案
```

**实现代码**：
```python
def decompose_task(task, depth=0):
    """层次化任务分解"""
    
    # 1. 判断是否需要继续分解
    if is_executable(task) or depth > MAX_DEPTH:
        return [task]
    
    # 2. 使用 LLM 进行分解
    prompt = f"""
请将以下任务分解为 3-5 个子任务：
任务：{task}

要求：
1. 子任务相互独立
2. 每个子任务具体可执行
3. 说明子任务之间的依赖关系

输出格式（JSON）：
{{
  "subtasks": [
    {"name": "...", "description": "...", "dependencies": []}
  ]
}}
"""
    
    response = llm.generate(prompt)
    subtasks = parse_json(response)
    
    # 3. 递归分解每个子任务
    result = []
    for subtask in subtasks:
        if not is_executable(subtask):
            result.extend(decompose_task(subtask, depth + 1))
        else:
            result.append(subtask)
    
    return result
```

#### 方法 2：依赖图规划（Dependency Graph Planning）

**核心思想**：使用有向图表示任务依赖关系，基于图算法确定执行顺序。

**实现方式**：
```python
from collections import deque

class TaskPlanner:
    def __init__(self):
        self.tasks = {}  # 任务字典
        self.dependencies = {}  # 依赖关系
    
    def add_task(self, task_id, task_func, dependencies=None):
        """添加任务"""
        self.tasks[task_id] = task_func
        self.dependencies[task_id] = dependencies or []
    
    def get_execution_order(self):
        """使用拓扑排序获取执行顺序"""
        # 计算入度
        in_degree = {task: 0 for task in self.tasks}
        for task, deps in self.dependencies.items():
            for dep in deps:
                in_degree[task] += 1
        
        # 拓扑排序
        queue = deque([task for task, degree in in_degree.items() if degree == 0])
        order = []
        
        while queue:
            task = queue.popleft()
            order.append(task)
            
            # 更新依赖该任务的任务的入度
            for other_task, deps in self.dependencies.items():
                if task in deps:
                    in_degree[other_task] -= 1
                    if in_degree[other_task] == 0:
                        queue.append(other_task)
        
        return order
    
    def execute(self):
        """按顺序执行任务"""
        order = self.get_execution_order()
        results = {}
        
        for task_id in order:
            # 获取依赖任务的结果
            dep_results = [
                results[dep] 
                for dep in self.dependencies[task_id]
            ]
            
            # 执行当前任务
            result = self.tasks[task_id](*dep_results)
            results[task_id] = result
        
        return results
```

**使用示例**：
```python
# 创建规划器
planner = TaskPlanner()

# 添加任务（带依赖）
planner.add_task("收集数据", collect_data)
planner.add_task("清洗数据", clean_data, ["收集数据"])
planner.add_task("分析数据", analyze_data, ["清洗数据"])
planner.add_task("生成报告", generate_report, ["分析数据"])
planner.add_task("可视化", visualize, ["分析数据"])
planner.add_task("最终演示", final_presentation, ["生成报告", "可视化"])

# 执行
results = planner.execute()
```

#### 方法 3：动态规划（Dynamic Planning）

**核心思想**：根据执行情况和环境变化动态调整计划。

**实现流程**：
```
1. 制定初始计划
2. 执行当前步骤
3. 监控执行结果和环境变化
4. 评估是否需要调整计划
5. 如需调整，重新规划
6. 重复 2-5 直到完成
```

**示例代码**：
```python
class DynamicPlanner:
    def __init__(self, initial_plan):
        self.plan = initial_plan
        self.executed = []
        self.remaining = initial_plan.copy()
    
    def execute_step(self):
        """执行当前步骤"""
        if not self.remaining:
            return None
        
        current_step = self.remaining[0]
        result = self.execute_with_monitoring(current_step)
        
        self.executed.append({
            "step": current_step,
            "result": result,
            "status": "success" if result else "failed"
        })
        
        return result
    
    def replan_if_needed(self, context_changes):
        """根据环境变化重新规划"""
        # 评估影响
        impact = self.assess_impact(context_changes)
        
        if impact.requires_replan:
            # 调整剩余计划
            new_plan = self.adjust_plan(impact)
            self.remaining = new_plan
            
            return True
        return False
    
    def adjust_plan(self, impact):
        """调整计划"""
        # 1. 移除受影响的步骤
        affected = [
            step for step in self.remaining
            if impact.affects(step)
        ]
        
        for step in affected:
            self.remaining.remove(step)
        
        # 2. 添加新的必要步骤
        new_steps = self.generate_new_steps(impact)
        self.remaining.extend(new_steps)
        
        # 3. 重新排序
        self.remaining = self.reorder_tasks(self.remaining)
        
        return self.remaining
```

```python
def select_tools_by_rules(task, available_tools):
    """基于规则选择工具"""
    
    # 规则库
    rules = {
        "搜索": ["搜索引擎", "知识库查询"],

    return selected
```

#### 策略 2：基于语义的选择

**原理**：使用向量相似度匹配任务和工具。

**实现**：
```python
class SemanticToolSelector:
    def __init__(self, embedding_model):
        self.embedding = embedding_model
        self.tool_embeddings = {}
    
    def index_tools(self, tools):
        """预计算工具向量"""
        for tool in tools:
            # 使用工具描述生成向量
            text = f"{tool.name}: {tool.description}"
            embedding = self.embedding.encode(text)
            self.tool_embeddings[tool.name] = embedding
    
    def select(self, task, top_k=3):
        """选择最相关的工具"""
        # 任务向量
        task_embedding = self.embedding.encode(task)
        
        # 计算相似度
        similarities = []
        for name, tool_emb in self.tool_embeddings.items():
            sim = cosine_similarity(task_embedding, tool_emb)
            similarities.append((name, sim))
        
        # 返回 top-k
        similarities.sort(key=lambda x: x[1], reverse=True)
        return similarities[:top_k]
```

### 2.3 工具调用执行

**执行流程**：
```python
class ToolExecutor:
    def __init__(self, tool_registry):
        self.registry = tool_registry
        self.execution_history = []
    
    def execute(self, tool_name, parameters):
        """执行工具调用"""
        tool = self.registry.get_tool(tool_name)
        
        if not tool:
            raise ValueError(f"工具不存在：{tool_name}")
        
        try:
            # 1. 参数验证
            self.validate_parameters(tool, parameters)
            
            # 2. 执行前检查
            self.pre_execution_check(tool, parameters)
            
            # 3. 执行工具
            start_time = time.time()
            result = tool.func(**parameters)
            end_time = time.time()
            
            # 4. 结果验证
            self.validate_result(tool, result)
            
            # 5. 记录执行历史
            self.record_execution({
                "tool": tool_name,
                "parameters": parameters,
                "result": result,
                "duration": end_time - start_time,
                "status": "success"
            })
            
            return result
            
        except Exception as e:
            # 错误处理
            self.record_execution({
                "tool": tool_name,
                "parameters": parameters,
                "error": str(e),
                "status": "failed"
            })
            raise
    
    def validate_parameters(self, tool, parameters):
        """验证参数"""
        for param_name, param_def in tool.parameters.items():
            if param_def.get("required") and param_name not in parameters:
                raise ValueError(f"缺少必需参数：{param_name}")
            
            if param_name in parameters:
                value = parameters[param_name]
                expected_type = param_def.get("type")
                if not isinstance(value, eval(expected_type)):
                    raise ValueError(f"参数类型错误：{param_name}")
```

## 3. 自我反思与修正

### 3.1 反思的类型

#### 类型 1：输出反思（Output Reflection）

**目的**：评估生成内容的质量。

**反思维度**：
```
1. 准确性
   - 信息是否正确
   - 计算是否准确
   - 逻辑是否严密

2. 完整性
   - 是否覆盖所有要点
   - 是否遗漏重要信息
   - 是否回答用户问题

3. 一致性
   - 前后是否矛盾
   - 与已知事实是否一致
   - 格式是否统一

4. 相关性
   - 是否切题
   - 是否有冗余信息
   - 是否满足用户需求
```

### 3.3 反思循环

**完整流程**：
```python
class ReflectionLoop:
    def __init__(self, max_iterations=3):
        self.max_iterations = max_iterations
    
    def execute_with_reflection(self, task, executor):
        """带反思的执行循环"""
        
        iteration = 0
        current_output = None
        
        while iteration < self.max_iterations:
            # 1. 执行任务
            current_output = executor.execute(task)
            
            # 2. 反思输出
            reflection = reflect_on_output(task, current_output)
            
            # 3. 评估是否需要修正
            if reflection["accuracy"] >= 4 and reflection["completeness"] >= 4:
                # 质量达标，返回结果
                return current_output
            
            # 4. 选择修正策略
            if reflection["accuracy"] < 3:
                # 错误严重，重新生成
                current_output = regenerate(task, {}, [reflection])
            elif reflection["completeness"] < 3:
                # 不完整，增量改进
                current_output = incremental_improve(
                    task, current_output, reflection["suggestions"]
                )
            else:
                # 小问题，直接修正
                current_output = direct_correction(
                    task, current_output, reflection["issues"]
                )
            
            iteration += 1
        
        # 达到最大迭代次数，返回当前结果
        return current_output
```

## 4. 多 Agent 协作

### 4.1 协作架构

#### 架构 1：中心化协作（Centralized Collaboration）

**特点**：
```
- 一个主 Agent 负责协调
- 多个工作 Agent 执行具体任务
- 主 Agent 负责任务分配和结果整合
```

**架构图**：
```
        ┌─────────────┐
        │ 主 Agent    │
        │ (Coordinator)│
        └──────┬──────┘
               │
       ┌───────┼───────┐
       │       │       │
  ┌────▼───┐ ┌▼────┐ ┌─▼────┐
  │Agent A │ │Agent B│ │Agent C│
  └────────┘ └─────┘ └──────┘
```

**实现示例**：
```python
class CentralizedCollaboration:
    def __init__(self, coordinator, workers):
        self.coordinator = coordinator
        self.workers = workers
    
    def execute(self, task):
        # 1. 主 Agent 分解任务
        subtasks = self.coordinator.decompose(task)
        
        # 2. 分配子任务
        results = {}
        for subtask in subtasks:
            worker = self.select_worker(subtask)
            results[subtask.id] = worker.execute(subtask)
        
        # 3. 整合结果
        final_result = self.coordinator.integrate(results)
        
        return final_result
    
    def select_worker(self, subtask):
        """选择合适的 Worker"""
        # 基于能力匹配
        for worker in self.workers:
            if worker.can_handle(subtask):
                return worker
        raise ValueError("No suitable worker found")
```

#### 架构 2：对等协作（Peer-to-Peer Collaboration）

**特点**：
```
- Agent 之间地位平等
- 通过对话协商完成任务
- 没有固定的主从关系
```

**架构图**：
```
  ┌────────┐
  │Agent A │◄────┐
  └───┬────┘     │
      │          │
  ┌───▼────┐     │
  │Agent B │     │
  └───┬────┘     │
      │          │
  ┌───▼────┐     │
  │Agent C │─────┘
  └────────┘
（网状协作）
```

**实现示例**：
```python
class PeerCollaboration:
    def __init__(self, agents):
        self.agents = agents
        self.message_queue = []
    
    def execute(self, task):
        # 1. 发布任务
        self.broadcast(task)
        
        # 2. 多轮对话协商
        for round in range(MAX_ROUNDS):
            # 每个 Agent 可以发言
            for agent in self.agents:
                message = agent.respond(self.message_queue)
                if message:
                    self.message_queue.append(message)
                
                # 检查是否达成共识
                if self.reach_consensus():
                    break
            
            if self.reach_consensus():
                break
        
        # 3. 整合方案
        solution = self.integrate_solutions()
        return solution
```

#### 架构 3：流水线协作（Pipeline Collaboration）

**特点**：
```
- 任务按顺序流经多个 Agent
- 每个 Agent 负责一个处理阶段
- 输出作为下一个 Agent 的输入
```

**架构图**：
```
任务 → [Agent A] → [Agent B] → [Agent C] → 结果
       (处理 1)    (处理 2)    (处理 3)
```

**实现示例**：
```python
class PipelineCollaboration:
    def __init__(self, stages):
        """
        stages: [(Agent, description), ...]
        """
        self.stages = stages
    
    def execute(self, task):
        current_input = task
        
        for agent, description in self.stages:
            # 每个 Agent 处理并传递
            current_input = agent.process(current_input)
        
        return current_input
```

**使用示例**：
```python
# 内容创作流水线
pipeline = PipelineCollaboration([
    (ResearchAgent, "调研"),
    (OutlineAgent, "大纲"),
    (WriterAgent, "写作"),
    (EditorAgent, "编辑"),
    (ReviewerAgent, "审核")
])

result = pipeline.execute("写一篇关于 AI 的文章")
```

```python
def assign_task(task, agents):
    """任务分配"""
    
    # 1. 分析任务需求
    requirements = analyze_requirements(task)
    
    # 2. 评估 Agent 能力
    scores = []
    for agent in agents:
        score = evaluate_capability(agent, requirements)
        scores.append((agent, score))
    
    # 3. 选择最优 Agent
    scores.sort(key=lambda x: x[1], reverse=True)
    best_agent = scores[0][0]
    
    return best_agent
```

#### 机制 2：冲突解决（Conflict Resolution）

**冲突类型**：
```
1. 资源冲突：多个 Agent 竞争同一资源
2. 决策冲突：对同一问题有不同方案
3. 优先级冲突：任务优先级不一致
```

**解决方法**：
```python
class ConflictResolver:
    def resolve(self, conflict_type, agents, context):
        if conflict_type == "resource":
            return self.resolve_resource_conflict(agents, context)
        elif conflict_type == "decision":
            return self.resolve_decision_conflict(agents, context)
        elif conflict_type == "priority":
            return self.resolve_priority_conflict(agents, context)
    
    def resolve_decision_conflict(self, agents, context):
        """决策冲突解决"""
        
        # 1. 收集各方案
        proposals = [agent.proposal for agent in agents]
        
        # 2. 评估各方案
        evaluations = []
        for proposal in proposals:
            score = self.evaluate_proposal(proposal, context)
            evaluations.append((proposal, score))
        
        # 3. 选择最优方案
        evaluations.sort(key=lambda x: x[1], reverse=True)
        best_proposal = evaluations[0][0]
        
        return best_proposal
```
