import re

def clean_text(text):
    # 1. 去除多余空白
    text = re.sub(r'\s+', ' ', text)
    
    # 2. 去除特殊字符
    text = re.sub(r'[^\w\s\u4e00-\u9fff,.!?]', '', text)
    
    # 3. 去除页眉页脚（根据规律）
    text = re.sub(r'^第\d+页.*', '', text, flags=re.MULTILINE)
    
    # 4. 标准化标点
    text = text.replace('（', '(').replace('）', ')')
    
    # 5. 去除过短的行（可能是噪声）
    lines = text.split('\n')
    lines = [line for line in lines if len(line.strip()) > 10]
    
    return '\n'.join(lines)
```

## 2. 文档切片策略

### 2.1 切片的核心目标

**为什么要切片**：
```
1. 适配向量数据库
   - 向量检索适合短文本（500-1000 字）
   - 长文档检索效果差
   - 需要精确到片段级别

2. 提升检索精度
   - 用户问题通常针对具体知识点
   - 小片段更容易精确匹配
   - 避免大段无关信息干扰

3. 节省 token 成本
   - 只检索相关片段
   - 减少上下文长度
   - 降低 LLM 调用成本
```

**切片的权衡**：
```
切片太小：
✅ 精确匹配
✅ 节省 token
❌ 语义不完整
❌ 检索片段过多

切片太大：
✅ 语义完整
✅ 上下文充分
❌ 匹配不精确
❌ 包含噪声多

目标：找到最佳平衡点
```

---

### 2.2 固定长度切片

#### 方法 1：字符数切片

**实现逻辑**：
```python
def fixed_length_chunk(text, chunk_size=500, overlap=100):
    """固定字符数切片"""
    chunks = []
    start = 0
    
    while start < len(text):
        end = start + chunk_size
        chunk = text[start:end]
        chunks.append(chunk)
        start = end - overlap  # 重叠部分
    
    return chunks

# 使用示例
text = "..."  # 长文档
chunks = fixed_length_chunk(text, chunk_size=500, overlap=100)
```

**参数建议**：
```
中文文档：
- 小片段：300-500 字（精确检索）
- 中片段：500-800 字（平衡方案）
- 大片段：800-1500 字（语义完整）

英文文档：
- 小片段：200-400 tokens
- 中片段：400-600 tokens
- 大片段：600-1000 tokens

重叠度（overlap）：
- 建议：10-20%
- 作用：避免切断语义
- 示例：500 字切片，重叠 100 字
```

**优点**：
```
✅ 实现简单，易于理解
✅ 切片均匀，便于管理
✅ 计算高效，适合批量处理
✅ 适合结构松散的文档
```

**缺点**：
```
❌ 可能切断句子和段落
❌ 忽略语义边界
❌ 上下文可能不完整
❌ 需要后处理拼接
```

**适用场景**：
```
- 博客文章、新闻稿
- 论坛帖子、聊天记录
- 结构松散的文档
- 快速原型验证
```

#### 方法 2：段落感知切片

**核心思想**：
```
在固定长度基础上，尽量保持段落完整性：
1. 以段落为单位
2. 累积到目标大小
3. 在段落边界切分
```

**实现逻辑**：
```python
def paragraph_aware_chunk(text, target_size=500, overlap=100):
    """段落感知切片"""
    # 1. 按段落分割
    paragraphs = text.split('\n\n')
    
    chunks = []
    current_chunk = []
    current_size = 0
    
    for para in paragraphs:
        para_size = len(para)
        
        # 2. 如果加入后超出目标，且当前 chunk 不为空
        if current_size + para_size > target_size and current_chunk:
            # 保存当前 chunk
            chunks.append('\n\n'.join(current_chunk))
            # 保留重叠部分
            overlap_text = '\n\n'.join(current_chunk[-3:])  # 最后 3 段
            current_chunk = [overlap_text]
            current_size = len(overlap_text)
        
        # 3. 添加段落
        current_chunk.append(para)
        current_size += para_size
    
    # 4. 添加最后一个 chunk
    if current_chunk:
        chunks.append('\n\n'.join(current_chunk))
    
    return chunks
```

**优点**：
```
✅ 保持段落完整性
✅ 语义更连贯
✅ 阅读体验更好
✅ 实现复杂度适中
```

**缺点**：
```
❌ 段落过长时仍会切断
❌ 切片大小不均匀
❌ 需要识别段落边界
```

**适用场景**：
```
- 技术文档
- 制度文件
- 论文报告
- 大多数结构化文档（推荐作为默认方案）
```

### 2.3 语义切片

#### 方法 1：基于句子相似度

**核心思想**：
```
使用句子嵌入模型，在语义边界处切分：
1. 将文档分为句子
2. 计算相邻句子相似度
3. 在相似度最低处切分
```

**实现流程**：
```python
from sentence_transformers import SentenceTransformer
import numpy as np

def semantic_chunk(text, model, max_chunk_size=500):
    """基于语义相似度的切片"""
    # 1. 分句
    sentences = split_into_sentences(text)  # 自定义分句函数
    
    # 2. 计算句子嵌入
    embeddings = model.encode(sentences)
    
    # 3. 计算相邻句子相似度
    similarities = []
    for i in range(len(sentences) - 1):
        sim = cosine_similarity(embeddings[i], embeddings[i+1])
        similarities.append(sim)
    
    # 4. 在相似度低的地方切分
    chunks = []
    current_chunk = [sentences[0]]
    
    for i, sim in enumerate(similarities):
        # 如果相似度低于阈值，或达到最大长度
        if sim < 0.6 or len(' '.join(current_chunk)) > max_chunk_size:
            chunks.append(' '.join(current_chunk))
            current_chunk = []
        current_chunk.append(sentences[i+1])
    
    if current_chunk:
        chunks.append(' '.join(current_chunk))
    
    return chunks
```

**参数建议**：
```
相似度阈值：
- 0.5-0.6：较松，切片较大
- 0.6-0.7：适中（推荐）
- 0.7-0.8：较紧，切片较小

模型选择：
- 中文：bge-base-zh, m3e-base
- 英文：all-MiniLM-L6-v2
- 多语言：e5-base-v2
```

**优点**：
```
✅ 语义连贯性最好
✅ 自动识别主题边界
✅ 切片质量高
```

**缺点**：
```
❌ 计算成本高（需嵌入所有句子）
❌ 处理速度慢
❌ 依赖嵌入模型质量
❌ 阈值选择需要调优
```

**适用场景**：
```
- 高质量要求的场景
- 文档语义结构复杂
- 预算充足
- 离线批处理
```

#### 方法 2：基于层次化结构

**核心思想**：
```
利用文档的天然层次结构（章节 - 小节 - 段落）：
1. 识别标题层级
2. 按章节切分
3. 大章节再细分为小片段
```

**实现示例**：
```python
def hierarchical_chunk(text):
    """层次化切片"""
    chunks = []
    
    # 1. 识别标题（Markdown 示例）
    sections = re.split(r'^(#+)\s+(.+)$', text, flags=re.MULTILINE)
    
    current_section = None
    current_content = []
    
    for i in range(0, len(sections), 3):
        if i == 0:
            continue
        
        level = len(sections[i])  # 标题级别
        title = sections[i+1]
        content = sections[i+2]
        
        # 2. 新章节开始
        if current_section:
            # 保存上一章节
            section_text = f"{current_section}\n\n{''.join(current_content)}"
            
            # 3. 如果章节太长，继续细分
            if len(section_text) > 1000:
                sub_chunks = fixed_length_chunk(section_text, 500)
                chunks.extend(sub_chunks)
            else:
                chunks.append(section_text)
        
        current_section = f"{'#' * level} {title}"
        current_content = [content]
    
    return chunks
```

**优点**：
```
✅ 保持文档结构
✅ 语义完整
✅ 便于理解上下文
✅ 可添加结构元数据
```

**缺点**：
```
❌ 依赖文档格式规范
❌ 实现复杂度高
❌ 不适用于无结构文档
```

**适用场景**：
```
- Markdown 文档
- 结构化的 Word/PDF
- 技术文档、书籍
- 制度文件
```

### 2.5 切片优化技巧

#### 技巧 1：智能重叠

**目的**：避免上下文丢失

**实现**：
```python
def smart_overlap(chunks, overlap_ratio=0.1):
    """智能重叠"""
    optimized = []
    
    for i, chunk in enumerate(chunks):
        if i == 0:
            optimized.append(chunk)
        else:
            # 计算前一个 chunk 的末尾部分
            prev_chunk = optimized[-1]
            overlap_length = int(len(prev_chunk) * overlap_ratio)
            overlap_text = prev_chunk[-overlap_length:]
            
            # 添加到当前 chunk 开头
            new_chunk = overlap_text + '\n\n' + chunk
            optimized.append(new_chunk)
    
    return optimized
```

### 3.3 元数据提取

#### 方法 1：规则提取

**适用场景**：格式规范的文档

**示例：从文件名提取**：
```python
import re
from datetime import datetime

def extract_metadata_from_filename(filename):
    """从文件名提取元数据"""
    # 命名规范：部门_类型_标题_版本_日期.pdf
    # 示例：HR_policy_employee_handbook_v2024.1_20240115.pdf
    
    pattern = r'(\w+)_(\w+)_(.+?)_(v[\d.]+)_(\d{8})\.pdf'
    match = re.match(pattern, filename)
    
    if match:
        dept, doc_type, title, version, date_str = match.groups()
        
        return {
            "department": dept,
            "doc_type": doc_type,
            "title": title.replace('_', ' '),
            "version": version,
            "created_date": datetime.strptime(date_str, '%Y%m%d').isoformat(),
            "source": "filename"
        }
    
    return None
```

def weighted_search(query, top_k=10):
    """加权检索排序"""
    # 1. 基础向量检索
    results = vector_db.search(query, top_k=top_k * 2)
    
    # 2. 计算权重分数
    for result in results:
        score = result["score"]
        metadata = result["metadata"]
        
        # 版本权重（新版本优先）
        version_score = 1.0
        if "2024" in metadata.get("version", ""):
            version_score = 1.2
        
        # 类型权重（制度优先）
        type_score = 1.0
        if metadata.get("doc_type") == "policy":
            type_score = 1.3
        elif metadata.get("doc_type") == "procedure":
            type_score = 1.1
        
        # 部门权重（核心部门优先）
        dept_score = 1.0
        if metadata.get("department") in ["总经办", "HR", "财务"]:
            dept_score = 1.15
        
        # 3. 综合分数
        result["weighted_score"] = score * version_score * type_score * dept_score
    
    # 3. 重新排序
    results.sort(key=lambda x: x["weighted_score"], reverse=True)
    
    return results[:top_k]

### 4.2 实现示例

**完整流水线代码**：
```python
class RAGDataPipeline:
    def __init__(self, config):
        self.config = config
        self.embedding_model = SentenceTransformer(config["embedding_model"])
        self.vector_db = Chroma(config["vector_db_path"])
    
    def process_document(self, file_path):
        """处理单个文档"""
        try:
            # 1. 格式转换
            text = self.convert_to_text(file_path)
            
            # 2. 数据清洗
            text = self.clean_text(text)
            
            # 3. 质量检查
            if not self.quality_check(text):
                raise ValueError("质量检查不通过")
            
            # 4. 元数据提取
            metadata = self.extract_metadata(file_path, text)
            
            # 5. 文档切片
            chunks = self.chunk_document(text, metadata)
            
            # 6. 向量化
            embeddings = self.embed_chunks(chunks)
            
            # 7. 入库
            self.store_chunks(chunks, embeddings, metadata)
            
            return {
                "success": True,
                "chunks_count": len(chunks),
                "metadata": metadata
            }
        
        except Exception as e:
            return {
                "success": False,
                "error": str(e),
                "file": file_path
            }
    
    def convert_to_text(self, file_path):
        """格式转换"""
        ext = os.path.splitext(file_path)[1].lower()
        
        if ext == '.pdf':
            return self.extract_pdf_text(file_path)
        elif ext in ['.docx', '.doc']:
            return self.extract_word_text(file_path)
        elif ext in ['.html', '.htm']:
            return self.extract_html_text(file_path)
        else:
            with open(file_path, 'r', encoding='utf-8') as f:
                return f.read()
    
    def clean_text(self, text):
        """数据清洗"""
        # 去除多余空白
        text = re.sub(r'\s+', ' ', text)
        # 去除特殊字符
        text = re.sub(r'[^\w\s\u4e00-\u9fff,.!?]', '', text)
        # 标准化标点
        text = text.replace('（', '(').replace('）', ')')
        return text
    
    def quality_check(self, text):
        """质量检查"""
        if len(text) < 100:
            return False
        # 更多检查...
        return True
    
    def extract_metadata(self, file_path, text):
        """元数据提取"""
        # 混合提取：规则 + LLM
        metadata = self.extract_from_filename(file_path)
        metadata.update(self.extract_with_llm(text[:2000]))
        return metadata
    
    def chunk_document(self, text, metadata):
        """文档切片"""
        strategy = self.config.get("chunk_strategy", "paragraph_aware")
        
        if strategy == "fixed":
            chunks = fixed_length_chunk(text, 500, 100)
        elif strategy == "paragraph_aware":
            chunks = paragraph_aware_chunk(text, 500, 100)
        else:
            chunks = paragraph_aware_chunk(text, 500, 100)
        
        # 添加元数据到每个切片
        for i, chunk in enumerate(chunks):
            chunk_metadata = {
                **metadata,
                "chunk_id": f"{metadata['doc_id']}_{i:03d}",
                "chunk_index": i,
                "total_chunks": len(chunks)
            }
            chunk = {"content": chunk, "metadata": chunk_metadata}
        
        return chunks
    
    def embed_chunks(self, chunks):
        """向量化"""
        texts = [chunk["content"] for chunk in chunks]
        embeddings = self.embedding_model.encode(texts)
        return embeddings
    
    def store_chunks(self, chunks, embeddings, metadata):
        """入库存储"""
        for i, (chunk, embedding) in enumerate(zip(chunks, embeddings)):
            self.vector_db.add(
                id=chunk["metadata"]["chunk_id"],
                text=chunk["content"],
                embedding=embedding,
                metadata=chunk["metadata"]
            )
```

        futures = [executor.submit(pipeline.process_document, fp) for fp in file_paths]
        
        for future in as_completed(futures):
            result = future.result()
            results.append(result)
            
            if result["success"]:
                print(f"✅ {result['metadata']['title']} - {result['chunks_count']} 切片")
            else:
                print(f"❌ {result['file']} - {result['error']}")
    
    return results
```

#### 优化 2：断点续传

```python
import pickle

class CheckpointManager:
    def __init__(self, checkpoint_path):
        self.checkpoint_path = checkpoint_path
        self.processed = set()
        self.load_checkpoint()
    
    def load_checkpoint(self):
        """加载检查点"""
        if os.path.exists(self.checkpoint_path):
            with open(self.checkpoint_path, 'rb') as f:
                self.processed = pickle.load(f)
    
    def save_checkpoint(self):
        """保存检查点"""
        with open(self.checkpoint_path, 'wb') as f:
            pickle.dump(self.processed, f)
    
    def is_processed(self, file_path):
        """检查是否已处理"""
        return file_path in self.processed
    
    def mark_processed(self, file_path):
        """标记为已处理"""
        self.processed.add(file_path)
        self.save_checkpoint()

# 使用示例
checkpoint = CheckpointManager("checkpoint.pkl")

for file_path in file_paths:
    if checkpoint.is_processed(file_path):
        print(f"跳过已处理：{file_path}")
        continue
    
    result = pipeline.process_document(file_path)
    
    if result["success"]:
        checkpoint.mark_processed(file_path)

**Q1: 为什么 RAG 系统中数据处理这么重要？**

**参考答案**：
```
核心原因：Garbage In, Garbage Out（垃圾进，垃圾出）

数据质量决定上限：
1. 检索质量依赖切片质量
   - 切片语义不完整 → 检索结果不相关
   - 切片包含噪声 → 干扰 LLM 判断
   - 切片大小不合理 → 匹配精度下降

   - 有元数据 → 可过滤、加权、权限控制
   - 元数据质量 → 决定过滤准确性

3. 清洗质量影响最终效果
   - 乱码和噪声 → 向量化失真
   - 格式混乱 → 切片边界错误
   - 内容缺失 → 回答不完整

实际影响对比：
```
优质数据 + 简单算法：70-80 分
劣质数据 + 复杂算法：50-60 分
```

最佳实践：
- 投入 60% 精力在数据处理
- 投入 30% 精力在检索优化
- 投入 10% 精力在生成优化
```

**Q5: 如何设计文档切片的Overlap（重叠）策略？**

**参考答案**：
```
为什么需要 Overlap：

问题：切片可能切断语义
示例：
```
切片 1："...员工累计工作已满 1 年不满"
切片 2："10 年的，年休假 5 天..."
```
→ 两个切片都无法完整理解

解决：添加重叠部分

Overlap 大小选择：

小重叠（5-10%）：
✅ 节省存储空间
✅ 减少冗余
❌ 可能仍会切断语义

中重叠（10-20%，推荐）：
✅ 平衡冗余和完整性
✅ 大多数场景适用
❌ 少量语义丢失

大重叠（20-30%）：
✅ 语义完整性最好
✅ 几乎不会切断
❌ 存储冗余大
❌ 检索结果重复

推荐方案：

1. 按文档类型
   - 制度文档：15%（条款独立）
   - 技术文档：20%（概念连贯）
   - 论文报告：20%（论证连续）
   - FAQ 问答：10%（问题独立）

2. 按切片大小
   - 小切片（300 字）：15%（45 字重叠）
   - 中切片（500 字）：20%（100 字重叠）
   - 大切片（1000 字）：15%（150 字重叠）

3. 智能 Overlap
   基于句子边界：
   ```
   不是机械地在固定位置切断
   而是在句子/段落边界处切分
   然后添加前后各 1-2 个句子
   ```

实现示例：
```python
def smart_overlap(text, chunk_size=500, overlap_ratio=0.2):
    overlap_size = int(chunk_size * overlap_ratio)
    
    chunks = []
    start = 0
    
    while start < len(text):
        end = start + chunk_size
        
        # 在句子边界处切分
        chunk = text[start:end]
        last_period = chunk.rfind('。')
        
        if last_period > chunk_size * 0.8:
            chunk = chunk[:last_period+1]
        
        chunks.append(chunk)
        start = end - overlap_size
    
    return chunks
```

验证方法：
1. 抽样检查切片边界
2. 看是否有句子被切断
3. 调整 overlap_ratio 直到满意
```

       adapter = next(a for a in adapters if a.supports(file_type))
       text = adapter.extract_text(file_path)
   ```

3. 分级处理策略
   ```
   一级（优质）：文本型 PDF、Word
   - 直接提取文字
   - 保留结构
   - 高质量
   
   二级（中等）：扫描版 PDF、图片
   - OCR 识别
   - 版面分析
   - 中等质量
   
   三级（较差）：手写、模糊图片
   - 高级 OCR
   - 人工辅助
   - 低质量或放弃
   ```

4. 元数据标准化
   ```
   不同来源 → 统一元数据 Schema
   
   文件系统：文件名、路径、修改时间
   数据库：  表名、记录 ID、更新时间
   API：     接口名、参数、调用时间
   
   ↓ 映射为统一格式
   
   {
     "source_type": "file|database|api",
     "source_id": "...",
     "title": "...",
     "created_date": "...",
     "modified_date": "..."
   }
   ```

5. 质量分级
   ```python
   def assess_quality(text):
       score = 0
       
       # 长度检查
       if len(text) > 500:
           score += 30
       
       # 乱码检查
       if special_char_ratio(text) < 0.05:
           score += 30
       
       # 结构检查
       if has_structure(text):
           score += 20
       
       # 语言检查
       if detect_language(text) == 'zh':
           score += 20
       
       return score  # 0-100
   
   # 根据分数分级处理
   score = assess_quality(text)
   if score >= 80:
       process_normal(text)  # 正常处理
   elif score >= 60:
       process_with_caution(text)  # 谨慎处理
   else:
       mark_for_review(text)  # 人工审核
   ```

实战建议：
- 优先处理高质量文档（80% 价值）
- 中等质量文档批量处理
- 低质量文档人工审核或放弃
- 持续优化解析器
```

**Q11: 数据处理中遇到过哪些坑？**

**参考答案**：
```
常见陷阱：

1. PDF 解析坑
   问题：
   - 扫描版当文本版处理（提取空白）
   - 多栏排版顺序错乱
   - 表格提取成纯文本（丢失结构）
   
   解决：
   - 先检测 PDF 类型（文本/扫描）
   - 版面分析后再提取
   - 表格专用解析器

2. 编码问题
   问题：
   - 文件编码不一致（UTF-8、GBK）
   - 特殊字符乱码
   - Emoji 处理不当
   
   解决：
   - 自动检测编码（chardet 库）
   - 统一转 UTF-8
   - 过滤或转义特殊字符

3. 切片切断语义
   问题：
   - 句子被切成两半
   - "如下所示"但内容在下一片
   - 代词"它"指代不明
   
   解决：
   - 段落感知切片
   - 增加重叠度（20%）
   - 父子索引

4. 元数据缺失
   问题：
   - 文档无标题
   - 部门信息缺失
   - 日期格式混乱
   
   解决：
   - LLM 补充提取
   - 建立默认值
   - 标准化日期格式

5. 重复文档
   问题：
   - 同一文档多个版本
   - 不同格式重复（PDF+Word）
   - 跨部门文档重复
   
   解决：
   - 文档去重（SimHash）
   - 版本管理
   - 建立文档关联

6. 隐私泄露
   问题：
   - 敏感信息未脱敏
   - 个人隐私泄露
   - 商业机密暴露
   
   解决：
   - 敏感信息识别（正则）
   - 自动脱敏
   - 权限控制

7. 过度清洗
   问题：
   - 把重要内容当噪声删除
   - 过度标准化丢失语义
   - 表格公式被清理
   
   解决：
   - 保守清洗策略
   - 白名单机制
   - 人工抽检

8. 性能瓶颈
   问题：
   - 大批量处理太慢
   - 向量化耗时
   - 内存溢出
   
   解决：
   - 并行处理
   - 批量向量化
   - 流式处理

9. 断点续传缺失
   问题：
   - 处理中断从头开始
   - 重复处理浪费资源
   - 无法追踪进度
   
   解决：
   - 检查点机制
   - 进度持久化
   - 失败重试

10. 质量监控缺失
    问题：
    - 处理质量无法量化
    - 问题发现滞后
    - 无法追踪趋势
    
    解决：
    - 建立质量指标
    - 自动化监控
    - 定期报告

实战建议：
- 上线前充分测试各种边界情况
- 建立完善的监控和告警
- 保留原始文档和处理日志
- 定期审查和优化流程
- 文档化所有规则和决策
```
