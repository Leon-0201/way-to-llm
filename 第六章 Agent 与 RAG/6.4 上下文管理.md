# 6.4 上下文管理

> **核心理解**：上下文管理是 Agent 的"记忆中枢"，决定了模型能记住多少信息、如何组织信息。优秀的上下文管理策略能在有限的上下文窗口内，最大化信息利用率，提升模型表现。

---

## 1. 三层记忆机制

### 1.1 为什么需要分层记忆

**问题背景**：
```
1. 上下文窗口限制
   - GPT-4: 128K tokens
   - Claude: 200K tokens
   - 但实际应用中仍可能不够用

2. 成本考虑
   - 长上下文 = 更高的 token 成本
   - 更长的响应延迟

3. 信息过载
   - 过多信息会干扰模型判断
   - "注意力稀释"问题

4. 信息类型不同
   - 临时对话 vs 持久事实
   - 通用知识 vs 个性化特征
   - 需要不同的管理策略
```

**三层记忆架构**：
```
┌─────────────────────────────────────┐
│     短期记忆 (Short-term Memory)     │
│  - 当前对话上下文                    │
│  - 快速访问、易变                    │
│  - 容量有限                          │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│     长期记忆 (Long-term Memory)      │
│  - 持久化存储的事实和经历            │
│  - 需要时检索                        │
│  - 容量理论上无限                    │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│   Profile 记忆 (Profile Memory)      │
│  - Agent 性格设定                    │
│  - 用户特征画像                      │
│  - 相对稳定、不常变更                │
└─────────────────────────────────────┘
```

---

## 2. 短期记忆（Short-term Memory）

### 2.1 定义与特点

**定义**：当前对话窗口内的信息，直接放入 prompt 中。

**特点**：
```
- 访问速度：极快（直接在上下文中）
- 容量：受上下文窗口限制（通常 4K-8K tokens）
- 持久性：会话结束即消失
- 成本：每次调用都需要携带
- 更新频率：每轮对话都更新
```

**适用场景**：
```
- 当前对话的轮次记录
- 临时的任务状态
- 即时的用户反馈
- 短期内的指代消解
```

---

### 2.2 实现策略

#### 2.2.1 策略 1：滑动窗口（Sliding Window）

**原理**：保留最近的 N 轮对话，超出则丢弃最早的消息。

```python
class SlidingWindowMemory:
    def __init__(self, max_tokens=4000):
        self.max_tokens = max_tokens
        self.messages = []
    
    def add_message(self, role: str, content: str):
        """添加消息"""
        self.messages.append({"role": role, "content": content})
        self._trim_if_needed()
    
    def _trim_if_needed(self):
        """修剪超出限制的消息"""
        while self._count_tokens() > self.max_tokens:
            if len(self.messages) > 2:  # 保留 system message
                self.messages.pop(1)  # 移除最早的用户消息
            else:
                break
    
    def get_messages(self) -> list:
        """获取对话历史"""
        return self.messages
    
    def _count_tokens(self) -> int:
        """估算 token 数"""
        total = 0
        for msg in self.messages:
            total += len(msg["content"]) // 4  # 粗略估算：4 字符≈1 token
        return total

# 使用示例
memory = SlidingWindowMemory(max_tokens=4000)
memory.add_message("system", "你是一个助手")
memory.add_message("user", "你好")
memory.add_message("assistant", "你好！有什么可以帮你的吗？")

# 获取对话历史
messages = memory.get_messages()
```

**优点**：
- ✅ 实现简单
- ✅ 保证最近的对话始终可用
- ✅ 性能稳定

**缺点**：
- ❌ 可能丢失重要但较早的信息
- ❌ 不区分信息重要性
- ❌ "一刀切"的丢弃策略

#### 2.2.2 策略 2：重要性评分（Importance-based）

**原理**：为每条消息评分，保留重要性高的消息。

```python
import time
from typing import List, Dict
from openai import OpenAI

class ImportanceMemory:
    def __init__(self, max_messages=50, llm=None):
        self.max_messages = max_messages
        self.messages = []
        self.llm = llm or OpenAI()
    
    def add_message(self, role: str, content: str, importance: float = None):
        """添加消息"""
        if importance is None:
            # 使用 LLM 评估重要性
            importance = self._evaluate_importance(content)
        
        self.messages.append({
            "role": role,
            "content": content,
            "importance": importance,
            "timestamp": time.time()
        })
        
        self._prune_if_needed()
    
    def _evaluate_importance(self, content: str) -> float:
        """评估信息重要性（1-10 分）"""
        prompt = f"""
        评估以下信息的重要性（1-10 分）：
        "{content}"
        
        评分标准：
        - 10: 关键信息（如用户偏好、重要事实、任务目标）
        - 7-9: 重要信息（如具体需求、决策）
        - 4-6: 一般信息（如普通问答）
        - 1-3: 闲聊（如问候、寒暄）
        
        分数：
        """
        
        response = self.llm.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": prompt}],
            max_tokens=10
        )
        
        try:
            score = float(response.choices[0].message.content.strip())
            return min(max(score, 1), 10)  # 限制在 1-10 范围
        except:
            return 5.0  # 默认中等重要性
    
    def _prune_if_needed(self):
        """修剪低重要性消息"""
        if len(self.messages) <= self.max_messages:
            return
        
        # 按重要性排序，保留最重要的
        self.messages.sort(key=lambda x: x["importance"], reverse=True)
        self.messages = self.messages[:self.max_messages]
    
    def get_messages(self) -> list:
        """按时间顺序返回消息"""
        self.messages.sort(key=lambda x: x["timestamp"])
        return self.messages

# 使用示例
memory = ImportanceMemory(max_messages=50)

# 添加消息（自动评估重要性）
memory.add_message("user", "你好")  # 闲聊，低重要性
memory.add_message("user", "记住我喜欢喝拿铁咖啡")  # 偏好，高重要性
memory.add_message("user", "帮我订明天上午 10 点的会议")  # 任务，高重要性

# 获取时会发现闲聊被优先丢弃
messages = memory.get_messages()
```

**优点**：
- ✅ 智能保留重要信息
- ✅ 适应不同对话场景
- ✅ 更好的长期对话质量

**缺点**：
- ❌ 需要额外的 LLM 调用（成本）
- ❌ 实现复杂度较高
- ❌ 重要性评估可能不准确

#### 2.2.3 策略 3：混合策略（推荐）

**原理**：结合滑动窗口和重要性评分，平衡性能和成本。

```python
class HybridMemory:
    def __init__(self, max_tokens=4000, critical_slots=10):
        self.max_tokens = max_tokens
        self.critical_slots = critical_slots  # 保留最重要的 N 条
        self.messages = []
        self.critical_messages = []  # 关键消息单独存储
    
    def add_message(self, role: str, content: str, is_critical: bool = False):
        """添加消息"""
        message = {
            "role": role,
            "content": content,
            "timestamp": time.time(),
            "is_critical": is_critical
        }
        
        if is_critical:
            # 关键消息永久保留
            self.critical_messages.append(message)
        else:
            # 普通消息进入滑动窗口
            self.messages.append(message)
            self._trim_if_needed()
    
    def _trim_if_needed(self):
        """修剪普通消息"""
        while self._count_tokens() > self.max_tokens and len(self.messages) > 0:
            self.messages.pop(0)  # 移除最早的普通消息
    
    def get_all_messages(self) -> list:
        """获取所有消息（关键消息 + 普通消息）"""
        # 关键消息始终在前
        return self.critical_messages + self.messages
    
    def _count_tokens(self) -> int:
        total = 0
        for msg in self.messages:
            total += len(msg["content"]) // 4
        return total

# 使用示例
memory = HybridMemory(max_tokens=4000, critical_slots=10)

# 普通对话
memory.add_message("user", "今天天气不错")
memory.add_message("assistant", "是的，适合出门")

# 关键信息（永久保留）
memory.add_message(
    "user", 
    "我对花生过敏",
    is_critical=True  # 标记为关键
)

memory.add_message(
    "user",
    "我的目标是学习 Python 编程",
    is_critical=True
)

# 获取时会发现关键消息始终存在
all_messages = memory.get_all_messages()
```

**优点**：
- ✅ 平衡性能和成本
- ✅ 关键信息不丢失
- ✅ 实现相对简单

**缺点**：
- ❌ 需要手动标记关键信息（或自动识别）

#### 2.3.2 技巧 2：对话摘要注入

```python
class SummarizingMemory:
    def __init__(self, llm, summary_interval=10):
        self.llm = llm
        self.summary_interval = summary_interval  # 每 10 轮生成一次摘要
        self.messages = []
        self.summary = ""
        self.round_count = 0
    
    def add_message(self, role: str, content: str):
        self.messages.append({"role": role, "content": content})
        self.round_count += 1
        
        # 定期生成摘要
        if self.round_count % self.summary_interval == 0:
            self.summary = self._generate_summary()
            # 清空历史，只保留摘要
            self.messages = []
    
    def _generate_summary(self) -> str:
        """生成对话摘要"""
        prompt = f"""
        总结以下对话的关键信息：
        {self._format_messages()}
        
        摘要（包含：用户问题、已解决方案、待办事项）：
        """
        
        response = self.llm.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": prompt}],
            max_tokens=200
        )
        
        return response.choices[0].message.content
    
    def get_context(self) -> str:
        """获取完整上下文（摘要 + 最近对话）"""
        context = ""
        
        if self.summary:
            context += f"【对话摘要】\n{self.summary}\n\n"
        
        if self.messages:
            context += f"【最近对话】\n{self._format_messages()}"
        
        return context
    
    def _format_messages(self) -> str:
        formatted = []
        for msg in self.messages[-5:]:  # 只保留最近 5 轮
            role = "用户" if msg["role"] == "user" else "助手"
            formatted.append(f"{role}: {msg['content']}")
        return "\n".join(formatted)
```

## 4. 长期记忆（Long-term Memory）

### 4.1 定义与特点

**定义**：持久化存储的事实、经历、知识等信息，需要时检索使用。

**特点**：
```
- 访问速度：较慢（需要检索）
- 容量：理论上无限
- 持久性：永久保存
- 成本：存储成本低，检索有成本
- 更新频率：中等（按需添加/更新）
```

**存储内容**：
```
1. 事实性知识
   - 用户明确告知的信息
   - 对话中确认的事实
   - 领域知识

2. 经历记忆
   - 过去的对话主题
   - 已完成的任务
   - 解决的问题

3. 程序性知识
   - 用户的操作习惯
   - 常用工作流程
   - 偏好工具
```

---

### 4.2 基于向量数据库的实现

#### 4.2.1 完整示例

```python
import chromadb
from chromadb.config import Settings
from openai import OpenAI
import time
import hashlib
from typing import List, Dict, Optional

class LongTermMemory:
    def __init__(self, collection_name="memory", persist_directory="./memory_db"):
        # 初始化 ChromaDB
        self.client = chromadb.Client(Settings(
            chroma_db_impl="duckdb+parquet",
            persist_directory=persist_directory
        ))
        
        self.collection = self.client.get_or_create_collection(
            name=collection_name,
            metadata={"hnsw:space": "cosine"}
        )
        
        self.openai = OpenAI()
    
    def add_memory(
        self, 
        content: str, 
        memory_type: str = "fact",
        user_id: str = None,
        metadata: dict = None
    ) -> str:
        """添加记忆"""
        # 生成嵌入向量
        embedding = self.openai.embeddings.create(
            input=content,
            model="text-embedding-3-small"
        ).data[0].embedding
        
        # 生成唯一 ID
        memory_id = self._generate_id(content)
        
        # 准备元数据
        full_metadata = {
            "content": content,
            "type": memory_type,
            "user_id": user_id,
            "created_at": time.time(),
            "access_count": 0,
            **(metadata or {})
        }
        
        # 存储到向量数据库
        self.collection.add(
            ids=[memory_id],
            embeddings=[embedding],
            documents=[content],
            metadatas=[full_metadata]
        )
        
        return memory_id
    
    def search_memories(
        self, 
        query: str, 
        top_k: int = 5,
        user_id: str = None,
        memory_type: str = None
    ) -> List[Dict]:
        """检索相关记忆"""
        # 生成查询向量
        query_embedding = self.openai.embeddings.create(
            input=query,
            model="text-embedding-3-small"
        ).data[0].embedding
        
        # 构建过滤条件
        where_filter = {}
        if user_id:
            where_filter["user_id"] = user_id
        if memory_type:
            where_filter["type"] = memory_type
        
        # 检索
        results = self.collection.query(
            query_embeddings=[query_embedding],
            n_results=top_k,
            where=where_filter if where_filter else None,
            include=["documents", "metadatas", "distances"]
        )
        
        # 格式化结果
        memories = []
        for i, doc in enumerate(results["documents"][0]):
            memories.append({
                "id": results["ids"][0][i],
                "content": doc,
                "metadata": results["metadatas"][0][i],
                "distance": results["distances"][0][i]
            })
        
        # 更新访问计数
        for mem in memories:
            self._update_access_count(mem["id"])
        
        return memories
    
    def _generate_id(self, content: str) -> str:
        """生成唯一 ID"""
        return f"mem_{hashlib.md5(content.encode()).hexdigest()}"
    
    def _update_access_count(self, memory_id: str):
        """更新访问计数"""
        # ChromaDB 不支持部分更新，需要删除后重新添加
        # 实际应用中可以使用支持更新的数据库
        pass
    
    def delete_memory(self, memory_id: str):
        """删除记忆"""
        self.collection.delete(ids=[memory_id])
    
    def get_user_profile(self, user_id: str) -> dict:
        """获取用户画像（从长期记忆中提取）"""
        memories = self.search_memories(f"用户 {user_id} 的信息", top_k=20)
        
        profile = {
            "preferences": [],
            "facts": [],
            "history": []
        }
        
        for mem in memories:
            mem_type = mem["metadata"].get("type", "fact")
            if mem_type == "preference":
                profile["preferences"].append(mem["content"])
            elif mem_type == "fact":
                profile["facts"].append(mem["content"])
            else:
                profile["history"].append(mem["content"])
        
        return profile

# 使用示例

# 添加记忆

    "用户喜欢喝拿铁咖啡，不喜欢加糖",
    memory_type="preference",
    user_id="user_123"
)

memory.add_memory(
    "用户在北京工作，从事软件工程师职业",
    memory_type="fact",
    user_id="user_123"
)

memory.add_memory(
    "用户正在学习 Python，已经掌握了基础语法",
    memory_type="progress",
    user_id="user_123"
)

# 检索记忆

)
```

### 4.3 记忆更新与遗忘

#### 4.3.1 记忆更新策略

```python
class MemoryManager:
    def __init__(self, long_term_memory: LongTermMemory):
        self.memory = long_term_memory
        self.llm = OpenAI()
    
    def update_memory(self, new_info: str, user_id: str, memory_type: str = "fact"):
        """更新记忆（检查是否重复，决定新增或更新）"""
        # 1. 检索相似记忆
        similar = self.memory.search_memories(
            new_info,
            top_k=1,
            user_id=user_id,
            memory_type=memory_type
        )
        
        # 2. 判断是否需要更新
        if similar and similar[0]["distance"] < 0.1:  # 非常相似
            # 使用 LLM 判断是否需要更新
            should_update = self._should_update(
                similar[0]["content"],
                new_info
            )
            
            if should_update:
                # 删除旧记忆，添加新记忆
                self.memory.delete_memory(similar[0]["id"])
                self.memory.add_memory(new_info, memory_type, user_id)
                return f"已更新记忆：{new_info}"
            else:
                return "信息已存在，无需更新"
        else:
            # 添加新记忆
            self.memory.add_memory(new_info, memory_type, user_id)
            return f"已添加新记忆：{new_info}"
    
    def _should_update(self, old_info: str, new_info: str) -> bool:
        """判断是否需要更新"""
        prompt = f"""
        判断以下两条信息是否需要更新（新信息是否提供了更准确或更新的内容）：
        
        旧信息：{old_info}
        新信息：{new_info}
        
        如果需要更新，回复"是"；否则回复"否"。
        """
        
        response = self.llm.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": prompt}]
        )
        
        return "是" in response.choices[0].message.content

# 使用示例
manager = MemoryManager(LongTermMemory())

    "用户喜欢喝咖啡",

    memory_type="preference"
))
# 输出：已添加新记忆：用户喜欢喝咖啡

# 更新信息
print(manager.update_memory(

    memory_type="preference"
))
```

#### 4.3.2 记忆遗忘策略

```python
class ForgettingMemory:
    def __init__(self, memory: LongTermMemory, half_life_days: int = 30):
        self.memory = memory
        self.half_life = half_life_days * 24 * 3600  # 转换为秒
    
    def calculate_recall_probability(self, created_at: float, access_count: int) -> float:
        """
        计算记忆被召回的概率
        基于艾宾浩斯遗忘曲线 + 访问频率
        """
        elapsed = time.time() - created_at
        
        # 基础遗忘曲线（指数衰减）
        base_decay = math.exp(-elapsed / self.half_life)
        
        # 访问频率增强（访问越多，遗忘越慢）
        frequency_boost = min(1.0, access_count * 0.1)  # 最多提升 100%
        
        # 综合概率
        probability = base_decay * (1 + frequency_boost)
        
        return min(probability, 1.0)
    
    def should_forget(self, memory: dict) -> bool:
        """判断是否应该遗忘某条记忆"""
        created_at = memory["metadata"].get("created_at", time.time())
        access_count = memory["metadata"].get("access_count", 0)
        
        recall_prob = self.calculate_recall_probability(created_at, access_count)
        
        # 考虑记忆类型
        memory_type = memory["metadata"].get("type", "fact")
        type_weight = {
            "preference": 0.8,  # 偏好不容易遗忘
            "fact": 0.5,        # 事实中等
            "progress": 0.3,    # 进展容易过时
            "history": 0.2      # 历史记录容易遗忘
        }.get(memory_type, 0.5)
        
        # 综合评分
        score = recall_prob * type_weight
        
        # 低于阈值则遗忘
        return score < 0.1
    
    def cleanup(self, user_id: str = None):
        """清理过期记忆"""
        # 获取所有记忆
        all_memories = self.memory.search_memories(
            "",
            top_k=10000,
            user_id=user_id
        )
        
        # 删除应该遗忘的记忆
        for mem in all_memories:
            if self.should_forget(mem):
                self.memory.delete_memory(mem["id"])
                print(f"已遗忘：{mem['content']}")

# 使用示例
forgetting_memory = ForgettingMemory(LongTermMemory(), half_life_days=30)

# 定期清理（如每周执行一次）
forgetting_memory.cleanup(user_id="user_123")
```

```python
class ConversationalAgent:
    def __init__(self):
        self.short_term = SlidingWindowMemory()
        self.long_term = LongTermMemory()
        self.llm = OpenAI()
    
    def chat(self, user_message: str, user_id: str) -> str:
        # 1. 检索相关长期记忆
        relevant_memories = self.long_term.search_memories(
            user_message,
            top_k=5,
            user_id=user_id
        )
        
        # 2. 构建上下文
        context = self._build_context(user_message, relevant_memories)
        
        # 3. 调用 LLM
        response = self.llm.chat.completions.create(
            model="gpt-4o",
            messages=context
        )
        
        # 4. 更新短期记忆
        self.short_term.add_message("user", user_message)
        self.short_term.add_message("assistant", response.choices[0].message.content)
        
        # 5. 提取新记忆（可选）
        self._extract_and_store_memories(user_message, response, user_id)
        
        return response.choices[0].message.content
    
    def _build_context(self, user_message: str, memories: list) -> list:
        """构建对话上下文"""
        context = []
        
        # 系统提示词
        context.append({"role": "system", "content": "你是一个智能助手"})
        
        # 相关记忆
        if memories:
            memory_text = "\n".join([f"- {m['content']}" for m in memories])
            context.append({
                "role": "system",
                "content": f"相关记忆：\n{memory_text}"
            })
        
        # 对话历史
        context.extend(self.short_term.get_messages())
        
        # 当前问题
        context.append({"role": "user", "content": user_message})
        
        return context
    
    def _extract_and_store_memories(self, user_msg, response, user_id: str):
        """从对话中提取并存储新记忆"""
        # 使用 LLM 提取
        prompt = f"""
        从以下对话中提取需要记住的信息（如果有）：
        用户：{user_msg}
        助手：{response.choices[0].message.content}
        
        需要记住的信息（JSON 数组，每项包含 content 和 type）：
        """
        
        extraction = self.llm.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": prompt}],
            response_format={"type": "json_object"}
        )
        
        data = json.loads(extraction.choices[0].message.content)
        
        for item in data.get("memories", []):
            self.long_term.add_memory(
                item["content"],
                memory_type=item.get("type", "fact"),
                user_id=user_id
            )
```

## 4. Profile 记忆（Profile Memory）

### 4.1 定义与特点

**定义**：存储 Agent 的性格设定、用户特征等相对稳定的信息。

**特点**：
```
- 访问速度：快（通常预加载到上下文）
- 容量：中等（通常 1K-3K tokens）
- 持久性：长期保存，手动更新
- 成本：一次性加载，复用多次
- 更新频率：低（按需更新）
```

**包含内容**：
```
1. Agent 侧（Agent Profile）
   - 性格设定（活泼、严肃、幽默等）
   - 角色定位（助手、导师、朋友等）
   - 行为准则（不做的事情、必须做的事情）
   - 语言风格（正式、随意、专业等）
   - 知识边界（专业领域、不擅长的领域）

2. 用户侧（User Profile）
   - 基本信息（姓名、职业、地区等）
   - 偏好（回答风格、内容深度等）
   - 目标（学习目标、工作目标等）
   - 禁忌（不喜欢的话题、敏感点等）
   - 历史互动模式
```

```python
        """更新偏好"""
        if '偏好设置' not in self.profile:
            self.profile['偏好设置'] = {}

        """记录互动"""
        if '历史互动统计' not in self.profile:
            self.profile['历史互动统计'] = {
                '总对话轮次': 0,

                '平均满意度': 0

            old_avg = stats['平均满意度']
            stats['平均满意度'] = (old_avg * count + rating) / (count + 1)
            stats['评分次数'] = count
```

```python
class ProfileManager:
    def __init__(self):
        self.agent_profile = None
        self.user_profiles = {}
    
    def load_agent_profile(self, path: str):
        """加载 Agent Profile（全局唯一）"""

    def load_user_profile(self, user_id: str, path: str = None):

        self.user_profiles[user_id] = UserProfile(user_id, path)

        if user_id and user_id in self.user_profiles:
            parts.append(self.user_profiles[user_id].to_prompt())
        
        return "\n\n".join(parts)
```

## 5. 三层记忆的协同工作

### 5.1 完整架构

#### 5.1.1 完整实现

```python
class ThreeLayerMemory:
    def __init__(self, config: dict = None):
        config = config or {}
        
        # 1. 短期记忆
        self.short_term = SlidingWindowMemory(
            max_tokens=config.get("short_term_max_tokens", 4000)
        )
        
        # 2. Profile 记忆
        self.profile_manager = ProfileManager()
        if config.get("agent_profile_path"):
            self.profile_manager.load_agent_profile(config["agent_profile_path"])
        
        # 3. 长期记忆
        self.long_term = LongTermMemory(
            collection_name=config.get("collection_name", "memory"),
            persist_directory=config.get("persist_directory", "./memory_db")
        )
        
        self.llm = OpenAI()
    
    def chat(self, user_message: str, user_id: str = None) -> str:
        """完整的对话流程"""
        # 1. 从 Profile 获取系统提示词
        system_prompt = self.profile_manager.get_system_prompt(user_id)
        
        # 2. 检索相关长期记忆
        relevant_memories = self.long_term.search_memories(
            user_message,
            top_k=5,
            user_id=user_id
        )
        
        # 3. 构建完整上下文
        context = self._build_full_context(
            system_prompt,
            user_message,
            relevant_memories
        )
        
        # 4. 调用 LLM
        response = self.llm.chat.completions.create(
            model="gpt-4o",
            messages=context
        )
        
        assistant_message = response.choices[0].message.content
        
        # 5. 更新短期记忆
        self.short_term.add_message("user", user_message)
        self.short_term.add_message("assistant", assistant_message)
        
        # 6. 提取并存储新记忆
        self._extract_and_store_memories(user_message, assistant_message, user_id)
        
        return assistant_message
    
    def _build_full_context(
        self,
        system_prompt: str,
        user_message: str,
        relevant_memories: list
    ) -> list:
        """构建完整上下文"""
        context = []
        
        # 1. 系统提示词（包含 Profile）
        if system_prompt:
            context.append({"role": "system", "content": system_prompt})
        
        # 2. 相关长期记忆
        if relevant_memories:
            memory_text = "\n".join([
                f"- {m['content']} (类型：{m['metadata']['type']})"
                for m in relevant_memories
            ])
            context.append({
                "role": "system",
                "content": f"【相关记忆】\n{memory_text}"
            })
        
        # 3. 短期记忆（对话历史）
        context.extend(self.short_term.get_messages())
        
        # 4. 当前用户消息
        context.append({"role": "user", "content": user_message})
        
        return context
    
    def _extract_and_store_memories(
        self,
        user_msg: str,
        assistant_msg: str,
        user_id: str
    ):
        """提取并存储新记忆"""
        prompt = f"""
        从以下对话中提取需要长期记住的信息：
        
        用户：{user_msg}
        助手：{assistant_msg}
        
        提取规则：
        1. 只提取事实性信息（偏好、事实、目标等）
        2. 不要提取临时性的对话内容
        3. 如果不需要记住任何内容，返回空数组
        
        返回 JSON 格式：
        {{
          "memories": [
            {{"content": "信息内容", "type": "preference|fact|progress"}},
            ...
          ]
        }}
        """
        
        extraction = self.llm.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": prompt}],
            response_format={"type": "json_object"}
        )
        
        data = json.loads(extraction.choices[0].message.content)
        
        for item in data.get("memories", []):
            self.long_term.add_memory(
                item["content"],
                memory_type=item.get("type", "fact"),
                user_id=user_id
            )

# 使用示例
config = {
    "short_term_max_tokens": 4000,
    "agent_profile_path": "agent_profile.yaml",
    "collection_name": "user_memory",
    "persist_directory": "./memory_db"
}

memory = ThreeLayerMemory(config)

# 对话
response = memory.chat("你好，我叫张三", user_id="user_123")
print(response)

response = memory.chat("我在北京工作，是个软件工程师", user_id="user_123")
print(response)

response = memory.chat("我想知道如何学习 Python", user_id="user_123")
print(response)

# 下次对话时，之前的信息会被自动检索和使用
```

## 6. 总结

### 6.1 三层记忆对比

| 维度 | 短期记忆 | Profile 记忆 | 长期记忆 |
|------|---------|-------------|---------|
| **内容** | 当前对话 | Agent 性格、用户特征 | 事实、经历、知识 |
| **访问速度** | 极快 | 快 | 较慢（需检索） |
| **容量** | 有限（4K-8K tokens） | 中等（1K-3K） | 无限 |
| **持久性** | 会话级 | 永久（手动更新） | 永久 |
| **更新频率** | 每轮对话 | 低（按需） | 中等 |
| **实现方式** | 滑动窗口 | YAML/JSON 文件 | 向量数据库 |
| **成本** | 每次携带 | 一次性加载 | 检索成本 |

### 6.2 最佳实践

```
1. 分层管理
   - 不同类型信息使用不同记忆层
   - 避免所有信息都放入短期记忆

2. 动态平衡
   - 根据任务调整各层比例
   - 对话密集型：增加短期记忆
   - 知识密集型：增强长期记忆

3. 智能提取
   - 自动从对话提取关键信息
   - 定期清理过期记忆

4. Profile 优先
   - 始终加载 Profile 记忆
   - 确保一致的人设和用户理解

5. 长期记忆优化
   - 定期清理无用记忆
   - 使用重要性评分
   - 实现遗忘机制
```
