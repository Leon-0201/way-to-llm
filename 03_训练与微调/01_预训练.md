# 3.1 预训练

## 1 定位与价值
预训练是大模型能力的主要来源，通过大规模无标注数据学习通用语言/多模态知识，为后续微调、对齐打下基础，成本占大模型全生命周期的80%以上，直接决定模型的能力上限。
## 2 核心底层理论：Scaling Law（缩放定律）
- 核心结论：大模型的效果与三个核心因素呈对数线性正相关，三者需要同比例缩放才能达到最优效率：
  1. 模型参数量N：模型越大，能力上限越高
  2. 训练数据量D：训练Token越多，知识覆盖越全面
  3. 训练计算量C：算力投入越大，训练越充分
- 实践指导价值：预训练前需要合理配比三者规模，避免资源浪费：小模型配过大的数据集/大模型配过小的数据集都无法达到最优性价比，三者同步提升才能实现能力的线性增长
- 典型配比参考：7B模型需匹配万亿Token级数据、70B模型需匹配3-5万亿Token级数据，才能充分释放模型能力
## 3 数据全流程处理
- 数据收集：
  - 数据源：网页、书籍、论文、代码、多模态数据等，覆盖不同领域、不同语言，保证数据多样性
  - 规模：7B模型需万亿Token级数据，70B模型需数万亿Token级数据，数据规模和模型能力正相关
  - 污染防控：收集阶段提前排除各类公开评测集数据，避免数据污染导致下游任务评测虚高、泛化能力下降；数据处理后需做污染检测，通过n-gram比对与公开测试集去重
- 数据清洗：
  - 基础清洗：去重、去低质内容（广告、垃圾信息）、去敏感内容、规范化格式
  - 质量过滤：
    - 规则过滤：基于长度（过滤过短/过长文本）、重复率（过滤重复字符/段落占比过高内容）
    - PPL过滤：用小型预训练模型计算文本困惑度，过滤PPL过高（乱码/不符合语言规律）和PPL过低（过于简单的重复内容）的异常文本
    - 模型过滤：用LLM-as-Judge对内容的信息量、正确性、有用性打分，过滤低分内容；采用关键词+分类模型两级过滤敏感/有害内容
  - 去重方法：
    - 精确去重：匹配完全一致的文本，去除完全重复内容
    - MinHash/LSH模糊去重：检测相似度较高的近似内容，控制整体语料重复率<1%
    - 重复数据危害：导致模型过拟合、记忆重复内容、泛化能力下降、生成时容易出现重复输出
- 数据配比：
  - 通用大模型典型配比：网页50-60%、书籍15-20%、代码10-15%、专业领域数据5-10%，避免某类数据占比过高导致能力偏科
  - 领域预训练配比：通用基础数据70-90% + 领域专业数据10-30%，避免领域数据占比过高导致通用能力灾难性遗忘
  - 高质量数据可以适当提高配比，低质数据即使量大也不会提升效果，甚至会损害模型能力
## 4 训练目标
- 核心目标：Next Token Prediction（下一词预测），给定上文，预测下一个Token的概率分布，是Decoder-only架构的标准训练目标，让模型学习语言规律、知识、逻辑推理能力
- 辅助目标：部分模型会加入Masked Language Modeling（MLM）、对比学习等辅助目标提升通用能力
- 多模态扩展：多模态模型会加入图文匹配、文本生成图像等辅助目标
## 5 工程实践核心要点
### 5.1 核心超参数最佳实践
- 学习率策略：采用warmup + 余弦退火组合，warmup步数通常为总步数的1-5%，峰值学习率根据模型规模调整（7B模型约2e-4 ~ 5e-4，70B模型约1e-4 ~ 3e-4）
- 优化器：默认采用AdamW，betas通常设置为(0.9, 0.95)，权重衰减系数0.1，eps=1e-8
- Batch size：与模型规模匹配，7B模型单A100卡batch size约为2-4M Token，全局batch size通常为2-8M Token
- 训练epoch：通用预训练通常为1-2个epoch，高质量领域数据可以增加到2-3个epoch，避免过多重复导致过拟合
### 5.2 关键工程优化手段
- 混合精度训练：采用BF16/FP16混合精度，BF16优先（无溢出风险），配合损失缩放解决梯度下溢问题，可提升训练速度30%以上，显存占用降低约50%
- 算子加速：FlashAttention/FlashAttention2优化注意力计算，速度提升2-4倍，显存占用大幅降低；融合算子减少内核调用开销
- 显存优化：梯度累积（用时间换空间，多次step累计梯度等效增大batch size）、激活重计算（重计算激活值而非存储，显存占用降低40%+，速度损失约20%）、梯度裁剪（防止梯度爆炸，阈值通常设为1.0）
- Checkpoint策略：定期保存checkpoint，支持断点续训，避免训练中断丢失进度
### 5.3 分布式训练并行策略
预训练通常采用3D并行组合最大化训练效率：
- 数据并行：默认采用ZeRO-1/2优化，跨节点分布数据，增大全局batch size
- 张量并行：单节点内8卡做张量并行，拆分注意力和FFN层参数，降低单卡显存压力
- 流水线并行：跨节点拆分模型层，减小单卡需要存储的模型参数规模，适合70B+超大模型
- 通信优化：采用InfiniBand/RoCE高速网络，减少多卡/多节点通信开销
### 5.4 预训练成本估算
核心成本为算力成本，辅助成本为数据成本、人力成本：
- 7B模型：万亿Token预训练约需32-64张A100，训练1-2周，算力成本约30-80万元
- 70B模型：3-5万亿Token预训练约需256-512张A100，训练1-2个月，算力成本约500-1500万元
- 成本优化：抢占式实例、混合云调度可降低30-50%算力成本
## 6 领域预训练（持续预训练）实践
- 适用场景：在通用预训练模型基础上注入领域专业知识，提升法律、医疗、金融、代码等垂直领域任务效果，相比全量从零预训练成本低90%以上
- 核心最佳实践：
  1. 数据配比：通用基础数据70-90% + 领域专业数据10-30%，避免领域数据占比过高导致通用能力灾难性遗忘
  2. 超参数设置：学习率为通用预训练的1/10左右（通常为1e-5 ~ 5e-5），避免大幅破坏模型已学到的通用知识
  3. 训练步数：根据领域数据规模调整，通常为几千到几万步，以领域验证集指标不再提升为停止标准
  4. 效果验证：同时评测领域任务效果和通用能力基准，避免领域能力提升的同时通用能力大幅下降
## 7 训练曲线与收敛判断
- 核心观测指标：训练集Loss、验证集Loss、下游任务评测指标
- 收敛判断标准：
  1. 验证集Loss连续多个step不再下降，反而上升，是最核心的收敛指标
  2. 训练集Loss下降到接近理论下限，不再明显下降
  3. 下游任务评测指标不再提升，模型能力达到瓶颈
- 常见异常诊断：
  - 验证集Loss先降后升：过拟合，需加正则化、增大数据量、提前停止训练
  - Loss持续震荡：学习率过高、batch size过小，需要降低学习率或增大batch size
  - Loss不下降：学习率过低、数据质量差、模型初始化问题、分布式通信异常
  - Loss spike（突然暴涨后回落或不回落）：常见原因包括训练数据出现异常脏数据、学习率过高导致梯度爆炸、分布式节点通信故障、混合精度溢出；排查路径：先检查对应step的训练数据是否有异常，再检查梯度范数是否正常，最后排查分布式通信日志

## 面试高频考点
### 算法岗通用题
#### Q1：**预训练数据处理的核心步骤是什么？**
- 数据收集：覆盖多领域多类型数据源，保证规模和多样性，同时防控数据污染
- 数据清洗：通过精确/模糊去重、规则/模型/PPL多层过滤，去低质、去敏感内容，严格保证数据质量，控制重复率<1%
- 数据配比：按场景需求合理混合不同类型数据，避免能力偏科
#### Q2：**大模型预训练的核心目标是什么？为什么Next Token Prediction可以让模型学到通用知识？**
核心目标是让模型学习大规模数据中的语言规律、世界知识、逻辑推理能力。下一词预测要求模型完全理解上文的语义、逻辑、隐含知识才能准确预测下一个词，在万亿级Token上训练后，模型自然学到了通用的语言能力和世界知识。
#### Q3：**预训练常用的工程优化手段有哪些？分别解决什么问题？**
- 混合精度训练：用BF16/FP16混合计算，提升训练速度30%以上，降低显存占用约50%
- FlashAttention：优化注意力计算，速度提升2-4倍，大幅降低显存占用
- 梯度累积：通过多次step累计梯度等效增大batch size，解决单卡显存不足问题
- 激活重计算：重计算激活值而非存储，降低显存占用40%+，适合超大模型训练
- 3D并行：数据+张量+流水线并行组合，支持上千卡分布式训练，提升超大模型训练效率
#### Q4：**领域预训练（持续预训练）需要注意什么？如何避免灾难性遗忘？**
- 数据配比：通用数据占70-90%，领域数据占10-30%，避免领域数据过多冲刷通用知识
- 学习率：用通用预训练1/10左右的小学习率，避免大幅修改模型权重
- 效果验证：同时监控领域任务指标和通用能力基准，保证领域能力提升的同时通用能力下降在可接受范围
#### Q5：**预训练数据质量和数据量哪个更重要？如何平衡？**
数据质量优先级远高于数据量，低质数据甚至会损害模型能力。平衡方法：
- 先保证数据质量，严格清洗过滤，再扩大数据规模
- 高质量数据可以重复训练（多次epoch），低质数据即使量大也没用
- 领域数据不足时优先提升质量，而不是凑规模
#### Q6：**如何判断预训练是否收敛？有哪些关键观测指标？**
核心看三个维度：
- 验证集Loss连续多个step不再下降，是最核心的收敛指标
- 下游任务评测指标不再提升，验证模型实际能力不再增长
- 训练集Loss下降到合理区间，没有明显震荡或上升。如果验证集Loss上升而训练集Loss还在降，说明已经过拟合，需要停止训练。
#### Q7：**Scaling Law的核心内容是什么？对预训练有什么指导意义？**
Scaling Law核心结论是大模型效果与参数量、训练数据量、计算量三者呈对数线性正相关，三者需要同比例缩放才能达到最优效率。指导意义：预训练前需要合理配比三者规模，避免小模型配过大数据集/大模型配过小数据集的资源浪费，三者同步提升才能实现能力的线性增长。比如7B模型需要匹配万亿Token级数据，70B模型需要匹配3-5万亿Token级数据才能充分释放能力。
#### Q8：**预训练数据重复会带来什么问题？如何防控？**
数据重复会导致模型过拟合、记忆重复内容、泛化能力下降、生成时容易重复。防控方法：
- 精确去重匹配完全一致的文本
- 用MinHash/LSH做模糊去重，检测相似度较高的近似内容
- 控制整体语料重复率在1%以内
- 训练epoch控制在1-2个，高质量数据不超过3个epoch
#### Q9：**什么是数据污染？有什么危害？如何避免？**
数据污染是指训练数据中包含下游评测集的数据，会导致模型在这些评测集上的结果虚高，实际泛化能力并未提升，无法正确评估模型能力。避免方法：
- 数据收集阶段提前排除各类公开评测集数据
- 数据处理后通过n-gram比对与公开测试集去重
- 评测前检查测试集数据是否出现在训练语料中
